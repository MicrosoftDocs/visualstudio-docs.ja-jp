# <a name="vs-connected-environment-for-azure-container-service-aks-preview"></a>Azure Container Service (AKS) 用の VS Connected Environment (プレビュー)
VS Connected Environment は、Kubernetes ですばやく開発するのに役立ちます。 Azure で完全に管理され、開発用に最適化された Kubernetes ベースの環境を作成してから、VS Code、Visual Studio、コマンド ラインなどの使い慣れたツールを使用して、クラウドでコンテナーを繰り返し開発します。

この方法には、次のようないくつかの利点があります。

* クラウド リソースにフル アクセスできる、インフラストラクチャレスの開発環境 (代表的な運用環境) が得られる。
* VS Code または Visual Studio を使用して、Kubernetes で直接 Node.js コンテナーと .NET Core コンテナーをデバッグできる。 その他のすべての言語をコマンド ライン インターフェイスで開発できます。
* 開発チーム全体で Kubernetes インスタンスを共有してコストを節約し、新しいチーム メンバーのためのローカル コンピューターのセットアップを最小化できる。
* コードを個別に開発し、依存関係のレプリケートやモックアップの作成なしで、他のコンポーネントでエンド ツー エンド テストを実行できる。
