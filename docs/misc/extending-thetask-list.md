# タスク一覧の拡張
[!INCLUDE[vsprvs](../code-quality/includes/vsprvs_md.md)] の**タスク一覧**を使用すると、ユーザーは、カスタム プログラミング タスクの追加、コードの行にリンクするタスク コメントの表示、タスク メッセージのカスタム カテゴリの作成および表示を行うことができます。  
  
 タスクは、<xref:Microsoft.VisualStudio.Shell.Interop.IVsTaskList> と <xref:Microsoft.VisualStudio.Shell.Interop.IVsTaskList2> を実装する <xref:Microsoft.VisualStudio.Shell.Interop.SVsTaskList> という名前のサービスによって処理されます。 基本的な**タスク一覧**機能を使用するには、<xref:Microsoft.VisualStudio.Shell.Interop.IVsTaskProvider> を実装してタスク プロバイダーを作成する必要があります。  
  
 <xref:Microsoft.VisualStudio.Shell.Interop.IVsTaskList.RegisterTaskProvider%2A> を呼び出して <xref:Microsoft.VisualStudio.Shell.Interop.SVsTaskList> サービスにタスク プロバイダーを登録します。これにより、後続のすべてのトランザクションでタスク プロバイダーを一意に識別するために使用する必要がある Cookie の値が返されます。  
  
 すべてのタスク プロバイダーの実装でタスクの内部リストが保持されます。 タスク プロバイダーは、**タスク一覧**に対して <xref:Microsoft.VisualStudio.Shell.Interop.IVsTaskList.RefreshTasks%2A> を呼び出して、表示されるタスク一覧を更新できます。 この場合、次の処理が実行されます。  
  
1.  サービスは、<xref:Microsoft.VisualStudio.Shell.Interop.IVsTaskProvider.EnumTaskItems%2A> メソッドを使用してタスク プロバイダーにコールバックします。  
  
2.  <xref:Microsoft.VisualStudio.Shell.Interop.IVsTaskProvider.EnumTaskItems%2A> のタスク プロバイダーの実装から <xref:Microsoft.VisualStudio.Shell.Interop.IVsEnumTaskItems> オブジェクトが返されます。  
  
3.  <xref:Microsoft.VisualStudio.Shell.Interop.IVsEnumTaskItems> オブジェクトは、<xref:Microsoft.VisualStudio.Shell.Interop.IVsTaskItem> オブジェクトのコレクションを反復処理します。  
  
4.  <xref:Microsoft.VisualStudio.Shell.Interop.SVsTaskList> サービスによって**タスク一覧**の表示が更新されます。  
  
 その他の機能も使用できます。<xref:Microsoft.VisualStudio.Shell.Interop.IVsTaskProvider3> インターフェイスを使用すると、すべてのタスク プロバイダーは列のカスタム セットを独自に提供できます。
