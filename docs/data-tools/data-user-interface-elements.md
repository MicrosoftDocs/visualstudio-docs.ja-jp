# データ ユーザー インターフェイス要素
このセクションでは、[!INCLUDE[vbprvb](../code-quality/includes/vbprvb_md.md)] または Visual C\# アプリケーションでのデータ アクセスをデザインするときに使用するすべてのダイアログ ボックスとウィザードについて説明します。  
  
 [!INCLUDE[note_settings_general](../data-tools/includes/note_settings_general_md.md)]  
  
## このセクションの内容  
 [Data Smart Tags](http://msdn.microsoft.com/ja-jp/1e0a848f-c57b-47ab-b884-eaaa40726f43)  
 データに対して使用できるスマート タグ コマンドについて説明します。  
  
 [&#91;データセットの追加&#93; ダイアログ ボックス](http://msdn.microsoft.com/ja-jp/0e03c0ff-212b-4bfa-ac51-3c2adb71ead0)  
 既存の型指定されたデータセットまたは新しい型指定されていないデータセット \(**System.Data.Dataset** クラスのインスタンス\) をフォームまたはコンポーネントに配置します。  
  
 [&#91;SQL 生成の詳細オプション&#93; ダイアログ ボックス](http://msdn.microsoft.com/ja-jp/41420450-1ff4-4a1a-b85b-6f6901538fef)  
 更新、オプティミスティック同時実行制御に対するオプションを指定し、データセットを更新することによって、データ アダプターに対する SQL ステートメントおよびストアド プロシージャの作成方法を制御できます。  
  
 [Choose a Data Source Instance Dialog Box](http://msdn.microsoft.com/ja-jp/51c47f06-fdc5-453e-9178-0a5a2c5c9f34)  
 プロジェクトのデータセットの一覧から目的のデータセットを選択できます。  
  
 [Choose a Data Source to Merge with Dialog Box](http://msdn.microsoft.com/ja-jp/accafff7-f6bd-481c-a121-fe8a76cd681d)  
 複数のデータ ソースを使用できる場合に、マージするデータ ソースを選択できます。  
  
 [Choose Key Dialog Box](http://msdn.microsoft.com/ja-jp/4ddbfbb7-a80a-412a-b80d-291d86376ca3)  
 列が複数キー制約に関与する場合に、キーを選択できます。  
  
 [コレクション エディター](../Topic/Collection%20Editors.md)  
 コレクションの各メンバーを作成および編集できるさまざまなコレクション エディターに関するトピックへのリンクを提供します。  
  
 [Connections Missing](http://msdn.microsoft.com/ja-jp/bb9b2e12-7f76-4ee5-acbb-5d20116ee044)  
 アプリケーション設定に存在しない接続文字列への参照がデータセットに含まれている場合に通知します。  
  
 [&#91;データ アダプター 構成エラー&#93; ダイアログ ボックス](http://msdn.microsoft.com/ja-jp/9ce65cd2-0c7d-4f51-8685-d68be5f3009b)  
 Visual Studio がデータ アダプターのインスタンスの作成とプロパティの設定を試行している間に発生した 1 つ以上のエラーを表示します。  
  
 [データアダプター構成ウィザード](http://msdn.microsoft.com/ja-jp/efff90cb-0e4c-4eb3-87dc-65dd9d418809)  
 データ アダプターがデータベースからデータセットにデータを読み込んだり、再び書き戻したりするために使用する SQL コマンドやストアド プロシージャを設定します。  このトピックでは、ウィザードの実行方法およびウィザードが完了した後の操作について説明します。  
  
 [&#91;データ アダプター プレビュー&#93; ダイアログ ボックス](http://msdn.microsoft.com/ja-jp/1f614cd3-4530-457e-84af-00ccbaea08cc)  
 データ アダプターによってデータセットにデータが入力される方法を確認できます。これは、アダプターが目的のデータを返すこと、テーブル マッピングが正しく動作すること、およびさまざまなパラメーター値の効果をテストする場合に役立ちます。  
  
 [&#91;データ ソース ログイン&#93; ダイアログ ボックス](http://msdn.microsoft.com/ja-jp/6f2d9a57-53c3-4841-bd37-a3643eb68d2e)  
 まだ認証されていないデータ ソース \(通常はデータベース\) へのアクセスを要求できます。  
  
 [DataSet Tab, Toolbox](http://msdn.microsoft.com/ja-jp/fa5f2d6f-924d-4262-ba1b-e9e7f90e7764)  
 型指定されたデータセットに追加できるオブジェクトを表示します。  
  
 [Do you want to include the password in the connection string Dialog Box](http://msdn.microsoft.com/ja-jp/193696a7-5213-4396-8328-05ac2df6ee94)  
 接続文字列にパスワードを埋め込むかどうかを制御できます。  
  
 [Edit Key Dialog Box](http://msdn.microsoft.com/ja-jp/f5c80e39-3a42-4284-b222-6ca009fd9675)  
 キーを定義および編集できます。  
  
 [&#91;外部キー制約&#93; ダイアログ ボックス](http://msdn.microsoft.com/ja-jp/45d15629-1f4d-40a7-8708-c9ddfebedc1e)  
 別のテーブルに関連付けられたデータセット テーブルの 1 つ以上の列に外部キー制約を設定できます。  
  
 [&#91;データセットの生成&#93; ダイアログ ボックス](http://msdn.microsoft.com/ja-jp/c0efdbaf-13b1-4ee8-ade6-f8a784126cdc)  
 1 つ以上のデータ アダプターによって提供された情報から新しい型指定されたデータセットを生成したり、既存のデータセットにテーブルを追加したりできます。  
  
 [Multiple BindingSources Dialog Box](http://msdn.microsoft.com/ja-jp/db76f70c-4fb5-479d-9b64-a67158d48f97)  
 複数の <xref:System.Windows.Forms.BindingSource> が使用可能な場合に、使用する <xref:System.Windows.Forms.BindingSource> を選択できます。  
  
 [Preview Data Dialog Box](http://msdn.microsoft.com/ja-jp/aa4f0d04-2695-4bb8-946d-54a97ae7287f)  
 プロジェクトのクエリによって返されたデータを確認できます。  
  
 [&#91;SQL スクリプトのプレビュー&#93; ダイアログ ボックス](http://msdn.microsoft.com/ja-jp/e9571e8b-821c-492d-9bc8-b44eba898bdd)  
 [データアダプター構成ウィザード](http://msdn.microsoft.com/ja-jp/efff90cb-0e4c-4eb3-87dc-65dd9d418809)の一部として表示され、データの読み書きを行うストアド プロシージャを作成するためにウィザードで使用される SQL スクリプトを確認できます。  
  
 [&#91;リレーションシップ&#93; ダイアログ ボックス](http://msdn.microsoft.com/ja-jp/ab8f4b0e-af4c-4725-a550-e2b2ebe43a02)  
 データセット内の 2 つのデータ テーブルの親レコードと子レコードに関する情報を保持するリレーションシップ \(**DataRelation** オブジェクト\) を作成できます。  
  
 [ダイアログ ボックス](../Topic/Search%20Criteria%20Builder%20Dialog%20Box.md)  
 データ バインド Windows フォームにパラメーター クエリを作成し、クエリの実行に必要なコントロールを自動的に追加できます。  
  
 [&#91;テーブル マップ&#93; ダイアログ ボックス](http://msdn.microsoft.com/ja-jp/fb4cec1e-f3c8-4773-b409-c2de15293fea)  
 データベース テーブルまたは他のデータ ソースのどの列がデータセット テーブルの列に対応するかを指定できます。  
  
 [&#91;一意の制約&#93; ダイアログ ボックス](http://msdn.microsoft.com/ja-jp/e71a60d7-fae2-4bd0-a1e8-43aae351707d)  
 型指定されていないデータセットのテーブルの 1 つ以上の列に UNIQUE 制約を設定できます。
