# <a name="source-control-plug-in-glossary"></a>ソース管理プラグインの用語集
次の役立つ用語と定義は、ソース管理プラグインの追加の SDK のドキュメントに関するものです。  
  
## <a name="definitions"></a>定義  
 チェックイン  
 ユーザーが作業用コピーを変更すると、ユーザーは、サーバーの全体のソース管理リポジトリに作業コピーからの変更を送信する必要があります。 これには、他のユーザーに提供されるファイルの新しいリビジョンが作成されます。 このプロセスには、チェックインは呼び出されます。  
  
 チェックアウト  
 通知を修正して、目的のリポジトリ、リポジトリから作業用コピーを要求する操作。 作業コピーは、プロジェクトはチェック アウト時点の状態を反映します。  
  
 クライアント  
 ソース コード管理システムを使用するプログラムです。 このドキュメントでは、Visual Studio IDE になります。  
  
 コメント  
 ソース管理操作を実行すると、ユーザーをリビジョンにアタッチできる変更を説明するメッセージ。  
  
 競合  
 2 人のユーザーが同じファイルのリージョンと同じリージョンの変更をチェックインするときの状況です。 通常、マージを実行する必要があります。  
  
 ディレクトリ  
 クライアント側のローカル フォルダーは、"ディレクトリ"と呼ばれます。 これは、ユーザーが実際に変更をコピーします。 特定のプロジェクトの多くの作業用コピーが存在することができます。通常各開発者は、自分のコピーを持っています。  
  
 Get  
 Get 操作では、ユーザーの作業用コピーを最新リポジトリのコピーが表示されます。 チェック アウトとは異なり、ユーザーは単に最新のコピーを必要がありますが、変更することなくがしようとしています。 ときに、get が実行されます。  
  
 履歴  
 通常これは、すべてのチェック アウト、チェックイン、更新プログラム、タグ、およびソース管理リポジトリでのリリースの概要です。  
  
 IDE  
 通常 Visual Studio 統合開発環境を指します。 ただし、ソース管理プラグインの API を認識するクライアント環境を参照してもします。  
  
 マージ  
 2 つまたは複数ソースの中には、コード ファイルが以前のファイルのすべての機能が組み込まれた新しいファイルに組み合わされたプロセスです。 この概念は、ここで&2; つ以上開発者の作業をファイルに同時にバージョン管理では必要不可欠です。  
  
 プロジェクト  
 ソース管理フォルダーは、プロジェクトとも呼ばれます。 Visual Studio では、プロジェクトやソリューションとのリレーションシップがありません。  
  
 プラグイン  
 ソース管理プラグインの API を実装することで、ソース管理機能を提供する DLL です。  
  
 リポジトリ  
 マスター コピーをソース管理システムでは、プロジェクトの詳細な改訂履歴を格納します。 各プロジェクトには、ただ&1; つのリポジトリがあります。  
  
 Revision  
 ファイルの履歴またはファイルのセット、コミットされた変更します。 改訂版はいずれかのスナップショットでは、プロジェクトの継続的に変化します。
