# <a name="source-control-plug-in-glossary"></a>ソース管理プラグインの用語集
次の役に立つ用語と定義は、ソース管理プラグイン SDK のドキュメントに関連しています。

## <a name="definitions"></a>定義
 チェックイン ユーザーが作業コピーに変更を加えた場合、ユーザーは作業コピーから中央ソース管理リポジトリに変更を送信する必要があります。 これにより、他のユーザーが使用できるファイルの新しいリビジョンが作成されます。 このプロセスはチェックインと呼ばれます。

 チェックアウト リポジトリから作業コピーを要求し、リポジトリに変更の意図を通知する行為。 作業コピーは、チェックアウトされた時点のプロジェクトの状態を反映します。

 クライアント ソース コード管理システムを使用するプログラム。 このドキュメントの目的のために、これは、Visual Studio IDE です。

 コメント ソース管理操作の実行時にユーザーがリビジョンに適用できる変更を説明するメッセージ。

 競合 2 人のユーザーが同じファイルの同じ領域に変更をチェックインしようとする状況。 通常、マージを実行する必要があります。

 ディレクトリ クライアント側のローカル フォルダは、ディレクトリと呼ばれます。 これは、ユーザーが実際に変更を加えるコピーです。 特定のプロジェクトの作業コピーは多数存在する場合があります。一般的に、各開発者は自分のコピーを持っています。

 取得操作では、リポジトリ コピーを使用して、ユーザーの作業コピーを最新の状態にします。 チェックアウトとは異なり、ユーザーが最新のコピーを必要としている場合に get が実行されますが、変更は行いません。

 履歴 ソース管理リポジトリで行われたすべてのチェックアウト、チェックイン、更新、タグ、およびリリースの概要です。

 IDE 一般に、Visual Studio 統合開発環境を指します。 ただし、ソース管理プラグイン API を認識する他のクライアント環境を参照することもできます。

 Merge 2 つ以上のソース コード ファイルを結合して、以前のファイルのすべての機能を組み込んだ新しいファイルを作成するプロセス。 この概念は、複数の開発者が同時にファイルを処理するバージョン管理において重要です。

 プロジェクト ソース管理フォルダーは、多くの場合、プロジェクトと呼ばれます。 これは、Visual Studio のプロジェクトやソリューションとは関係ありません。

 プラグイン ソース管理プラグイン API を実装することにより、ソース管理機能を提供する DLL。

 リポジトリ ソース管理システムがプロジェクトの完全なリビジョン履歴を格納するマスタ コピー。 各プロジェクトには、1 つのリポジトリがあります。

 リビジョン ファイルまたはファイルのセットの履歴にコミットされた変更。 リビジョンは、継続的に変化するプロジェクトのスナップショットの 1 つです。
