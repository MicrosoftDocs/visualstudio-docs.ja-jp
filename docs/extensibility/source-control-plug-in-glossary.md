# <a name="source-control-plug-in-glossary"></a>ソース管理プラグインの用語集
次の便利な用語と定義は、ソース管理プラグイン SDK のドキュメントに適用されます。  
  
## <a name="definitions"></a>定義  
 チェックイン  
 ユーザーが作業コピーを変更すると、ユーザーは、サーバーの全体のソース管理リポジトリへの作業コピーからの変更を送信する必要があります。 これには、他のユーザーに提供されるファイルの新しいリビジョンが作成されます。 このプロセスには、チェックインは呼び出されます。  
  
 チェック アウト  
 作業コピーを要求するために変更するユーザーの意図のリポジトリを通知する、リポジトリからの操作。 作業コピーは、プロジェクトがチェック アウトされて時点の状態を反映します。  
  
 クライアント  
 ソース コード管理システムを使用するプログラム。 このドキュメントでは、Visual Studio IDE を勧めします。  
  
 コメント  
 ソース管理操作を実行すると、リビジョンにユーザーをアタッチできる変更を説明するメッセージ。  
  
 競合  
 2 人のユーザーが同じファイルの同じリージョンに変更をチェックインするときの状況です。 通常、マージを実行する必要があります。  
  
 ディレクトリ  
 クライアント側のローカル フォルダーは、"ディレクトリ"と呼ばれます。 これは、ユーザーが実際に変更をコピーします。 特定のプロジェクトの多くの作業コピーがあります。通常各開発者は、自分のコピーを持っています。  
  
 Get  
 Get 操作では、ユーザーの作業コピーを最新のリポジトリのコピーが表示されます。 チェック アウトとは異なりは、ユーザーだけで最新のコピーを必要がありますが、変更しないようにしているつもりの get は実行されます。  
  
 履歴  
 すべてのチェック アウト、チェックイン、更新プログラム、タグ、およびソース管理リポジトリでのリリースの概要では通常です。  
  
 IDE  
 通常、Visual Studio 統合開発環境を指します。 ただしも参照して、ソース管理プラグイン API を認識する他のクライアント環境です。  
  
 マージ  
 以前のファイルのすべての機能が組み込まれている新しいファイルを形成する 2 つ以上のソースの中にコード ファイルの結合するプロセス。 この概念が、ここで 2 つ以上開発者の作業をファイルに同時にバージョン管理で重要です。  
  
 プロジェクト  
 ソース管理フォルダーは、プロジェクトとしても呼ばれます。 Visual Studio では、プロジェクトやソリューションとのリレーションシップがありません。  
  
 プラグイン  
 ソース管理プラグイン API を実装することでソース管理機能を提供する DLL です。  
  
 リポジトリ  
 マスター コピーをソース管理システムでは、プロジェクトのすべてのリビジョン履歴を格納します。 各プロジェクトには、1 つだけのリポジトリがあります。  
  
 Revision  
 ファイルの履歴またはファイルのセットにコミットされた変更します。 リビジョンは、いずれかの継続的に変化するプロジェクトでのスナップショットです。
