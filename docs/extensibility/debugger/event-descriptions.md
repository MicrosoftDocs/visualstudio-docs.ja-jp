# <a name="event-descriptions"></a>イベントの説明
各イベントの種類は、特定の目的がします。  
  
## <a name="events-and-the-reasons-for-their-use"></a>イベントと、使用する理由  
  
|イベント|説明|  
|-----------|-----------------|  
|ドキュメント イベントをアクティブ化します。|デバッグ エンジン (DE) が、IDE を開くか、前面にドキュメントを表示するときに発生します。|  
|バインドされているブレークポイントまたはブレークポイント エラー イベント|ブレークポイントがバインドされているか、ブレークポイントをバインドできません。 し、エラーが返されますと送信されます。|  
|ブレークポイントがバインドされていないイベント|バインドされたブレークポイントをコードからバインド解除すると発生します。|  
|イベントを停止することができます。|ユーザー コードの特定の時点で停止するかどうかを決定する、IDE に送信されます。|  
|ブレークポイント イベント|コードまたはデータのブレークポイントがヒットしたときに発生します。|  
|ドキュメントのテキストのイベント|ドキュメント内のテキストが変更されたときに発生します。 これらのイベントは、経由で送信されません、`IDebugEventCallBack2::Event`メソッドです。|  
|エンジンは、イベントを作成します。|エンジンが最初に作成されたときに送信します。|  
|エントリ ポイント イベント|デバッグ中のプログラムがその初期化コードを実行し、その最初のユーザー エントリ ポイントに達すると送信されます。|  
|例外イベント|実行中のプログラムに例外が発生すると送信されます。|  
|式の評価の完了イベント|非同期の式の評価が完了すると送信されます。|  
|シンボルのイベントを検索します。|モジュールのシンボルを検索するユーザーに確認する必要があります、DE たびに送信されます。|  
|読み込み完了イベント|のみとプログラムを初期ロードが完了して最初のコードが、プログラムの実行前に送信されます。|  
|メッセージ イベント|ユーザーに送信されたメッセージは送信されます。|  
|モジュール読み込みイベント|新しいモジュールがロードまたはアンロードされるときに送信します。|  
|文字列の出力イベント|デバッグ出力を書き込むときに送信します。|  
|作成し、破棄イベント|Visual Studio IDE を追跡できますデバッグ中のプログラムの状態、作成またはプロセス、プログラム、プロパティ、セッション、およびスレッドの破棄をアナウンスに送信されます。|  
|手順の完了イベント|ステップが完了すると送信されます。|  
|スレッド名の変更イベント|ユーザーが、スレッドの名前を変更したときに送信されます。|  
|プログラム名の変更イベント|ユーザーがプログラムの名前を変更したときに送信されます。|
