# <a name="visual-studio-debugger-glossary"></a>Visual Studio デバッガーの用語集
デバッグ SDK で使用される[!INCLUDE[vsprvs](../../../code-quality/includes/vsprvs_md.md)]用語を次に示します。

## <a name="terms"></a>用語
 バインドされたブレークポイント コードに設定されたブレークポイントの抽象化。 バインドされたブレークポイントとコード ストリーム内のブレークポイント命令の間には、1 対 1 の関係があります。 コードがアンロードされると、バインドされたブレークポイントがバインド解除される場合があります。

 因果関係 複数の物理スレッド、プロセス、およびマシン間で実行の論理スレッドを追跡し、そのスレッドの有効期間内の任意の時点でその論理スレッドの呼び出し履歴を再構築する機能を提供します。

 コード コンテキスト デバッグ エンジンに既知のコード内の位置の抽象化を提供します。 ほとんどの実行時アーキテクチャでは、コード コンテキストはプログラムの命令ストリーム内のアドレスです。 コードが命令で表されない非伝統的言語の場合、コード コンテキストは他の手段で表される場合があります。

 コード パス分岐が行われるか、または関数呼び出しが行われるコード内の実行ポイントを表します。 スタック トレースは、基本的に関数呼び出しコード パスの一覧です。

 デバッグ エンジン (DE) ランタイム アーキテクチャのデバッグを可能にするコンポーネント。 デバッグ エンジンは、インタプリタまたはオペレーティング システムと連携して動作し、実行制御、ブレークポイント、式の評価などのデバッグ サービスを提供します。

 ドキュメント コンテキスト デバッグ エンジンに既知のソース ファイル ドキュメント内の位置の抽象化を提供します。 ほとんどの言語では、ドキュメントコンテキストはソースファイル内の位置です。 ソース ファイルがテキストでない非伝統的な言語の場合、ドキュメント コンテキストは他の方法で表される場合があります。 *「ドキュメントの位置*」も参照してください。

 ドキュメント位置 IDE に認識されているソース ファイル内の位置を抽象化します。 ほとんどの言語では、ドキュメントの位置はソース ファイル内の位置です。 非伝統的言語では、ドキュメントの位置は他の方法で表される場合があります。 ドキュメント*コンテキスト*も参照してください。

 エラー ブレークポイント 保留中のブレークポイントでエラーを記述するための抽象化。 エラー ブレークポイントは、保留中のブレークポイントの場所、保留中のブレークポイントに関連付けられた式、または保留中のブレークポイントがコードの場所にバインドできないその他の情報のエラーを記述できます。

 評価コンテキスト 式評価用のプログラミング コンテキストの抽象化を提供します。 通常、評価コンテキストはスコープです。 式コンテキストで式の評価を行う場合、式コンテキストは作成のポイントに一致するスコープルールを提供します。 たとえば、スタック フレームで作成された式コンテキストは、ローカル変数、メソッド パラメーター、クラス メンバー (該当する場合)、およびグローバル変数を評価するためのコンテキストを提供します。

 インターセプトされた例外 現在のスタック フレームに例外処理機構が存在しない場合でも、デバッグ エンジンによってインターセプトされる例外。

 JustMyCode ユーザーに属するコードのみをデバッグし、システム コードなどの中間コードを無視するという概念です。

 保留中のブレークポイント : コードの読み込み前、読み込み中、および読み込み後のブレークポイントの抽象化と、ブレークポイントを仮想化する方法を提供します。 保留中のブレークポイント:

- 1 つ以上のプログラムのコードにブレークポイントをバインドするために必要なすべての情報が含まれています。

- 1 つまたは複数のプログラムの複数のコードの場所にバインドできます。

- コードにバインドしないでください。

  コードが読み込まれるたびに、プログラム内のすべての保留中のブレークポイントがチェックされ、バインドできるかどうかが確認されます。 保留中のブレークポイントには、バインドされたすべてのバインドされたブレークポイントが含まれると言います。

  プロセス 物理 Win32 プロセス。 プロセスには複数のプログラムを含めることができます。 *「プログラム*」も参照してください。

  プログラム特定のランタイム アーキテクチャ内で実行される単一の名前空間。 「*処理*」も参照してください。

  セッション デバッグ マネージャー (SDM) 任意の数のデバッグ エンジンを管理する任意の数のコンピューター上の複数のプロセス内のプログラムをデバッグします。 基本レベルでは、SDM はデバッグ エンジンのマルチプレクサです。 さらに、SDM は、デバッグ セッションの統一されたビューを IDE に提供します。

  スタック フレーム特定のフレームと入れ子になった関数呼び出しの特定のレベルでの計算の状態を表します。

  スレッド少なくとも 1 つのプログラムで実行されるスタック ベースの命令実行の一般化された概念。

  警告ブレークポイント 保留中のブレークポイントで警告を記述するための抽象化。 警告ブレークポイントは、保留中のブレークポイントがまだコードの場所にバインドされていない理由を説明します。 これは、保留中のブレークポイントによって記述された場所、または他の理由でコードがまだ読み込まれていない可能性があります。
