# <a name="evaluation-tools-for-visual-studio"></a>Visual Studio の評価ツール
## <a name="craftsmanship-checklist-for-visual-studio"></a>ビジュアルスタジオの職人技チェックリスト
 このチェックリストを使用して、視覚的および対話の詳細に関するユーザー エクスペリエンスの品質を評価します。

### <a name="overview"></a>概要

- すべてのコマンドが、コマンドが実行されたことをユーザーに伝えるフィードバックを得ることを確認します。

- すべての UI 要素とコントロールが、すべてのテーマとハイ コントラスト モードで表示されていることを確認します。

- 非アクティブな選択とアクティブな選択が、標準モードとハイコントラスト モードの両方で常に区別されていることを確認します。

- フォーカスが常に表示され、見える状態であることを確認します。

### <a name="performance"></a>パフォーマンス

- コマンドの完了に 1 秒以上かかる場合は、何らかの 「ビジー」標識が表示されていることを確認します。

- コマンドの完了に 10 秒以上かかる場合は、明示的な進行状況バー (終了 (優先) または不確定のいずれかが表示されることを確認します。

### <a name="ui-text"></a>UI テキスト

- すべてのラベルが文またはタイトルの大文字と小文字で、テキストが完全に小文字でなくなっていることを確認します。

    ||正しい|誤|
    |-|-------------|---------------|
    |**コマンド テキスト (すべて)**|センテンスケース:<br /><br /> **ディレクトリ名:**|ディレクトリ名:|
    |**ボタン テキスト (クライアント)**|タイトルケース:<br /><br /> **[ デフォルトとして設定 ]**|既定値として設定|
    |**ボタンテキスト(オンライン)**|センテンスケース:<br /><br /> **[ デフォルトとして設定 ]**||

- グループ ヘッダーとボタンを除くすべてのラベルがコロンで終わり、ペアになっているコントロールの前に表示されることを確認します。

- ユーザー入力エンドを省略記号 **[.]** でキャプチャする UI を起動するボタン、コマンド、およびコマンド リンクを確認します。

  例 :

  - ダイアログの **[詳細設定..]** ボタン。

  - ダイアログ自体を起動するのはコマンドの意図であるため、[ツール] メニューの **[ツール > オプション**] の下のコマンド オプションは省略記号を取得できません。

- 業界標準の用語を除き、UI に省略形が含まれていることを確認します。 たとえば、HTML も TCP/IP も、OOM (メモリ不足) と PII (個人を特定できる情報) にする必要がありますが、スペルアウトする必要はありません。

### <a name="keyboard-access"></a>キーボード アクセス

- キーボードを使用して各タスクを実行する方法があることを確認します。 通常、これは各コントロールのキーボード アクセスを通じて行われますが、一部の高度な視覚領域では、コード ビューへの移動などの回避策を使用できます。

- コントロールを論理的な順序 (左から右、上から下) でタブで表示できることを確認します。 これはほとんどのコントロールのベスト プラクティスですが、すべてのコントロールがこの方法を必要としているわけではありません。 たとえば、ラジオ ボタン コントロールがタブ ストップが 1 つのグループに含まれているかどうかを確認します。

- すべてのコントロールにラベルがあること、および各ラベルにニーモニックが付いてきたことを確認します (例外には、タブ内のラベル付きコントロールに続くラベルのないコントロールが含まれます)。

- ニーモニック競合がないことを確認します。

### <a name="fonts"></a>フォント

- すべてのフォント(面、サイズ、色)が一貫して使用されていることを確認し、階層を維持します。

- すべての UI 要素が環境フォント サービスを使用していることを確認します。 (Visual [Studio のフォントと書式設定を](../../extensibility/ux-guidelines/fonts-and-formatting-for-visual-studio.md)参照してください)

     サービスが使用されているかどうかを確認するには、[**ツール] > [フォントと色] >オプション**を参照してください。 [設定] ドロップダウンで [環境フォント] を選択し、スタイル上で異なるフォントフェイス (ハリントンやコミックサンズなど) に変更し、サイズを 12 pt に設定します。 次に、[OK] をクリックします。 IDE の再起動が必要な場合がありますが、ほとんどの UI はすぐに変更されます。 再起動してもフォントの変更を受け取らない領域では、環境フォントが使用されていません。

- 環境フォント サイズが変更された場合、サービスの派生フォント (太字や拡大テキストなど) が、"標準" テキストに関連してフォントのサイズと書式が保持されていることを確認します。

- フォントの拡大によるクリッピングのバグがないことを確認します。 クリップされるフォントは、固定高さコントロールまたは固定高さのコンテナーの結果である可能性があります。

### <a name="dialogs"></a>ダイアログ

- ダイアログ タイトルが、ダイアログ を起動したコマンドと同じであることを確認します。

- すべての標準コントロールがオペレーティング システムと一致していることを確認します: 背景色は標準であり、どのコントロールも標準コントロールとは異なる表示を行う特別な再テンプレート スタイルを持つ必要があります。

- フォーム内の余白が 12 ピクセルで、均一で一貫性のあるように見えるようにします。

- ダイアログが IDE シェルまたはダイアログを生成したウィンドウの中央に表示されることを確認します。

- 便利な場合は、ダイアログを変更可能にする必要があります。 サイズ変更可能なダイアログでは、サイズ変更時に、ダイアログの他の部分が一定のまま、適切なコントロールのサイズを変更する必要があることを確認します。

- サイズ変更可能なダイアログボックスが、ユーザーが調整したサイズ (サイズ、場所、ダイアログコントロールの展開など) が保持されていることを確認します。

- タイトル バーにアイコンがないことを確認します。

- タイトル バーに最小化ボタンと最大化ボタンがないことを確認します。

#### <a name="dialog-operation-buttons-vs-client-only"></a>ダイアログ操作ボタン(VSクライアントのみ)

- 操作ボタンが次の順序であることを確認します: **[OK]、[****キャンセル**]、[適用] の順**にクリック**します。

- **[OK] ボタン**と **[キャンセル]** ボタンが標準サイズである 75 x 23 ピクセルであることを確認します。

- **[OK] ボタン**と **[キャンセル]** ボタンのサイズが文字列の長さに関係なく同じであることを確認します。

- 操作ボタンのラベルが標準より広いボタンを必要とする場合は、対応する**キャンセル**ボタンのサイズが同じであることを確認します。

- ボタンと関連するコントロールの間に 6 ピクセルのパディングがあることを確認します。

- **[OK] ボタン**と **[キャンセル]** ボタンにニーモニック (下線付きの文字で定義されたアクセス キー) が表示されないことを確認します。

- 既定では、1 つのボタン (通常**は [OK]**) にフォーカスがあることを確認します。

- Esc**キー**がダイアログをキャンセルすることを確認します。

- Enter を処理するコントロールにフォーカスがない場合 **、Enter**が既定のボタンを実行することを確認します。

- **[OK] ボタン**と **[キャンセル]** ボタンがダイアログの右下隅に配置されていることを確認します。 まれに、右上に縦に積み重ねることができる例外もあります。

- 他のボタンがダイアログ内の水平配置にある場合にのみ、垂直構成が使用されることを確認します。

### <a name="control-standards"></a>管理基準

#### <a name="general"></a>全般

- 可能であれば、ユーザーの操作を高速化し、安全な結果や共通の結果にユーザーを誘導するための、有効なデフォルト値があることを確認します。

- 以前のエクスペリエンスに基づいて何が起こるかをユーザーが知ることができるように、標準コントロールが同じように動作することを確認します。

#### <a name="label-controls"></a>ラベル コントロール

- 各コントロールにラベルがあり、各ラベルがコントロールと視覚的にペアになっていること (通常は 4 ~ 6 ピクセルの範囲内) と、対応するコントロールの近くにあることを確認します。

- ラベルが左に配置されている場合は、ラベルのベースラインが入力テキストのベースラインに合うように、ラベルが左の端に配置され、水平方向に配置されていることを確認します。

- 複数の重ねラベルと入力コントロールがコントロールの左側に配置されている場合、ラベルが左にフラッシュされ、ダイアログの端から等しい距離にフラッシュされ、コントロールから右と同じ距離がフラッシュされないことを確認します。 ペアは、グループ化を示すために追加の間隔が必要な場合を除き、均等に分散する必要があります。

#### <a name="input-controls-text-boxes-and-combo-boxes"></a>入力コントロール (テキスト ボックスおよびコンボ ボックス)

- 既定の環境フォントを使用する場合、テキスト ボックス、コンボ ボックス、およびボタンの表示高さがすべて 23 ピクセルであることを確認します。

- ヒント テキストを使用する場合は、色がカラー`Environment.ControlEditHintText`サービスを使用するように設定されていることを確認します。

- フィールドが必須フィールドで、そのフィールドとして識別する必要がある場合は、以下を確認します。

  - 背景がに`Environment.ControlEditRequiredBackground`設定され、前景が`Environment.ControlEditRequiredHintText`

  - **"必須>"\<** として表示されるヒント テキストがコントロール内に存在することを示します。

#### <a name="button-controls"></a>ボタン コントロール

- 長いテキストを収容しない限り、ボタンの最小サイズが 75 x 23 ピクセルであることを確認します。

- ボタンの左右の余白が 3 ~ 5 ピクセルで、コンテンツの余白が埋め込みであることを確認します。

- **[Browse..]** ボタン (または同様の機能) の代わりに省略記号 **[..]** のみを含む小さな正方形のボタンを使用してもかまいません。 使用する場合は、ボタンのサイズが 23x23 であることを確認します。

- ダイアログボックスに複数の **[Browse...]** ボタンがある場合は、短縮版 (省略記号のみ **[..])** がすべての人に使用されていることを確認します。

- 省略記号 **[..]** ボタンにニーモニックが含まれていないことを確認します。 フォーカスが横の入力コントロールにある場合、1 つのタブで省略記号ボタンにフォーカスを移動します。

- より多くのユーザー入力をキャプチャするセカンダリ UI を起動するボタン、コマンド、およびコマンド リンクが省略記号 **[.. ]** で終了する必要があることを確認します。

#### <a name="hyperlinks"></a>ハイパーリンク

- ハイパーリンク コントロールがアクティブなときに赤く点滅することを確認します。 これは、カラーサービスが使用されていないという指標です

- 使用される VS カラーが次の点であることを確認します。

  - `Environment.ControlLinkText`

  - `Environment.ControlLinkTextHover`

  - `Environment.ControlLinkTextPressed`

- 段落に埋め込まれていない場合は、ハイパーリンクが青で下線が付かっていることを確認します。

#### <a name="check-boxes"></a>チェック ボックス

- チェック ボックスに複数行のテキストが含まれている場合は、ボックスがテキストの最初の行に揃え、すべての行にわたって垂直方向に中央揃えで配置されないことを確認します。

- チェック ボックスが常にバイナリ選択を示し、ユーザーを移動したり、新しいウィンドウやページを開いたりしないことを確認します。

- チェック ボックスに入力コントロールに関連するオプションが表示される場合は、そのコントロールの左に位置し、そのコントロールの下に非常に近い位置に配置されていることを確認して、その関係を示します。

- ダイアログまたはページの内容全体を有効にする手段として、チェック ボックスが使用**されていない**かどうかを確認します。

#### <a name="group-boxes"></a>グループボックス

- ダイアログボックス内に、ダイアログの内容全体を含むグループボックスが 1 つも含まれていないことを確認します。

- 各グループ ボックスに少なくとも 2 つのコントロールがあることを確認します。

- ダイアログにグループ ボックスが 3 つ以上ある場合は、まれに問題ありません。

- ネストされたグループ ボックスがないことを確認します。

### <a name="icons"></a>[小さいアイコン]

- 暗いテーマでアイコンが正しく反転表示されることを確認します。

- すべてのアイコンがコア概念に基づいていることを確認します。

- 各アイコンが明確で、認識しやすく、(ステータス修飾子/言語なし)2つ以上の概念が含まれていないことを確認します。

- ベース アイコンがスペースの中央に表示されることを確認します。

- ハイコントラスト モードでは、すべてのアイコンが読みやすいように見えるようにします。

- 使用する色が色の使用基準と一致していることを確認します。

- アイコンの周囲にハロー(罫線)がないことを確認します。 (存在する場合、ハローは隣接する UI の背景色と一致する必要があります)。

### <a name="touch-enabled-ui"></a>タッチ対応 UI

- インタラクティブコントロールがタッチ可能なほどの大きさであることを確認します ( 最小**サイズは 23 x 23 ピクセル**)

- 最も頻繁に使用されるコントロールのサイズが**40 x 40 ピクセル**以上であることを確認します。

- インタラクティブコントロールの間隔が**5 ピクセル**以上あることを確認する
