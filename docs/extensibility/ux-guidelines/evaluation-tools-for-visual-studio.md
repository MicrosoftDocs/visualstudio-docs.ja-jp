# Visual Studio の評価ツール
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

## Visual Studio の職人気質のチェックリスト  
 このチェックリストを使用すると、表示と対話の詳細についてのユーザー エクスペリエンスの品質を評価できます。  
  
### 概要  
  
-   すべてのコマンドは、それらのコマンドが行われていることをユーザーに指示するフィードバックに生じることを確認します。  
  
-   すべての UI 要素とコントロールがすべてのテーマとハイ コントラスト モードで表示されていることを確認します。  
  
-   非アクティブとアクティブな選択が常に、差別化標準とハイ コントラスト モードの両方を確認します。  
  
-   フォーカスが常に表示され、明確なことを確認します。  
  
### パフォーマンス  
  
-   完了する 1 秒以上かかるかどうか、コマンドにはいくつかの種類「ビジー」インジケーターが表示されていることを確認します。  
  
-   コマンドが完了するには、明示的な進行状況バーか、10 秒以上を受け取る場合は、ことを確認不定になります \(推奨\)、未確定のいずれかが表示されます。  
  
### UI テキスト  
  
-   すべてのラベルが文またはタイトル ケースをテキストが小文字まったくないことを確認します。  
  
    ||修正|正しくないです。|  
    |-|--------|--------------|  
    |**コマンド テキスト \(すべて\)**|文のケース:<br /><br /> **ディレクトリ名:**|ディレクトリ名:|  
    |**ボタンのテキスト \(クライアント\)**|タイトル ケース:<br /><br /> **\[既定値として設定する\]**|セット AS DEFAULT|  
    |**ボタンのテキスト \(オンライン\)**|文のケース:<br /><br /> **\[既定値として設定\]**||  
  
-   グループ ヘッダーやボタン以外のすべてのラベルが最後にコロンと、ペアとなるコントロールの前に確認します。  
  
-   ボタン、コマンド、およびユーザー入力をキャプチャするための UI を起動するコマンドのリンクが、省略記号で終わることを確認 **\[...\]**します。  
  
     次に例を示します。  
  
    -   **\[詳細\]** ダイアログ ボックス上のボタンをクリックします。  
  
    -   \[ツール\] メニューのコマンド オプション \(**ツール \> オプション**\) コマンドの目的は、ダイアログ ボックス自体を起動するため、省略記号を取得できません必要があります。  
  
-   UI に業界標準の条項を除くの省略形が含まれていないことを確認します。 たとえば、HTML も TCP\/IP しなければなりません明記、OOM \(メモリ不足\) と PII \(個人を特定できる情報\) にする必要があります。  
  
### キーボード アクセス  
  
-   キーボードを使用して各タスクを実行する方法があることを確認します。 通常、各コントロールに対してキーボード アクセスを介して行われますが、視覚的分野によっては、コード ビューに移動などの回避策はもかまいません。  
  
-   論理的な順序 \(左から右、上から下\) でのコントロールを tab キーできることを確認します。 これは、ほとんどのコントロールの最善の方法がないすべてのコントロールはこのアプローチが必要です。 たとえば、そのオプション ボタンの制御は、1 つのタブ ストップを使用してグループを確認します。  
  
-   すべてのコントロールにラベルがあることと、各ラベルにニーモニックがあることを確認 \(サポートされない例をいくつかのラベルのないコントロール\] タブで、ラベル付きコントロールに続く場合があります\)。  
  
-   ニーモニック競合がないことを確認します。  
  
### フォント  
  
-   \(フェイス、サイズ、色\) のすべてのフォントが一貫して使用して、階層の管理を確認します。  
  
-   すべての UI 要素が、環境フォントのサービスを使用することを確認します。 \(参照 [フォントおよび Visual Studio の書式設定](../../extensibility/ux-guidelines/fonts-and-formatting-for-visual-studio.md)\)  
  
     サービスを使用しているかを確認するには **ツール \> オプション \> フォントおよび色**します。 設定のドロップダウン リストには、\[環境フォントを選択する \(可読性や漫画 San\) などの別 stylistically フォント フェイスを変更し、12 pt に、サイズを設定します。 \[Ok\] をクリックします。 IDE を再起動する必要がありますが、ほとんどの UI はすぐに変更します。 再起動時にものフォントの変更は取らない領域は、環境フォントを使用しません。  
  
-   サービス \(たとえば、太字や拡大したテキスト\) から派生したものですが、サイズ、および環境フォントのサイズが変更されたときに、「通常」のテキストに合わせて書式設定を維持することを確認します。  
  
-   拡大したフォントのための領域のバグがないことを確認します。 固定フォントは、コントロールの高さを固定または固定の高さのコンテナーの結果で、可能性があります。  
  
### ダイアログ ボックス  
  
-   ダイアログのタイトルがそれを起動したコマンドと同じであることを確認します。  
  
-   すべての標準コントロールにオペレーティング システムと整合性があることを確認します。 標準の背景色はされ、コントロールには特別な re テンプレート スタイルを標準コントロールから別のように見えることはありません。  
  
-   フォーム内の余白は 12 ピクセルにする必要があり、一貫性のある表示を確認します。  
  
-   ダイアログ ボックスが IDE シェルまたはそれらから派生した子ウィンドウ内で中央揃えで表示されることを確認します。  
  
-   ときに便利ですが、ダイアログ ボックスがサイズ変更可能な場合があります。 サイズはダイアログの場合は、サイズ変更時に適切なコントロールする必要がありますのサイズ変更される一定に保たれるダイアログ ボックスの他の部分間を確認します。  
  
-   サイズ変更可能なダイアログ ボックスがどのユーザー調整サイズ \(サイズ、場所、ダイアログ コントロールの拡張\) を保持することを確認します。  
  
-   タイトル バーにアイコンがないことを確認します。  
  
-   タイトル バーに最小化ボタンがないことを確認します。  
  
#### ダイアログの操作ボタン \(VS クライアントのみ\)  
  
-   次の順序で操作ボタンがあることを確認: **OK**, 、**キャンセル**, 、**適用**します。  
  
-   いることを確認 **OK** と **キャンセル** ボタンは、標準のサイズ: 75 x 23 ピクセルです。  
  
-   いることを確認 **OK** と **キャンセル** ボタンは、文字列の長さに関係なく同じサイズのです。  
  
-   操作ボタンのラベルには、標準よりも大きくなるボタンが必要とする場合、対応する **キャンセル** ボタンは、等しいサイズのです。  
  
-   ボタンと関連付けられているコントロール間の 6 ピクセル余白があることを確認します。  
  
-   いることを確認、 **OK** と **キャンセル** ボタンには、ニーモニック \(アクセス キーの下線付きの文字で定義されている\) はありません。  
  
-   その 1 つのボタンのことを確認 \(通常 **OK**\) 既定でフォーカスがあります。  
  
-   いることを確認 **Esc** ダイアログ ボックスをキャンセル  
  
-   いることを確認 **Enter** 入力を処理するコントロールにフォーカスがない場合は、既定のボタンを実行します。  
  
-   いることを確認、 **\[ok\]** と **キャンセル** ダイアログ ボックスの右下隅の下にあるボタンを配置します。 まれなケースを除き、右上の垂直方向に重ねるにでも許容されます。  
  
-   垂直方向の構成は、その他のボタンがダイアログ内で水平方向の配置の場合にのみ使用を確認します。  
  
### 制御標準  
  
#### 全般  
  
-   可能であれば、適切な既定値はユーザーとの対話と結果が安全なまたは一般的な方向にユーザーに指示を高速化することを確認します。  
  
-   ユーザーは、以前の経験に基づくどうなるかを知ることには標準のコントロールが同じように動作することを確認します。  
  
#### ラベル コントロール  
  
-   各コントロールにラベルがあることと、各ラベルは、コントロール \(通常は 4 ~ 6 ピクセルの範囲\) 以内と組み合わされて、視覚的にしは他のコントロールにより、対応するコントロールに近いことを確認します。  
  
-   ラベルの配置はフラッシュすることを確認コントロールを使用して左上に配置されている場合は、左端し、左に配置されている場合、ラベルの基準が入力テキストのベースラインに調整できるように、水平方向に中央に配置します。  
  
-   複数の積み上げラベルと入力コントロールがコントロールの左側に配置されている場合は、ラベルが左揃えと、ダイアログ ボックスの端から等しい距離、フラッシュしない右と、コントロールから等しい距離を確認します。 ペアは、グループ化を示すために追加スペースを必要な場合を除き、均等に配分する必要があります。  
  
#### 入力コントロール \(テキスト ボックスやコンボ ボックス\)  
  
-   既定環境フォントを使用する場合は、テキスト ボックス、コンボ ボックスとボタンの表示の高さすべて 23 ピクセルとはことを確認します。  
  
-   ヒント テキストを使用する場合は、色に設定されていることを確認 `Environment.ControlEditHintText` 色サービスを使用します。  
  
-   フィールドがよう特定する必要がある必須フィールドの場合は、次のように確認します。  
  
    -   バック グラウンドに設定されている `Environment.ControlEditRequiredBackground` に設定されているフォア グラウンドと `Environment.ControlEditRequiredHintText`  
  
    -   として表示されるコントロール内のヒント テキストが **「\< 必要 \>」**  
  
#### ボタン コントロール  
  
-   長いテキストに対応しない限りは、\[ボタンの 75 x 23 \(ピクセル単位\) の最小サイズであることを確認します。  
  
-   ボタンが 3 ~ 5 \(ピクセル単位\) だけでなく、コンテンツの埋め込みの余白を右残されたことを確認します。  
  
-   省略記号のみを含む小さな正方形のボタンを使用してもかまいません **\[...\]** 上の代わりに、 **\[参照...\]** ボタン \(または同様の機能\)。 使用する場合は、ボタンが 23 x 23 のサイズを確認します。  
  
-   1 つ以上を使用する必要がある場合 **\[参照...\]** いることを確認して、ダイアログに短縮したバージョン \(省略記号のみ **\[...\]**\) のすべてに使用します。  
  
-   確認を省略記号 **\[...\]** ボタンには、ニーモニックはありません。 その横にある入力コントロールにフォーカスがある場合、1 つのタブは、省略記号ボタンにフォーカスを移動する必要があります。  
  
-   確認ボタン、コマンド、および複数のユーザー入力をキャプチャするセカンダリの UI を起動するコマンドのリンクは、省略記号で終わる必要があります **\[...\]**します。  
  
#### ハイパーリンク  
  
-   ハイパーリンク コントロールがアクティブになったときに赤を点滅しないことを確認します。 これは、色のサービスが使用していないが、インジケーターです。  
  
-   使用されている VS の色があることを確認します。  
  
    -   `Environment.ControlLinkText`  
  
    -   `Environment.ControlLinkTextHover`  
  
    -   `Environment.ControlLinkTextPressed`  
  
-   ハイパーリンクが段落に埋め込まれている場合を除きに下線を青に表示されることを確認します。  
  
#### チェック ボックス  
  
-   チェック ボックスに複数行のテキストがある場合は、ボックスによりすべての行にわたっていない垂直方向に中央揃えのテキストの最初の行を揃えることを確認します。  
  
-   確認のチェック ボックス常にバイナリの選択肢を示すとはしないユーザーを移動または新しいウィンドウまたはページを開きます。  
  
-   チェック ボックスが入力コントロールに関連するオプションを表している場合は、配置されているフラッシュ左と非常に閉じるの関係を示すためにそのコントロールの下を確認します。  
  
-   チェック ボックスがあることを確認 **決して** ダイアログまたはページの内容全体を有効にするための手段として使用します。  
  
#### グループ ボックス  
  
-   ダイアログ ボックスが、ダイアログ ボックスの内容全体を含む 1 つのグループ ボックス内に含まれていないことを確認します。  
  
-   各グループ ボックス内の少なくとも 2 つのコントロールがあることを確認します。  
  
-   3 つ以上のグループ ボックス、ダイアログ ボックスであることはほとんどありませんなります。  
  
-   入れ子になったグループ ボックスがないことを確認します。  
  
### アイコン  
  
-   ダーク テーマで、アイコンが正しく反転表示することを確認します。  
  
-   すべてのアイコンを中心的な概念に基づいていることを確認します。  
  
-   各アイコンが個別の容易に識別および、\(なし状態修飾子\/言語\) の 3 つ以上の概念が含まれていないことを確認します。  
  
-   基本のアイコンが、領域内の中央に表示されることを確認します。  
  
-   すべてのアイコンがハイ コントラスト モードで読みやすい表示されることを確認します。  
  
-   使用する色により使用状況の標準の色を揃えることを確認します。  
  
-   アイコンの周りには、ハロー \(境界線\) がないことを確認します。 \(存在する場合は、ハローは隣接する UI の背景色一致する必要があります。\)  
  
### タッチ対応の UI  
  
-   インタラクティブな管理が簡単に押せる – 最小にするのに十分な大きさであることを確認 **23 x 23 ピクセル** サイズ  
  
-   最もよく使用されるコントロールが、少なくともであることを確認 **40 x 40 ピクセル** サイズ。  
  
-   インタラクティブ コントロールが、少なくともあることを確認 **の間隔が 5 ピクセル** それらの間
