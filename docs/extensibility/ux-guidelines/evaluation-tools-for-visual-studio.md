# <a name="evaluation-tools-for-visual-studio"></a>Visual Studio の評価ツール
## <a name="craftsmanship-checklist-for-visual-studio"></a>Visual Studio 用の機能チェックリスト
 このチェックリストを使用して、ユーザー エクスペリエンスの品質をビジュアルと対話の細部に関して評価します。

### <a name="overview"></a>概要

- すべてのコマンドに、コマンドが実行されたことをユーザーに知らせるフィードバックが備わっていることを確認します。

- すべての UI 要素とコントロールが、すべてのテーマで、またハイ コントラスト モードで表示されることを確認します。

- 標準とハイ コントラストの両モードで、アクティブでない選択とアクティブな選択が常に見分けられることを確認します。

- フォーカスが常にはっきり視認できることを確認します。

### <a name="performance"></a>パフォーマンス

- コマンドの完了に 1 秒より長くかかる場合に、何らかの "ビジー" インジケーターが表示されることを確認します。

- コマンドの完了に 10 秒より長くかかる場合に、確定的 (推奨) または非確定的な進行状況バーがはっきり表示されることを確認します。

### <a name="ui-text"></a>UI テキスト

- すべてのラベルで文または各単語の先頭文字が大文字であり、全部が小文字のテキストが存在しないことを確認します。

    ||正解|不正解|
    |-|-------------|---------------|
    |**コマンド テキスト (すべて)**|文の先頭文字を大文字にする:<br /><br /> **Directory name:**|Directory Name:|
    |**ボタン テキスト (クライアント)**|各単語の先頭文字を大文字にする:<br /><br /> **[ Set As Default ]**|SET AS DEFAULT|
    |**ボタン テキスト (オンライン)**|文の先頭文字を大文字にする:<br /><br /> **[ Set as default ]**||

- グループ ヘッダーとボタンを除くすべてのラベルがコロンで終わり、ペアになっているコントロールの前に配置されていることを確認します。

- UI を表示してユーザー入力を取得するボタン、コマンド、およびコマンド リンクで、ラベルの末尾に省略記号 **[...]** が付いていることを確認します。

  例 :

  - ダイアログの **[詳細設定...]** ボタン。

  - [ツール] メニュー下のコマンド オプション (例: **[ツール] > [オプション]** ) については、ダイアログを表示すること自体がコマンドの目的であるため、省略記号を付けないでください。

- 業界標準の用語を除いて、UI に省略形が含まれていないことを確認します。 たとえば、HTML や TCP/IP はそのまま使用できますが、OOM (out of memory: メモリ不足) や PII (personally identifiable information: 個人を特定できる情報) のような省略形は使用しないでください。

### <a name="keyboard-access"></a>キーボード アクセス

- キーボードを使用して各タスクを実行する方法があることを確認します。 通常、これは各コントロール用のキーボード アクセスによって実現されますが、視覚性の高い一部の領域については、コード ビューへの移動などの回避策が許容されます。

- Tab キーによるコントロール間の移動が論理的な順序 (左から右、上から下) で行えることを確認します。 これはほとんどのコントロールでベスト プラクティスですが、この方針はすべてのコントロールに必須ではありません。 たとえば、一連のラジオ ボタン コントロールが、1 つのタブ ストップがあるグループに含まれていることを確認します。

- すべてのコントロールにラベルがあること、また各ラベルにニーモニックがあることを確認します (例外としては、タブ内のラベル付きコントロールに続く一部のラベルなしコントロールなどがあります)。

- ニーモニックの競合がないことを確認します。

### <a name="fonts"></a>フォント

- すべてのフォント (書体、サイズ、色) が一貫して使用されており、階層を維持していることを確認します。

- すべての UI 要素で環境フォント サービスを使用していることを確認します。 (「[Visual Studio のフォントと書式設定](../../extensibility/ux-guidelines/fonts-and-formatting-for-visual-studio.md)」を参照)

     サービスが使用されているかどうかを確認するには、 **[ツール] > [オプション] > [フォントおよび色]** にアクセスします。 [設定] ドロップダウンで、[環境フォント] を選択し、スタイルが異なるフォント (Harrington や Comic Sans など) に変更し、サイズを 12 ポイントに設定します。 次に、[OK] をクリックします。 IDE の再起動が必要な場合がありますが、ほとんどの UI はすぐに変更されます。 再起動してもフォントの変更が反映されない領域は、環境フォントを使用していません。

- サービスから派生したフォント (太字や拡大テキストなど) について、環境のフォント サイズを変更しても、"通常" テキストと異なるサイズや書式設定が維持されることを確認します。

- 拡大フォントが原因のクリッピング バグがないことを確認します。 コントロールやコンテナーの高さが固定されていると、フォントのクリッピングが発生する可能性があります。

### <a name="dialogs"></a>ダイアログ

- ダイアログのタイトルが、そのダイアログを起動したコマンドと同じであることを確認します。

- すべての標準コントロールがオペレーティング システムと整合していることを確認する: 標準の背景色を使用します。また、どのコントロールも、再テンプレート化して特別なスタイルを適用し、標準コントロールと異なる外観にしてはなりません。

- フォーム内の余白が 12 ピクセルであり、均一で一貫した外観であることを確認します。

- ダイアログを生成した IDE シェルまたはウィンドウ内でダイアログが中央に表示されることを確認します。

- 利便性がある場合は、ダイアログのサイズを変更できるようにしてください。 サイズ変更可能なダイアログで、サイズ変更時に、ダイアログの他の部分は変わらないままで適切なコントロールのサイズが変更されることを確認します。

- サイズ変更可能なダイアログで、ユーザーが調整したサイズ (サイズ、位置、ダイアログ コントロールの展開など) が保持されることを確認します。

- タイトル バーにアイコンがないことを確認します。

- タイトル バーに最小化および最大化ボタンがないことを確認します。

#### <a name="dialog-operation-buttons-vs-client-only"></a>ダイアログ操作ボタン (VS クライアントのみ)

- 操作ボタンが次の順序になっていることを確認します: **[OK]** 、 **[キャンセル]** 、 **[適用]** 。

- **[OK]** および **[キャンセル]** ボタンが標準のサイズ (75 x 23 ピクセル) であることを確認します。

- **[OK]** および **[キャンセル]** ボタンが、文字列の長さに関係なく同じサイズであることを確認します。

- ラベルを操作ボタンに収めるためにボタンの幅を標準より広げる必要がある場合、対応する **[キャンセル]** ボタンのサイズが同じであることを確認します。

- ボタンとそれに関連したコントロールの間に 6 ピクセルのパディングがあることを確認します。

- **[OK]** および **[キャンセル]** ボタンにニーモニック (下線付きの文字で定義されたアクセス キー) がないことを確認します。

- 既定で 1 つのボタン (通常は **[OK]** ) にフォーカスがあることを確認します。

- **Esc** でダイアログがキャンセルされることを確認します

- Enter を処理するコントロールにフォーカスがない場合に、**Enter** によって既定のボタンが実行されることを確認します。

- **[OK]** および **[キャンセル]** ボタンの位置がダイアログの右下隅であることを確認します。 まれな例外として、右上で縦に並べることは許容されます。

- 縦構成の使用は、ダイアログ内で他のボタンが横に並んでいる場合に限られることを確認します。

### <a name="control-standards"></a>コントロールの標準

#### <a name="general"></a>全般

- ユーザー対話を高速化し、安全または一般的な結果にユーザーを導く適切な既定値が、可能であれば用意されていることを確認します。

- これから起きることが以前の経験に基づいてユーザーにわかるよう、標準コントロールが同じように動作することを確認します。

#### <a name="label-controls"></a>ラベル コントロール

- 各コントロールにラベルがあること、および、各ラベルがそのコントロールと視覚的にペアになっていて (通常、4 ～ 6 ピクセルの範囲内)、他のコントロールよりもそれに対応するコントロールに近いことを確認します。

- 上および水平方向の中央に配置する場合に、ラベルは左揃えで、コントロールは左端に配置されることを確認します。これにより、左に配置する場合に、ラベルのベースラインが入力テキストのベースラインと揃います。

- 積み重ねた複数のラベルと入力コントロールをコントロールの左側に配置する場合に、ラベルが左揃えで、ダイアログの端から等距離であることを確認します。右揃えにしたり、コントロールから等距離にしたりしないでください。 グループ化を示すために追加の間隔が必要な場合を除き、ペアの散らばり具合を均等にしてください。

#### <a name="input-controls-text-boxes-and-combo-boxes"></a>入力コントロール (テキスト ボックスとコンボ ボックス)

- 既定の環境フォントを使用する場合、テキスト ボックス、コンボ ボックス、およびボタンの表示高がすべて 23 ピクセルであることを確認します。

- ヒント テキストを使用する場合、カラー サービスを使用して色が `Environment.ControlEditHintText` に設定されていることを確認します。

- フィールドが必須フィールドであり、そのように識別される必要がある場合は、次のことを確認します。

  - 背景が `Environment.ControlEditRequiredBackground` に、前景が `Environment.ControlEditRequiredHintText` にそれぞれ設定されていること

  - **"\<Required>"** と表示されるヒント テキストがコントロール内にあること

#### <a name="button-controls"></a>ボタン コントロール

- テキストが長い場合を除き、ボタンが 75 x 23 ピクセルの最小サイズであることを確認します。

- ボタンの左右に 3 ～ 5 ピクセルの余白があり、コンテンツのパディングも同様であることを確認します。

- **[参照...]** ボタン (または同様の機能) の代わりに、省略記号 **[...]** のみが表示された小さい正方形のボタンを使用することは認められています。 使用する場合は、ボタンのサイズが 23 x 23 であることを確認します。

- 複数の **[参照...]** ボタンがダイアログにある場合は、すべてに対して短縮バージョン (省略記号 **[...]** のみ) が使用されていることを確認します。

- 省略記号 **[...]** ボタンにニーモニックがないことを確認します。 フォーカスがその横の入力コントロールにあるときに、1 つのタブでフォーカスを省略記号ボタンに移動する必要があります。

- さらにユーザー入力を受け付けるセカンダリ UI を起動するボタン、コマンド、およびコマンド リンクは省略記号 **[...]** で終わる必要があることを確認します。

#### <a name="hyperlinks"></a>ハイパーリンク

- アクティブなときにハイパーリンク コントロールが赤色で点滅しないことを確認します。 これは、カラー サービスが使用されていないことを示します。

- 使用されている VSColor が次のとおりであることを確認します。

  - `Environment.ControlLinkText`

  - `Environment.ControlLinkTextHover`

  - `Environment.ControlLinkTextPressed`

- 段落に埋め込まれていない場合に、ハイパーリンクが下線なしの青色で表示されることを確認します。

#### <a name="check-boxes"></a>チェック ボックス

- チェック ボックスのテキストが複数行になる場合、ボックスの縦方向の位置はすべての行の中央に寄せるのではなく 1 行目に合わせるようにします。

- チェック ボックスでは常に 2 択の選択肢を提示し、新しいウィンドウやページにユーザーを誘導したり、それらを開いたりしないようにします。

- 入力コントロールに関連したオプションをチェック ボックスで提示する場合、両者間の関係を示すために、コントロールのすぐ下に左揃えでチェック ボックスを配置します。

- ダイアログまたはページの内容全体を有効にする手段としてチェック ボックスを使用 **していない** ことを確認します。

#### <a name="group-boxes"></a>グループ ボックス

- ダイアログの内容全体を含む 1 つのグループ ボックスがダイアログ内に含まれていないことを確認します。

- 各グループ ボックス内に少なくとも 2 つのコントロールがあることを確認します。

- ほとんどの場合、3 つ以上のグループ ボックスがダイアログ上に存在することは望ましくありません。

- 入れ子になったグループ ボックスが存在しないことを確認します。

### <a name="icons"></a>アイコン

- ダーク テーマの使用時にアイコンが正しく反転表示されることを確認します。

- すべてのアイコンがコア コンセプトに基づいていることを確認します。

- それぞれのアイコンは、他との区別がはっきりしていて、認識しやすく、3 つ以上の概念が含まれていない (状態の修飾子や文言を含まない) ことを確認します。

- 基本アイコンが領域内の中央に表示されることを確認します。

- すべてのアイコンがハイ コントラスト モードで読みやすく表示されていることを確認します。

- 使用されているすべての色が、色の使用に関する標準に適合していることを確認します。

- アイコンのまわりにハロー (境界線) がないことを確認します (存在する場合、ハローは隣接する UI の背景色と一致している必要があります)。

### <a name="touch-enabled-ui"></a>タッチ対応 UI

- 対話型コントロールが十分に大きく、タッチしやすい (**23 x 23 ピクセル** 以上のサイズである) ことを確認します。

- 最もよく使われるコントロールのサイズが **40 x 40 ピクセル** 以上であることを確認します。

- 対話型コントロールは互いに **間隔が 5 ピクセル** 以上空いていることを確認します。
