# <a name="evaluation-tools-for-visual-studio"></a>Visual Studio の評価ツール
## <a name="craftsmanship-checklist-for-visual-studio"></a>Visual Studio の職人気質チェックリスト
 このチェックリストを使用して、ビジュアルおよび対話の詳細のユーザーエクスペリエンス品質を評価します。

### <a name="overview"></a>概要

- すべてのコマンドが、コマンドが実行されたことをユーザーに通知するフィードバックになることを確認します。

- すべてのテーマおよびハイコントラストモードですべての UI 要素とコントロールが表示されていることを確認します。

- 標準モードとハイコントラストモードの両方で、非アクティブな選択とアクティブな選択が常に区別されることを確認します。

- フォーカスが常に表示され、明らかになっていることを確認します。

### <a name="performance"></a>[パフォーマンス]

- コマンドの完了に1秒以上かかる場合は、"ビジー" インジケーターがいくつか表示されていることを確認します。

- コマンドが完了するまでに10秒以上かかる場合は、明示的な進行状況バー ([優先] または [不確定]) が表示されることを確認します。

### <a name="ui-text"></a>UI テキスト

- すべてのラベルが文またはタイトルのケースであり、どのテキストも完全に小文字でないことを確認します。

    ||正解|不正解|
    |-|-------------|---------------|
    |**コマンドテキスト (すべて)**|文の場合:<br /><br /> **ディレクトリ名:**|ディレクトリ名:|
    |**ボタンテキスト (クライアント)**|タイトルケース:<br /><br /> **[既定値として設定]**|既定値として設定|
    |**ボタンテキスト (オンライン)**|文の場合:<br /><br /> **[既定値として設定]**||

- グループヘッダーとボタンを除くすべてのラベルがコロンで終わり、ペアになっているコントロールの前に配置されていることを確認します。

- ユーザー入力をキャプチャするための UI を起動するボタン、コマンド、およびコマンドリンクが、省略記号 **[...]** で検証されていることを確認します。

  例 :

  - ダイアログの **[Advanced...]** ボタン。

  - ダイアログ自体を起動することが目的であるため、[ツール] メニュー ([**ツール > オプション**)] の下にあるコマンドオプションは省略記号を表示しません。

- 業界標準の用語を除き、UI に省略形が含まれていないことを確認します。 たとえば、OOM (メモリ不足) と PII (個人を特定できる情報) では、HTML と TCP/IP のどちらもスペルを指定する必要はありません。

### <a name="keyboard-access"></a>キーボード アクセス

- キーボードを使用して各タスクを実行する方法があることを確認します。 一般に、これは各コントロールのキーボードアクセスによって行われますが、一部の高度な視覚面では、コードビューへの移動などの回避策が許容されます。

- コントロールのタブを論理的な順序 (左から右、上から下) に設定できることを確認します。 ほとんどのコントロールではこれがベストプラクティスですが、すべてのコントロールでこのアプローチが必要になるわけではありません。 たとえば、ラジオボタンコントロールが、1つのタブストップを持つグループに含まれていることを確認します。

- すべてのコントロールにラベルがあり、各ラベルにニーモニックがあることを確認します (例外には、タブのラベル付きコントロールに続く、ラベル付けされていないコントロールがあります)。

- ニーモニックの競合がないことを確認します。

### <a name="fonts"></a>フォント

- すべてのフォント (顔、サイズ、色) が一貫して使用され、階層を維持していることを確認します。

- すべての UI 要素で環境フォントサービスが使用されていることを確認します。 ( [Visual Studio のフォントと書式設定を](../../extensibility/ux-guidelines/fonts-and-formatting-for-visual-studio.md)参照してください)

     サービスが使用されているかどうかを確認するには、[ **ツール] > [オプション] > [フォントおよび色] を選択** します。 [設定] ドロップダウンで、[環境フォント] を選択し、フォントフェイスを様式上誤用別 (Harrington や漫画など) に変更し、サイズを 12 pt に設定します。 次に、[OK] をクリックします。 IDE の再起動が必要になる場合がありますが、ほとんどの UI は直ちに変更されます。 再起動時にもフォントの変更を取得しない領域では、環境フォントが使用されません。

- サービスの派生フォント (たとえば、太字や拡大されたテキスト) が、環境のフォントサイズを変更したときに、"通常の" テキストに関連してサイズと書式設定を保持することを確認します。

- フォントの拡大に起因するクリッピングバグがないことを確認します。 クリッピングされるフォントは、高さが固定されたコントロールまたは高さが固定されたコンテナーの結果である可能性があります。

### <a name="dialogs"></a>ダイアログ

- ダイアログのタイトルが、それを起動したコマンドと同じであることを確認します。

- すべての標準コントロールがオペレーティングシステムと一致していることを確認する: 背景色は標準であり、コントロールには、標準コントロールとは異なる方法で表示される特別な再テンプレートスタイルを設定する必要があります。

- フォーム内の余白が12ピクセルで、均一で一貫している必要があることを確認します。

- ダイアログが IDE シェルまたはそれを生成したウィンドウの中央に表示されていることを確認します。

- 便利な場合は、ダイアログのサイズを変更する必要があります。 サイズ変更が可能なダイアログの場合は、サイズ変更時に、ダイアログの他の部分が変わらず、適切なコントロールのサイズを変更する必要があることを確認します。

- サイズ変更可能なダイアログで、ユーザーが調整したサイズ (サイズ、位置、ダイアログコントロールの展開など) が保持されていることを確認します。

- タイトルバーにアイコンが表示されていないことを確認します。

- タイトルバーに最小化ボタンと最大化ボタンが表示されていないことを確認します。

#### <a name="dialog-operation-buttons-vs-client-only"></a>ダイアログ操作ボタン (VS クライアントのみ)

- 操作ボタンが次の順序になっていることを確認します。 **[OK**]、 **[キャンセル**]、[ **適用**]。

- **[OK]** ボタンと **[キャンセル**] ボタンが標準サイズ (75 x 23 ピクセル) であることを確認します。

- 文字列の長さに関係なく、 **[OK]** ボタンと **[キャンセル** ] ボタンのサイズが同じであることを確認します。

- [操作] ボタンのラベルの幅が標準よりも大きいことが必要な場合は、対応する **[キャンセル** ] ボタンが同じサイズであることを確認します。

- ボタンと関連付けられたコントロールの間に6ピクセルの余白があることを確認します。

- **[OK]** ボタンと **[キャンセル**] ボタンにニーモニック (下線付きの文字で定義されているアクセスキー) がないことを確認します。

- 既定では、1つのボタン (通常は **OK**) にフォーカスがあることを確認します。

- **Esc** によってダイアログがキャンセルされることを確認します。

- Enter が、Enter を処理するコントロールにフォーカスがない場合は、既定のボタン **が実行さ** れることを確認します。

- ダイアログの右下隅に **[OK]** ボタンと **[キャンセル** ] ボタンが配置されていることを確認します。 まれに、右上で垂直方向に積み重ねることが許容されます。

- ダイアログ内で他のボタンが水平方向に配置されている場合にのみ、垂直方向の構成が使用されていることを確認します。

### <a name="control-standards"></a>制御標準

#### <a name="general"></a>全般

- 可能であれば、ユーザーの操作を高速化し、ユーザーを安全または一般的な結果に導くための適切な既定値があることを確認します。

- 標準コントロールが同じように動作することを確認して、ユーザーが以前のエクスペリエンスに基づいて何が起こるかを把握できるようにします。

#### <a name="label-controls"></a>ラベルコントロール

- 各コントロールにラベルがあり、各ラベルがコントロール (通常は4-6 ピクセル範囲内) と視覚的に組み合わされており、他のコントロールよりも対応するコントロールに近いことを確認します。

- ラベルが左に配置されている場合は、コントロールの左端に沿って左揃えに配置されていることを確認します。これにより、ラベルのベースラインが左側に配置されている場合は、ラベルのベースラインを入力テキストのベースラインに揃えます。

- 複数の積み上げラベルおよび入力コントロールがコントロールの左側に配置されている場合、ラベルは、ダイアログの端からの距離が均等になるようにします。また、コントロールと同じ距離にすることもできます。 グループ化を示すためにスペースを追加する必要がない場合は、ペアを均等に分散する必要があります。

#### <a name="input-controls-text-boxes-and-combo-boxes"></a>入力コントロール (テキストボックスとコンボボックス)

- 既定の環境フォントを使用する場合、テキストボックス、コンボボックス、およびボタンの表示の高さが23ピクセルになることを確認します。

- ヒントテキストが使用されている場合は、カラーサービスを使用して色がに設定されていることを確認し `Environment.ControlEditHintText` ます。

- フィールドが必須フィールドであり、そのように識別される必要がある場合は、次のことを確認します。

  - 背景がに設定され、 `Environment.ControlEditRequiredBackground` 前景がに設定されている。 `Environment.ControlEditRequiredHintText`

  - コントロール内に **" \<Required> "** と表示されるヒントテキストがあることを示します。

#### <a name="button-controls"></a>ボタン コントロール

- テキストが長い場合を除き、ボタンのサイズが 75 x 23 ピクセル以上であることを確認します。

- ボタンの余白が3-5 ピクセルに左右されること、およびコンテンツの埋め込みがあることを確認します。

- **[参照]** ボタン (または同様の機能) ではなく、省略記号 **[...]** だけを含む小さな正方形ボタンを使用できます。 使用する場合は、ボタンのサイズが23x23 であることを確認します。

- ダイアログボックスに **[Browse...]** ボタンが複数ある場合は、すべてに対して短縮バージョン (省略記号のみ **[...]**) が使用されていることを確認します。

- 省略記号 **[...]** ボタンにニーモニックがないことを確認します。 フォーカスが横の入力コントロールにある場合、1つのタブでフォーカスが省略記号ボタンに移動します。

- より多くのユーザー入力をキャプチャするセカンダリ UI を起動するボタン、コマンド、およびコマンドリンクが、省略記号 **[...]** で終了する必要があることを確認します。

#### <a name="hyperlinks"></a>ハイパーリンク

- ハイパーリンクコントロールがアクティブなときに赤色で点滅しないことを確認します。 これは、カラーサービスが使用されていないことを示します。

- 使用される VS の色が次のようになっていることを確認します。

  - `Environment.ControlLinkText`

  - `Environment.ControlLinkTextHover`

  - `Environment.ControlLinkTextPressed`

- 段落に埋め込まれていない限り、ハイパーリンクが下線なしで青色で表示されることを確認します。

#### <a name="check-boxes"></a>チェック ボックス

- チェックボックスに複数行のテキストが含まれている場合は、ボックスがテキストの最初の行に合わせて配置されていることを確認します。

- チェックボックスが常にバイナリの選択肢を示し、ユーザーが移動したり、新しいウィンドウやページを開いたりしないことを確認します。

- チェックボックスに入力コントロールに関連するオプションが表示される場合は、そのコントロールが左に配置され、そのコントロールの下に非常に近いことを確認して、その関係を示します。

- ダイアログまたはページの内容全体を有効にする手段として、チェックボックスが使用されてい **ない** ことを確認します。

#### <a name="group-boxes"></a>グループボックス

- ダイアログの内容全体を含む単一のグループボックスがダイアログに含まれていないことを確認します。

- 各グループボックス内に少なくとも2つのコントロールがあることを確認します。

- ダイアログに2つ以上のグループボックスがあることはほとんどありません。

- 入れ子になったグループボックスがないことを確認します。

### <a name="icons"></a>アイコン

- ダークテーマでアイコンが正しく反転して表示されることを確認します。

- すべてのアイコンが主要な概念に基づいていることを確認します。

- 各アイコンが区別され、認識しやすく、3つ以上の概念が含まれていないことを確認します (状態修飾子/言語が指定されていません)。

- 基本アイコンがスペースの中央に表示されていることを確認します。

- すべてのアイコンがハイコントラストモードで読みやすく表示されていることを確認します。

- 色の使用基準に合わせて使用されている色があることを確認します。

- アイコンの周りにハロー (枠線) がないことを確認します。 (存在する場合、ハローは隣接する UI の背景色と一致している必要があります)。

### <a name="touch-enabled-ui"></a>タッチ対応の UI

- 対話型コントロールのサイズが、touchable (最小 **23x23 ピクセル)** であることを確認します。

- 最も頻繁に使用されるコントロールのサイズが **4 0x40 ピクセル** 以上であることを確認します。

- 対話型コントロールの **間隔が5ピクセル** 以上であることを確認します。
