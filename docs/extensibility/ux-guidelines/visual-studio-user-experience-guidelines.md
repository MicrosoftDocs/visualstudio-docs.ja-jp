# <a name="visual-studio-user-experience-guidelines"></a>Visual Studio ユーザー エクスペリエンス ガイドライン
Visual Studio の新機能を設計する場合は、Visual Studio ユーザー エクスペリエンス のガイドラインに従います。 これらのガイドラインには、一般的なユーザー モデルと対話パターンに関する情報が含まれているので、新しいユーザー インターフェイス (UI) の設計者は、シームレスで一貫性のある開発環境でユーザー エクスペリエンスを作成できます。

マイクロソフト製品用のソフトウェアを開発する場合は、次のリソースの使用を学びます。

- [Windows ユーザー エクスペリエンスの相互作用ガイドライン](/windows/win32/uxguide/guidelines)は、Windows デスクトップの動作と外観の基本です。

- これらの Visual Studio ガイドラインは、プラットフォーム固有のガイドラインであり、環境に適した場所であれば、Windows の一般的なガイドラインに取って代わるものです。

- [UWP アプリのコントロールとパターン](/windows/uwp/design/controls-and-patterns)は、Visual Studio 内の一部のインスタンスで使用される可能性のある新しいパターンのリファレンスです。

このガイドラインでは、Visual Studio 用の設計時に何を行うかについて説明し、場合によっては、その方法に関する情報も含まれています。 Visual Studio 統合開発環境 (IDE) に統合されたソリューションのコードを記述できるようにする特定のアプリケーション プログラミング インターフェイス (API) について詳しくは、Visual Studio[ソフトウェア開発キット (SDK) を](../visual-studio-sdk.md)参照してください。
