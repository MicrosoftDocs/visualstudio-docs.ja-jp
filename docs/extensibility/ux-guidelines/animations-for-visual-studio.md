## アニメーションの基礎  
  
### Visual Studio でのアニメーションのベスト プラクティス  
 Visual Studio の IDE 全体で一貫性があり、わかりやすいアニメーションのスタイルを確実にこれらの規則に従います。  
  
-   **オプションを選択します。**特定の目的に使用するものにアニメーションを制限します。  
  
-   **タイミングや速度が重要な** を迅速かつ自然な遷移を感じることを確認します。  
  
    -   完了すると、1 の 0.5 秒 \(500 ミリ秒\) 内の遷移がアニメーション化されます。  
  
    -   頻繁に発生するアニメーションをすばやく、ユーザーのワークフローを中断しない必要があります。  
  
    -   アニメーションは高速か、理解しにくいがない非常に低速行われているいずれかの遷移を完了するはせっかちである少し目障りな感じにはなりません。  
  
    -   変数のタイミングを使用して、重要性を強調します。 たとえば、クラス図の項目のシーケンスを移動する、項目間の遷移を高速化し、重要な項目に注目する速度が低下します。  
  
-   **段階的な非線形イージングを使用して** を別の 1 つの状態から穏やかなかつ自然な躍動感を与える  
  
-   可能であれば、 **ホバー時の微妙なアニメーションを使用して** をマウスの下の対話型の要素を示します。  
  
-   依存する場合頻度の高いアニメーションの機能で、 **をオフにするための手段を提供** でローカル \(すべての機能\) でのオプションとして、 **ツール \> オプション** ダイアログ。  
  
-   **、一度に 1 つだけのアニメーションを実行する** や、1 つだけの情報を提供します。  
  
-   **注意が重要です。**ほとんどの場合、その目的を処理するために必要に応じてユーザーの注意をアニメーションがありません。 タイミング、順序、および動作の細かい変更が加えは知覚に大きく影響する可能性があり、効果的かつ非効率的なアニメーションの違いを行うことができます。  
  
-   アニメーションを使用して何かに注目する **ユーザーを中断する価値があることを確認する**の思考です。  
  
-   **進行状況や状態を表示すると** アニメーションによって。  
  
    -   基になるプロセスを推進しない場合は、進行状況の移動を示すを停止します。  
  
    -   不確定であるプロセスから不確定なプロセスを識別します。  
  
    -   アニメーションに特定の入力候補およびエラー状態があることを確認します。  
  
    -   実際の使用の追加情報を提供することで実際の値であるかどうかを確認して \[ステータスの表示効果アニメーションの使用を最小限に抑えます。 例についてには、一時的な状態の変更と緊急事態が含まれます。  
  
#### 行わないでください。  
  
-   小さな動き \(小さなフット プリントに移動\) を使用して、優先フェードインし、オブジェクトの移動の経過と共に変化します。  
  
-   実際の画面の広い領域に実行されるアニメーションを使用します。 サイズに関係なくこのスタイルのアニメーションは、ユーザーの邪魔になるです。  
  
-   ユーザーは、現在フォーカスされているオブジェクトまたはとの対話に関係なくアニメーションを使用します。  
  
-   点滅を繰り返すように点滅する通知に応答するユーザーを強制するなどの状態をリセットするユーザーとの対話を必要とするアニメーションを使用します。 任意の方法でとやり取りすると、それらを破棄するための十分なことがあります。  
  
 これらのベスト プラクティスのアプリケーションの詳細については、次を参照してください。 [アニメーション パターン](../../extensibility/ux-guidelines/animations-for-visual-studio.md#BKMK_AnimationPatterns)します。  
  
### アニメーションのメトリック  
  
-   システムは、10 ミリ秒未満のユーザーの操作に対応する視覚的にします。  
  
-   移行アニメーションは 500 ミリ秒を完了するよりも長いになりません。  
  
-   長い時間が必要な遷移を補正する 1 つの方法が 2 つの部分に分割し、します。たとえば、アニメーションの最初の部分でした \(最大 500 ミリ秒\) のコンテナーにコンテンツのフェードを続けて空コンテンツ コンテナー \(最大 500 ミリ秒\) となります。  
  
-   読み込み時間に計算する場合は、決定の進行状況インジケーター \(% 完了の進行状況インジケーター\) が適しています。  
  
-   計算できません読み込み時間の場合は、カーソルや埋め込みの回転アニメーション \(読み込みまたは操作のインジケーター\) などのビジー状態インジケーターは適切です。  
  
### Communicator とアニメーション  
 Visual Studio の UI では、アニメーションは、コミュニケーション ツールとしてのみ機能します。  さまざまな UI での構造の変更などの情報を通信に使用されます。たとえば、メニューが開きます終了したりした場合。 アニメーションは、インストールの進行状況の表示などの複雑なシステムの時間に依存する動作を視覚化または読者の注意のアラートと通知するために使用します。  
  
 UI アニメーションは、通常 4 つの方法で機能: 可視化、読者の注意、シミュレーション、および応答時間\/進捗状況。  
  
#### 視覚化します。  
 アニメーションは、オブジェクトの 3 次元の性質を強調し、その空間構造の視覚化やすきます。 これを実現するには、アニメーション可能性があります必要がある完全な円でオブジェクトを回転する緩やかに変化を切り替えるにまたは近いオブジェクトをロール オーバーまたはフォーカスを強調するために、サイズが長きます。  
  
 3 次元のオブジェクトが移動されて、ユーザー コントロールと \(プログラムまたは手動で\)、デザイナーが事前に判断する必要がありますがどのようにに最適をアニメーション化するオブジェクトの最適な知識を提供する移動します。 これは、アニメーションできるようにプログラミングし、ユーザー制御の移動ユーザー オブジェクトを操作する方法を理解するように要求され、オブジェクトの上にカーソルを配置することで、ユーザーがアクティブ化します。 1 つの軸または同時に向きへの移動を制限します。いずれかの規模の設定、回転、または平行移動しますが、複数同時に実行しません。  
  
 視覚エフェクトのカテゴリには、データ、リレーションシップ、状態の側面が含まれています。 構造体、シーケンス、および時間です。  
  
##### データ  
 複雑で、変数の情報を示しています。  
  
-   チャートやグラフなどの情報の視覚エフェクト内を移動  
  
-   シーケンスのステップ実行、ガイド ツアーは、およびページング  
  
-   詳細情報を呼び出すこと、ポイント、および特定の情報を強調表示  
  
-   詳細およびフォーカスのある要素の上に追加の情報をオーバーレイ  
  
-   別の構造上または組織の 1 つの形式から変換  
  
-   時刻のスライダーやジョブ\-シャトル ホイール、トランスポート コントロール \(再生、停止、および一時停止\) を使用して時間の経過と共に変更内容を表します。  
  
##### 関係  
  
-   項目は相互に関連付ける方法、またはどの項目は特定の項目に関係を示しています。  
  
-   階層および親と子または兄弟関係を表示します。  
  
-   別の 1 つの要素を生成します。  
  
-   他の要素を 1 つの要素を最小化します。  
  
-   別テザリングされた 1 つの要素  
  
##### 状態  
  
-   コンテンツの更新。  
  
-   ユーザーのフォーカスと選択  
  
-   進行状況  
  
-   エラー  
  
##### 構造体  
  
-   1 つのノードにある構造体のピボット  
  
-   配列し直す  
  
-   最小限に抑えると最大化、展開し、折りたたみ  
  
##### シーケンス  
  
-   スライド ショーのシーケンス  
  
-   画像を回転  
  
##### 時間  
  
-   時間、時間の経過とスクリーン キャストを表示する変更  
  
-   破棄、元に戻す、およびやり直しに移動します。  
  
-   履歴状態を復元します。  
  
#### 読者の注意  
 ユーザーの注意をいくつかから 1 つの要素を描画するかを最新の情報をユーザーに警告が、目標である場合、アニメーションは適切かもしれません。 などのアプリケーションのスタート ページには、ページが読み込まれた後に、所定の位置にスライドを作業の開始\] ボタンが採用しています。  
  
 原則としては、画面に移動最後の要素は、ユーザーの注意を魅了します。  アニメーション化された要素の一連のユーザーの注意は、前回移動するオブジェクトに従います。  
  
##### 警告  
  
-   ユーザーに警告、注意を引く、進行状況の表示  
  
-   あるいる処理が適切か不適切なを表示または進行中または進行中の変更を表示  
  
-   検索の詳細については、現在のタスクについて学習するなど、タスクの中のユーザーを確認します。  
  
##### 通知  
  
-   エラー条件についてユーザーに警告します。  
  
-   割り込み、ユーザーに確認を希望する別のものに参加するには  
  
-   ゆっくりとプロセスが完了したかなど、ダウンロードが完了すると、変更したユーザーに通知します。  
  
#### シミュレート  
 このカテゴリは、突き詰めると次元について説明します。  
  
-   オブジェクトはどこから来るかに移動する場所を示しています  
  
-   展開し折りたたみまたは開閉  
  
-   パン、スクロール、およびページになります  
  
-   重なり、z オーダー  
  
-   カルーセル、アコーディオン  
  
-   反転と回転 UI  
  
#### 応答と進行状況インジケーター  
 進行状況インジケーターには、複数の重要な利点があります。  
  
-   両方の不確定であると、不確定な進行状況インジケーターは、システムがないクラッシュし、問題に取り組んでいるユーザーを組み込みます。  
  
-   不確定なインジケーターでは、アクションに沿った距離感の進行状況、\[完了\] に近づいて感じだけでなくユーザーを付与します。  
  
##  <a name="BKMK_AnimationPatterns"></a> アニメーション パターン  
  
### 概要  
 Visual Studio でのアニメーションは、特定の機能を提供し、ユーザーの生産性を妨げる可能性になっています。 含める準拠する一般的なアニメーションの特徴:  
  
-   小規模および控え目です  
  
-   自然でリアルです  
  
-   微妙なと末  
  
-   高速で効率的です  
  
-   内部で接続されていないが緩和します。  
  
 次の図は、Visual Studio での使用はお勧めするアニメーションのスタイルを示します。 アニメーションと繊細なアニメーションなどいないのフェードイン\/フェードアウトが最も頻繁に使用します。 拡大\/縮小などのアニメーションの動きの制限のあるアプリケーションは、X と Y 位置変更、および回転します。  
  
 ![Recommended animation styles for Visual Studio](../../extensibility/ux-guidelines/media/1202-a_vsanimstyles.png "1202\-a\_VSAnimStyles")  
  
 **Visual Studio で推奨されるアニメーションのスタイル**  
  
#### 表示または非表示  
 このパターンを持つ要素を視野アウトして、遷移のアニメーションを受けずに表示されてから切り替えます。  
  
 ![Appear&#47;Disappear animation in Visual Studio](../../extensibility/ux-guidelines/media/1202-b_appearanddisappear.png "1202\-b\_AppearAndDisappear")  
  
##### 正しい使用法  
 すぐに表示\/非表示できるように、ユーザーは気を取られるも障害物が必要な最新の UI 要素。 さらに、低速移動のアニメーションは、表示消えるスタイルでは発生しません、パフォーマンスのドラッグとして認識されたことがあります。  
  
##### 不適切な使用  
 場合、変更内容、UI が突然アイデアを持たないユーザーが表示され、コンテキストを理解するを支援アニメーションを追加します。  
  
##### アニメーションのプロパティ  
 遅延時間は、0 秒では一般的です。  
  
##### 例  
  
-   ツール ウィンドウを自動的に隠す  
  
-   IntelliSense やパラメーターのヘルプなどの UI エディターのキーボードによってアクティブ化されます。  
  
-   展開\-折りたたみコード領域  
  
#### フェードインとフェードアウト  
 このパターンで UI 要素が非表示 \(0% の不透明度\) から遷移、またはその逆に表示されている \(不透明度 100%\)。  
  
 ![Fade&#45;in&#47;Fade&#45;out animation in Visual Studio](../../extensibility/ux-guidelines/media/1202-c_fadeinfadeout.png "1202\-c\_FadeInFadeOut")  
  
##### 正しい使用法  
 これにより、UI アニメーションは、最もよく推奨します。 微妙な効果フローを中断することがなく目的を追加することをお勧めします。 場合によっては、ユーザー気付かない可能性がありますもアニメーションし、滑らかでフローさせる UI システムと判断するだけであることです。  
  
##### アニメーションのプロパティ  
  
-   不透明度を開始します 0% フェードイン、フェードアウトを 100%。  
  
-   不透明度を終了します 100% をフェードイン、フェードアウトの 0%。  
  
-   長さ: 200 ミリ秒 standalone、組み合わせのアニメーション シーケンスの一部として使用すると、100 ミリ秒  
  
-   スタイルのイージング: は InOut がサイン \(正弦\)  
  
##### 例  
  
-   ツール ウィンドウを自動的に隠す  
  
-   メニューを開く\/閉じる.  
  
-   前景色および背景\] タブの切り替え
