# <a name="animations-for-visual-studio"></a>Visual Studio のアニメーション
## <a name="animation-fundamentals"></a>アニメーションの基礎

### <a name="animation-best-practices-in-visual-studio"></a>アニメーションのベスト プラクティス
Visual Studio IDE 全体で一貫性があり、ユーザーフレンドリなアニメーション スタイルを確保するには、次の規則に従います。

- **選択的にしてください。** アニメーションを特定の目的に合わせて制限します。

- **タイミングと速度は、** 遷移が迅速かつ自然に感じられるように重要です。

  - アニメーション化されたトランジションを 1.5 秒 (500 ミリ秒) 以内に完了します。

  - 頻繁に発生するアニメーションは、ユーザーのワークフローを中断しないほどの速さである必要があります。 ループでアニメーションを見て、それが正しく感じるまでタイミングを調整します。

  - アニメーションは、理解するのが難しいほど速く、または耳障りであってはならないが、トランジションが終わるのを待ちきりをするほど遅くはない。

  - 重要度を強調するために、可変タイミングを使用します。 たとえば、クラス ダイアグラム上の項目のシーケンスを移動する場合、項目間の遷移を速度を上げて、重要な項目に注目するように速度を低下させます。

- ある状態から別の状態への**緩やかな非線形緩和を使用**して、穏やかで自然な動きを感じさせよ。

- 可能な場合は、マウスの下にインタラクティブな要素を示すために **、ホバー時に微妙なアニメーションを使用**します。

- フィーチャのアニメーションに大きく依存している場合は、[**ツール] >オプション**ダイアログボックスのオプションとして、(すべての機能に対して) ローカルでアニメーションをオフにする**手段を提供**します。

- **一度に 1 つのアニメーションだけを実行**し、1 つの情報だけを伝える必要があります。 複数のオブジェクトを移動したり、複数の物を伝えようとすると、混乱を招く可能性があります。

- **繊細さは重要です。** ほとんどの場合、アニメーションは目的を果たすためにユーザーの注意を必要としません。 タイミング、シーケンス、動作の微妙な変化は知覚に大きな影響を与える可能性があり、効果的なアニメーションと効果のないアニメーションの違いを生み出す可能性があります。

- アニメーションを使って何かに注意を引くときは、ユーザーの思考の訓練**を中断する価値があることを確認してください**。

- アニメーションを使用して**進行状況またはステータスを表示する場合**:

  - 基礎となるプロセスが進んでいないときに、進行状況の動きを表示しない。

  - 不確定プロセスと確定プロセスを区別する。

  - アニメーションに、識別可能な完了状態と失敗状態があることを確認します。

  - 実際の使用に関する追加情報を提供することで、状態を示す効果アニメーションの使用を最小限に抑え、実際の値を持っていることを確認します。 例としては、一時的な状態の変化や緊急事態が含まれます

#### <a name="animation-donts"></a>アニメーションは、しない:

- 小さな動き(小さなフットプリントでの動き)を使用しないでください。 オブジェクトの移動よりもフェードや変更を優先します。

- 画面の広い領域で行われるアニメーションを使用しないでください。 サイズに関係なく、このスタイルのアニメーションはユーザーに対して気が散ります。

- ユーザーが現在フォーカスを設定しているオブジェクトや操作しているオブジェクトとは無関係のアニメーションは使用しないでください。

- ユーザーの操作が状態をリセットする必要があるアニメーションは、点滅を停止させるために点滅する通知に応答するようにユーザーに強制するように使用しないでください。 何らかの方法でそれらと対話することは、それらを却下するのに十分であるべきです。

これらのベスト プラクティスのアプリケーションの詳細については、「[アニメーション パターン](../../extensibility/ux-guidelines/animations-for-visual-studio.md#BKMK_AnimationPatterns)」を参照してください。

### <a name="animation-metrics"></a>アニメーションメトリック

- システムは、10 ミリ秒未満でユーザージェスチャに目に見えて反応する必要があります。

- アニメーションの切り替えが完了するまでに 500 ミリ秒以上かかる必要はありません。

- 時間の長いトランジションを補正する方法の 1 つは、2 つの部分に分ける方法です。 たとえば、アニメーションの最初の部分は空のコンテンツ コンテナ (最大 500 ミリ秒) で、その後にコンテンツがコンテナにフェードインします (最大 500 ミリ秒)。

- 計算できる読み込み時間については、決定要因の進行状況インジケータ (パーセント完了進行状況インジケータ) が優先されます。

- 計算できない読み込み時間の場合、カーソルや埋め込みスピニングアニメーション(読み込みまたは作業インジケータ)などのビジーインジケータが適切です。

### <a name="animation-as-communicator"></a>コミュニケーターとしてのアニメーション
Visual Studio UI では、アニメーションは通信ツールとしてのみ機能します。  UI の構造の変化 (メニューを開いたり閉じたりするなど) のさまざまな情報を伝達するために使用されます。 アニメーションは、インストールの進行状況の視覚化など、複雑なシステムの時間依存の動作を視覚化するのに役立ちます。 アニメーションは、アラートや通知で注目を集めるためにも使用できます。

 UI アニメーションは、通常、視覚化、注目を集める、シミュレートする、応答時間/進行状況のインジケーターの 4 つの方法で機能します。

#### <a name="visualize"></a>視覚化
アニメーションを使用すると、オブジェクトの 3 次元の性質を強調し、ユーザーが空間構造を簡単に視覚化できるようになります。 そのためには、アニメーションでオブジェクトを円全体で回転させるか、ゆっくりと前後に回すか、オブジェクトを近づけて、ロールオーバーまたはフォーカスを強調するためにサイズをわずかに大きくする必要があります。

3 次元オブジェクトはユーザー コントロールと共に移動できますが、デザイナーは、オブジェクトを最適に理解できる動きをアニメーション化する方法を事前に (プログラムまたは手動で) 決定する必要があります。 このプログラムされたアニメーションは、オブジェクトの上にカーソルを置くことによってユーザーがアクティブにできますが、ユーザーが制御する動きは、ユーザーがオブジェクトを操作する方法を理解する必要があります。 一度に 1 つの軸または方向に移動を制限します。スケール、回転、または移動のいずれかを行いますが、同時に複数の操作を行わないでください。

[視覚化] カテゴリには、データ、リレーションシップ、状態、構造、シーケンス、および時間の側面が含まれます。

##### <a name="data"></a>Data
複雑な情報と変数情報を示します。

- グラフやグラフなどの情報の視覚化を通じて移動する

- シーケンスのステップ実行、ガイドツアー、ページング

- 詳細の呼び出し、特定の情報のポイント、ハイライト表示

- フォーカスのある要素の上に詳細と追加情報をオーバーレイする

- ある構造または組織表現から別の構造表現へのモーフィング

- タイム スライダー、ジョグ アンド シャトル ホイール、およびトランスポート コントロール (再生、停止、および一時停止) を使用して、時間の経過に伴う変化を表します。

##### <a name="relationships"></a>リレーションシップ

- アイテムが互いにどのように関連しているか、または特定のアイテムに関連するアイテムを示す

- 階層と親子関係または兄弟関係を表示する

- ある要素が別の要素をスポーンする

- 1 つの要素が別の要素に最小化される

- ある要素が別の要素につながれた

##### <a name="state"></a>State

- コンテンツの更新

- ユーザーのフォーカスと選択

- 進行状況

- エラー

##### <a name="structure"></a>構造体

- 1 つのノードでの構造のピボット

- 方向の変更

- 最小化、最大化、展開、折りたたみ

##### <a name="sequence"></a>Sequence

- スライドショーのシーケンス

- 写真をめく

##### <a name="time"></a>Time

- 時間経過、時間経過、スクリーンキャストの変化を表示する

- ごみ箱に移動、元に戻す、やり直す

- 履歴状態を復元する

#### <a name="attract-attention"></a>注目を集める
目的は、複数の要素のうち 1 つの要素にユーザーの注意を引くか、更新された情報をユーザーに警告する場合、アニメーションが適切である可能性があります。 たとえば、アプリケーションのスタート ページでは、ページの読み込み後に所定の位置にスライドする [はじめに] ボタンが使用される場合があります。

原則として、画面上の最後に移動する要素は、ユーザーの注目を集めています。  一連のアニメーション要素では、ユーザーの注意は最後に動くオブジェクトに従います。

##### <a name="alert"></a>アラート:

- ユーザーに警告、注意を引く、進行状況を表示する

- 何かが正しく行われているか、正しく行われていないことを示すか、進捗や進行状況の変更を表示する

- オンラインで詳細情報を見つけたり、現在のタスクについて学習したりするなど、タスク中にユーザーにプロンプトを表示する

##### <a name="notifications"></a>通知

- エラー状態についてユーザーに警告する

- ユーザーに割り込み、他のユーザーに出席したいかどうかを確認する

- ダウンロードが完了したときのように、プロセスが完了または変更されたことをユーザーに丁寧に通知します。

#### <a name="simulate"></a>シミュレート
このカテゴリは、身体性と次元性をカバーしています。

- オブジェクトがどこから来るか、どこに行くかを説明する

- 展開、折りたたみ、開閉

- パン、スクロール、ページめくり

- スタッキングとzオーダー

- カルーセルとアコーディオン

- UI の反転と回転

#### <a name="response-and-progress-indicators"></a>応答と進捗の指標
進捗インジケータには、いくつかの顕著な利点があります。

- 確定および不確定な進行状況インジケーターは、システムがクラッシュしておらず、問題に取り組んでいるユーザーを安心させます。

- 確定指標は、アクションがどこまで進行しているかをユーザーに知り、フィニッシュに近づく感覚を与えます。

## <a name="animation-patterns"></a><a name="BKMK_AnimationPatterns"></a>アニメーションパターン

### <a name="overview"></a>概要
Visual Studio のアニメーションは、ユーザーの生産性を低下させることなく、特定の機能を提供することを目的とします。 一般に、Visual Studio のアニメーションは次のようになります。

- 小さくて目立たない

- 自然で現実的

- 微妙で落ち着いた

- 高速かつ効率的

- リラックスして、急いでいない

この図は、Visual Studio で推奨されるアニメーション スタイルを示しています。 フェードイン/フェードアウトのようなアニメーションや微妙なアニメーションは、最も頻繁に使用されません。 展開や縮小、X と Y の位置変更、回転などの移動アニメーションの用途は限られています。

![Visual Studio で推奨されるアニメーションのスタイル](../../extensibility/ux-guidelines/media/1202-a_vsanimstyles.png "1202-a_VSAnimStyles")<br />Visual Studio で推奨されるアニメーションのスタイル

#### <a name="appear-and-disappear"></a>表示され、消える
このパターンでは、要素は遷移アニメーションなしで、表示から非表示に切り替わります。

![アニメーションを表示および非表示にする](../../extensibility/ux-guidelines/media/1202-b_appearanddisappear.png "1202-b_AppearAndDisappear")<br />アニメーションを表示および非表示にする

##### <a name="correct-usage"></a>正しい使用法
ユーザーが気を散らしたり妨げたりすることがないように、即座に表示または非表示にする必要がある新しい UI 要素。 また、低速で動くアニメーションは、表示と非表示のスタイルでは発生しないパフォーマンス ドラッグとして認識される場合があります。

##### <a name="incorrect-usage"></a>不適切な使用法
UI が突然表示される場合、ユーザーは何が起こったのか分からず、アニメーションを追加するとコンテキストを理解するのに役立ちます。

##### <a name="animation-properties"></a>アニメーションプロパティ
遅延時間は、通常 0 秒です。

##### <a name="examples"></a>使用例
- ツール ウィンドウを自動的に非表示にする

- キーボードでアクティブ化されたエディター UI (IntelliSense やパラメーター ヘルプなど)

- コード領域の展開と折りたたみ

#### <a name="fade-in-and-fade-out"></a>フェードインとフェードアウト
このパターンでは、UI 要素は表示されない (不透明度が 0%) から可視 (不透明度 100%) またはその逆に遷移します。

![フェードインとフェードアウトアニメーション](../../extensibility/ux-guidelines/media/1202-c_fadeinfadeout.png "1202-c_FadeInFadeOut")<br />フェードインとフェードアウトアニメーション

##### <a name="correct-usage"></a>正しい使用法
これは、最も一般的に推奨される UI アニメーションです。 それは流れを中断することなく興味を追加する微妙な効果です。 場合によっては、ユーザーは、滑らかで流れる UI システムを認識するアニメーションがあることに気付かない場合があります。

##### <a name="animation-properties"></a>アニメーションプロパティ

- 開始不透明度: フェードインの場合は 0%、フェードアウトの場合は 100%

- 終了不透明度:フェードインの場合は100%、フェードアウトの場合は0%

- 持続時間: 200 ミリ秒のスタンドアロン、組み合わせアニメーション シーケンスの一部として使用した場合は 100 ミリ秒

- イージングスタイル:シネインアウト

##### <a name="examples"></a>使用例

- ツール ウィンドウを自動的に非表示にする

- メニューを開閉する

- 背景と前景タブの切り替え

##### <a name="correct-usage"></a>正しい使用法

- 持続時間: 200 ミリ秒のスタンドアロン、組み合わせアニメーション シーケンスの一部として使用した場合は 100 ミリ秒

##### <a name="examples"></a>使用例
