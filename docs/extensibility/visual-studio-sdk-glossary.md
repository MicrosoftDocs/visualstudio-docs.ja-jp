## <a name="terms"></a>用語
 アドイン ユーティリティ アプリケーション、ドライバ、またはプライマリ アプリケーションに追加されたその他のソフトウェア。 Visual Studio 統合開発環境 (IDE) では、アドインは IDE の機能を拡張するオートメーション ベースのアプリケーションです。

 オートメーション モデル 以前のバージョンの Visual Studio で拡張機能モデルとして知られていたオートメーション モデルは、IDE を駆動する基になるルーチンにアクセスできるプログラミング インターフェイスです。 アドイン、ウィザード、およびマクロは、オートメーション モデルのオブジェクトを使用して、IDE の機能を制御または拡張します。

 コマンド UI コンテキスト UI コマンドまたは要素の表示と GUID の関連付け (ツール バーなど)。 コマンド UI コンテキストは、ウィンドウにアタッチされていないという点で選択コンテキストとは異なります。

 コマンド UI コンテキストは、次の用途に使用できます。

- 特定のウィンドウがアクティブになったときに表示されるツール バーに GUID を割り当てます。

- コマンドを読み込んだり、VSPackage を実行したりすることなく、コマンドの可用性に GUID を割り当てます。

- アクティブなキーのバインドに影響を与える GUID を割り当てます。

- マクロの記録を有効にする GUID を割り当てます。

- デバッグ モードをアクティブ化するか、エディターでデザイン モードと実行モードを切り替えるには、GUID を割り当てます。

  DocData ドキュメント/ビューの分離がある世界でドキュメントの基になるデータをカプセル化する COM オブジェクト (たとえば、テキスト エディターの場合、これはすべてのテキスト エディター ビューの基になるテキスト バッファー)。 EditorFactory がこのオブジェクトを提供しない場合、IDE はそのオブジェクトに代わってオブジェクトを製造します。 このオブジェクトの責任は、この同じ`DocData`に対する複数のビューのデータの永続性と共有セマンティクスを管理することです。 オブジェクトが`DocData`インターフェイスをサポート`IOleCommandTarget`している場合は、UIShell のコマンド ルーティングに含まれます。

  拡張 IDE を変更、カスタマイズ、または追加する機能。 拡張機能は、オートメーション モデルまたは VSPackages を使用して作成します。

  外部エディタ Microsoft Word など、IDE に固有ではないエディターです。 独自のメカニズムを使用して登録されており、IDE の外部で使用できます。 このエディタを埋め込むことができる場合は、IDE のウィンドウ内に表示されます。 埋め込みできない場合は、別のトップレベル ウィンドウが作成されます。

  情報プロバイダ情報プロバイダは、キーワードを検索し、オブジェクトの形式でトピックのリストを返すことができるモジュールです`IVsUserContextItem`。 情報プロバイダに**F1**および検索キーワード項目を提供するには、コンパイル済みのヘルプ ファイル (*.HxS*) とシステムを使用します。 これらのファイルのヘルプ トピックには、[ダイナミック ヘルプ] ウィンドウに表示されるトピックの一覧が表示され、ユーザーが**F1**キーを押したかどうかが示されます。
