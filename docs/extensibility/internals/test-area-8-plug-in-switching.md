# <a name="test-area-8-plug-in-switching"></a>テスト領域 8: プラグインの切り替え
[!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)]統合開発環境 (IDE) には、現在のソース管理プラグインを変更するユーザー インターフェイス (UI) があります。 このテスト領域は、ソリューションソース管理に使用するプラグインを選択するプロセスのテスト ケースを提供します。

## <a name="command-menu-access"></a>コマンド メニューへのアクセス
 テスト[!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)]ケースでは、次の統合開発環境メニュー パスが使用されます。

- 現在のソース管理プラグイン:**ツール** -> **オプション** -> **ソース管理** -> **プラグインの選択**.

- ソース管理バインドの変更:**ファイル** -> **ソース管理** -> **の変更ソース管理**.

## <a name="common-expected-behavior"></a>一般的な予期される動作
 Visual Studio を終了するか、ソリューションを再読み込みせずに、ソリューションのソース管理プラグインを変更できます。 また、現在のソース管理プラグインは、ソリューションが読み込まれたときに、ソリューションで使用されるプラグインに自動的に変更されます。

## <a name="test-cases"></a>テスト ケース
 プラグインの切り替えテスト領域の具体的なテスト ケースを次に示します。

### <a name="case-8a-automatic-change"></a>ケース 8a: 自動変更

#### <a name="expected-behavior"></a>想定されている動作
 ユーザーがソース管理下にあるソリューションを読み込むとき、ソリューションは自動的に読み込まれ、適切なソース管理プラグインが現在として選択されます。

| アクション | テストステップ | 検証する期待される結果 |
| - | - | - |
| ソース管理プラグインの自動変更 | 1. 現在のプラグインを選択する (**ツール** -> **オプション** -> **ソース管理** -> **プラグインの選択**.)<br />2. 新しいプロジェクトを作成します。<br />3. ソース管理にソリューションを追加します。<br />4. 別のプラグインを選択します ([!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)]例: )。<br />5. ソリューションのアンロードを受け入れます。<br />6. ディスクからソリューションを再度開きます。 | ソリューションが開かれます。<br /><br /> テスト対象のプラグインは、現在のソース管理プラグインです。 |

### <a name="case-8b-solution-based-change"></a>ケース 8b: ソリューションベースの変更

#### <a name="expected-behavior"></a>想定されている動作
 ソリューションは、関連付けられているソース管理プラグインを変更できます。

| アクション | テストステップ | 検証する期待される結果 |
|----------------------------------| - | - |
| ソリューションのプラグインの変更 | 1. 現在のプラグインとしてテスト対象プラグインを選択します (**[ツール** -> **オプション]** -> **ソース管理** -> **プラグインの選択**)。<br />2. 新しいプロジェクトとソリューションを作成します。<br />3. ソース管理にソリューションを追加します。<br />4. ソース管理からソリューションのバインドを解除する **([ソース管理の変更**] ダイアログ ボックスを使用して)。<br />5. 別のプラグインを選択します ([!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)]例: )。<br />6. アンロードされた場合は、ディスクからソリューションを再ロードします。<br />7. ソリューションをソース管理に追加します。<br />8. ソース管理からソリューションのバインドを解除します **([ソース管理の変更**] ダイアログ ボックスを使用)。<br />9. もう一度テスト対象のプラグインを選択します。<br />10. アンロードされた場合は、ディスクからソリューションを再ロードします。<br />11. ソリューションを元の場所にバインドします **([ソース管理の変更**] ダイアログ ボックスを使用します)。 | ソリューションは、選択したプラグインを使用してソース管理に追加されます。 |
