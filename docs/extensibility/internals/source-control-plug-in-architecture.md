# ソース管理プラグイン アーキテクチャ
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

[!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] の統合開発環境をソース管理プラグインを実装してアタッチ \(IDE\) してソース管理のサポートを追加できます。  IDE で定義されているソース管理プラグイン API によってソース管理プラグインに接続します。  IDE ではツール バーとメニューのコマンドで構成されるユーザー インターフェイスを提供することに \(UI\) よりソース管理システムのバージョン コントロールの機能を公開します。  ソース管理プラグインはソース管理機能を実装します。  
  
## ソース管理プラグインのリソース  
 ソース管理プラグインは [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] IDE にバージョン管理アプリケーションを作成し接続に役立つリソースを提供します。  ソース管理プラグインは [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] IDE に統合できるようソース管理プラグインが実装する必要のある API の仕様を格納します。  またそのコード例 \(C\+\+ で記述された実装ソース管理プラグイン API が必要な関数のスケルトン ソース管理プラグインを示す実装が含まれます。  
  
 ソース管理プラグイン API の仕様はソース管理プラグイン API に実行される関数に必要な設定でソース管理の DLL を作成すると選択したソース管理システムを行うことができます。  
  
## コンポーネント  
 図のソース管理のアダプターのパッケージはソース管理プラグインでサポートされている関数呼び出しのソース管理操作のユーザー要求を変換する IDE コンポーネントです。  発生したためにはIDE およびソース管理プラグインとプラグインはIDE の間で情報を渡す前に有効なダイアログが必要です。  のこのダイアログ ボックスではそれらの両方が同じ言語を話さなければ必要があります。  このドキュメントに記載されているソース管理プラグイン API はこの交換に共通の用語です。  
  
 ![ソース コード管理アーキテクチャ ダイアグラム](../../extensibility/internals/media/vs_sccsdk_plug_in_arch.png "vs\_sccsdk\_plug\_in\_arch")  
アーキテクチャ図の相互作用の表示とその間ソース管理プラグイン  
  
 アーキテクチャ図に示すように図のようにラベルとシェル [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] シェル エディターはソリューション エクスプローラーなどのユーザーのプロジェクトに関連付けられたコンポーネントをホストする。  ソース管理のアダプターのパッケージはIDE でソース管理プラグインの間の相互作用を処理します。  ソース管理のアダプターのパッケージは独自のソース管理の UI を提供します。  これはソース管理操作の範囲の先頭および定義するためにユーザーが対話できます。トップレベルの UI があります。  
  
 ソース管理プラグインは図に示すように2 部分で構成される独自の UI を持つことができます。  「販売店 UI 」というラベルの付いたボックスではソース管理プラグインの作成者として機能するカスタム ユーザー インターフェイス要素を表します。  これらはソース管理プラグインによってユーザーにより高度なソース管理操作を開始するときに直接表示されます。  「ヘルパー UI 」というラベルの付いたボックスでは一連のソース管理の IDE に間接的に呼び出すプラグインの UI 機能です。  ソース管理プラグインはIDE を提供する特別なコールバック関数によって IDE に UI 関連のメッセージを渡します。  ヘルパーの UI は IDE のシームレスに統合 \(ENT0ENT \[入力\] ボタンを使用して\) 容易にするためより統合エンド ユーザーの操作を提供します。  
  
 したがってソース管理プラグインは [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] とIDE によって提供されるソース管理のアダプターのパッケージまたはソース管理の UI の変更をシェルから取り出させますことはできません。  これはエンド ユーザー用の統合環境で使用するさまざまなソース管理プラグイン API 関数の実装によって提供される柔軟性を最大限に利用する必要があります。  ソース管理プラグイン API のドキュメントの参照セクションは高度なソース管理プラグインの機能に関する情報も含まれています。  これらの機能を開発するにはソース管理プラグインは初期化中に IDE に高度な機能を宣言し各機能の特定の最新機能を実装する必要があります。
