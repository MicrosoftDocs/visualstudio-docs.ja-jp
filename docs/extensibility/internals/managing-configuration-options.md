# 構成オプションの管理
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

新しいプロジェクトを作成するとプロジェクトは配置する包まれてのビルド実行および決定するプロジェクトとソリューション構成設定を管理する必要があります。  次のトピックではプロジェクトやソリューション構成について説明します。  
  
## このセクションの内容  
 [概要](../../extensibility/internals/configuration-options-overview.md)  
 [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] のプロジェクトに複数の構成をサポートする方法について説明します。  
  
 [\[プロパティ ページ\]](../../extensibility/internals/property-pages.md)  
 ユーザーがプロパティ ページを使用してプロジェクト構成の依存関係プロパティと独立したプロパティを表示および変更する方法を示します。  
  
 [ソリューション構成](../../extensibility/internals/solution-configuration.md)  
 格納されているものまたはソリューション構成  **開始**  と  **ビルド**  のコマンドの動作を指定する方法についての情報をソリューション構成に示します。  
  
 [プロジェクト構成オブジェクト](../../extensibility/internals/project-configuration-object.md)  
 プロジェクトの構成が UI オブジェクトに構成情報の表示を管理する方法について説明します。  
  
 [構築するためのプロジェクトの構成](../../extensibility/internals/project-configuration-for-building.md)  
 特定のソリューションのソリューション構成の一覧が ENT0ENT \[出力\] ダイアログ ボックスでマネージどのように処理するかを説明します。  
  
 [展開を管理するためのプロジェクトの構成](../../extensibility/internals/project-configuration-for-managing-deployment.md)  
 配置をサポートする 2 とおりの方法 [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] のサポートと配置プロジェクトの動作を定義します。  
  
 [出力のプロジェクト構成](../../extensibility/internals/project-configuration-for-output.md)  
 構成のサポートは出力項目が使用できるようにすることができるメソッドおよびインターフェイス ビルド処理について説明します。
