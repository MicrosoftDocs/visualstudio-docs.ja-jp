# テスト領域 3: チェック アウト]、[チェック アウトを元に戻す
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

このソース管理プラグインのテストの領域をカバーを使用してバージョン ストアから取得した項目を編集し、元に戻して、 **チェック アウト** と **チェック アウトの取り消し** コマンドです。  
  
 **チェック アウト**: マーク バージョン ストア内の項目をチェック アウトとローカルのコピーを変更読み取り\/書き込み可能にします。  
  
 **チェック アウトの取り消し**: でチェックするように、バージョン ストア内の項目をマークをローカル コピーをチェック アウト \(オプション\) に応じて前に、の状態を元に戻します。  
  
## コマンド メニューへのアクセス  
 次 [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] 統合開発環境\] メニューの \[パスは、テスト\_ケースで使用します。  
  
##### 参照してをください。  
  
-   **ファイル**, 、**ソース管理**, 、**をチェック アウト**します。  
  
-   **ファイル**, 、**をチェック アウト**します。  
  
-   ショートカット メニューの \[ **をチェック アウト**します。  
  
-   チェック アウトの取り消し: **ファイル**, 、**ソース管理**, 、**チェック アウトを元に戻す**します。  
  
## 一般的な想定される動作  
  
-   \[チェック アウト操作の後に、対象のファイルやフォルダー マークが付けられたバージョン ストア内のチェック アウトします。  
  
-   バージョン ストアは、正しいユーザーをチェック アウトを求めます。  
  
-   チェック アウトの日付と時刻は、正しい \(ごとのユーザーの設定\) です。

### ケース 3 b: チェック アウトの切断  
 非接続モードで動作するいると、ユーザーがバージョン ストアに直接接続されていない場合に、継続的なソース管理サポートのいくつかのレベルにできます。 これは、参加しているソリューションとプロジェクトのすべての関連情報をローカルにキャッシュすることによって行います。  
  
 排他チェック アウト操作は、ソース管理ストアに接続されている場合にのみ発生します。 共有チェック アウト操作は、いつでも、接続または切断されたかどうかに発生します。 そのため、バージョン ストアにのみから切断されている場合、 **共有を確認して** \(CO\) コマンドが有効になっています。 切断された状態、 **チェック アウトの取り消し** 、ユーザーが行った変更を置換する古いバージョンを取得できないためには無効になっています。  
  
 バージョンに、ユーザーが再接続するときは、参加しているソリューションのすべてのチェック アウト状態を格納され、プロジェクトが同期されます。 これは、ユーザーが実行のチェック アウトをストアに必要な更新します。 同期の問題が発生すると、ユーザーは通常どおり \(接続\) の操作を続行することです。  
  
#### 予想される動作  
  
-   使用できません **チェックのみを** コマンドをバージョン ストアから切断された状態です。  
  
-   使用できません **チェック アウトの取り消し** コマンドをバージョン ストアから切断された状態です。  
  
-   **共有チェック アウト** コマンドは動作します。  
  
|アクション|テスト ステップ|期待どおりの結果を確認するには|  
|-----------|--------------|---------------------|  
|ファイルを確認してから、同期用の接続、切断された状態|1.  ソース管理の変更\] ダイアログ ボックスを使用して、制御されたプロジェクトを切断 \(**ファイル**, 、**ソース管理**, 、**変更のソース管理**l\)。<br />2.  ファイルをチェック アウトします。<br />3.  チェック アウトの警告\] ダイアログ ボックスには、\(切断\) をクリックします。<br />4.  ファイルを編集します。<br />5.  ソース管理の変更\] ダイアログ ボックスを使用して接続します。<br />6.  編集したファイルの最新バージョンを取得します。|一般的な想定される動作|  
  
### ケース 3 c: クエリの編集\/クエリを保存 \(QEQS\)  
 ソース管理下にあるアイテムは、編集、変更追跡し、ファイルを管理するユーザーを容易に保存します。 「チェックイン」コントロールの項目を編集する際に QEQS が実行しようとした編集をインターセプトし、接続を編集するファイルをチェック アウトするかかどうかユーザーに確認します。 に応じて **ツール**, 、**オプション** 設定、ユーザーは、いずれかの確認を強制的に、ファイルを編集するために出力をメモリ内のコピーを編集して、後でチェック アウトできる場合があります。 場合、ユーザーの **ツール**, 、**オプション** 設定をチェック アウト\] ダイアログ ボックスを表示し、だけチェック アウトに設定されていませんし、ユーザーが自分の編集を行うと、ファイルに自動的にチェック アウト、可能な場合です。  
  
#### 予想される動作

### ケース 3 e: チェック アウトの取り消し  
 **チェック アウトの取り消し** にチェック アウトの状態ファイルをキャンセルし、ファイルに加えられた変更のチェックインを回避するために使用します。  
  
#### 予想される動作  
  
-   既定値は、ユーザーのに基づいて **ローカル バージョンをチェック アウト** 設定します。 場合はチェック アウトの取り消しの既定値は常にチェック アウトされているバージョンに戻すには、ユーザーがローカル バージョンをチェック アウトを選択します。  
  
-   元に戻す、内のアイコンの同意 **ソリューション エクスプ ローラー** の影響を受けるは更新ファイルと、項目を削除してから、 **保留中のチェックイン** ウィンドウです。

|-----------|--------------|---------------------|
