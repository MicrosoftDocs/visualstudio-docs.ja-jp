# <a name="test-area-3-check-outundo-checkout"></a>テスト領域 3: チェック アウト/チェック アウトの取り消し
このソース管理プラグインのテストの領域を使用して、バージョン ストアから項目を編集し、元に戻してをカバーする、**チェック アウト**と**チェック アウトの取り消し**コマンド。

**チェック アウト**:マークとしてバージョン ストア内の項目がチェック アウトは、読み取り/書き込みをローカル コピーを変更します。

**チェック アウトを元に戻す**:マークとしてバージョン ストア内の項目のチェックインは、ローカル コピーをチェック アウト (オプション) に応じて前に、の状態を元に戻します。

## <a name="command-menu-access"></a>コマンド メニューへのアクセス

次[!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)]統合開発環境のメニューのパスは、テスト_ケースで使用されます。

##### <a name="check-out"></a>チェックアウト：

- **ファイル**、**ソース管理**、**をチェック アウト**します。

- **ファイル**、**をチェック アウト**します。

- ショートカット メニューの **をチェック アウト**します。

- チェック アウトを取り消し。**ファイル**、**ソース管理**、**チェック アウトを元に戻す**します。

## <a name="common-expected-behavior"></a>共通の想定される動作

- チェック アウト操作には後、対象のファイルやフォルダーとしてマークされたバージョン ストアでチェック アウトします。

- バージョン ストアは、適切なユーザーにチェック アウトを属性です。

- チェック アウトの日付と時刻は正しい (ごとのユーザーの設定です)。

### <a name="case-3b-disconnected-check-out"></a>ケース 3 b:非接続のチェック アウト

切断モードで動作するいると、ユーザーがバージョン ストアに直接接続されていない場合に、継続的なソース管理サポートのいくつかのレベルにできます。 これは、参加しているソリューションとプロジェクトに関するすべての関連情報をローカルにキャッシュすることによって行います。

排他チェック アウト操作は、ソース管理ストアに接続されているときにのみ発生します。 接続または切断されたかどうか、共有チェック アウト操作は、いつでも発生します。 そのため、のみのバージョン ストアから切断されている場合、**チェック アウト共有**(CO) コマンドが有効になっています。 切断された状態、**チェック アウトの取り消し**ユーザーが行った変更を置換する古いバージョンを取得できないためには無効になっています。

ユーザーが、バージョンに再接続するときに、参加しているソリューションのすべてのチェック アウト状態を格納され、プロジェクトが同期されます。 これは、ユーザーが実行をチェック アウトのストアに必要な更新プログラムを行います。 同期が発生した後、ユーザーが通常どおり (接続されている) の操作を続行できません。

- 使用できません**チェックのみを**バージョン ストアから切断されていてもコマンドします。

- 使用できない**チェック アウトの取り消し**バージョン ストアから切断されていてもコマンドします。

- **共有チェック アウト**works コマンドします。
