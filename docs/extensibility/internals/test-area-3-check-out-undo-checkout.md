# <a name="test-area-3-check-outundo-checkout"></a>テストの領域 3: チェック アウト/チェック アウトを元に戻す
このソース管理プラグインのテストの領域の説明を使用して、バージョン ストアから項目を編集し、元に戻す、**チェック アウト**と**チェック アウトの取り消し**コマンド。  
  
 **チェック アウト**: マーク バージョン ストア内の項目チェック アウトされているように変更を読み取り/書き込みのローカル コピーします。  
  
 **チェック アウトを元に戻す**: チェックインするように、バージョン ストア内の項目を示し、ローカルのコピーをチェック アウト (オプション) に応じて前に、の状態を元に戻します。  
  
## <a name="command-menu-access"></a>コマンド メニューへのアクセス  
 次[!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)]統合開発環境] メニューの [パスは、テストの場合に使用されます。  
  
##### <a name="check-out"></a>チェックアウト：  
  
-   **ファイル**、**ソース コントロール**、**チェック アウト**です。  
  
-   **ファイル**、**チェック アウト**です。  
  
-   ショートカット メニューの **チェック アウト**です。  
  
-   チェック アウトを取り消す:**ファイル**、**ソース コントロール**、**チェック アウトを元に戻す**です。  
  
## <a name="common-expected-behavior"></a>共通の想定される動作  
  
-   チェック アウト操作後に、対象のファイルやフォルダーとしてマークされます、バージョン ストアでチェック アウトします。  
  
-   バージョン ストアは、適切なユーザーにチェック アウトを属性します。  
  
-   チェック アウトの日付と時刻は、正しい (ごとのユーザーの設定) です。

### <a name="case-3b-disconnected-check-out"></a>ケース 3 b: チェック アウトの切断  
 非接続モードで動作するいると、ユーザーがバージョン ストアに直接接続されていない場合に継続的なソース コントロールのサポートのいくつかのレベルにできます。 これは、参加しているソリューションとプロジェクトのすべての関連情報をローカルにキャッシュすることによって行います。  
  
 排他チェック アウト操作は、ソース管理ストアに接続されている場合にのみ発生します。 接続または切断されたかどうかは、共有チェック アウト操作をいつでも発生します。 そのため、のみのバージョン ストアから切断されている場合、**チェック アウトの共有**(CO) コマンドを有効にします。 切断されていて、**チェック アウトの取り消し**ユーザーによって行われた変更を置換する古いバージョンを取得できないためには無効になっています。  
  
 ユーザーが、バージョンに再接続するときに、参加しているソリューションのすべてのチェック アウト状態を格納およびプロジェクトが同期されます。 これはストアに必要な更新プログラム、ユーザーが実行をチェック アウトします。 同期が発生した後、ユーザーは通常どおり (接続されている) の作業を続行することです。  
  
#### <a name="expected-behavior"></a>想定される動作  
  
-   使用できません**チェックのみを**バージョン ストアから切断されている間にコマンドします。  
  
-   使用できません**チェック アウトの取り消し**バージョン ストアから切断されている間にコマンドします。  
  
-   **共有チェック アウト**コマンドは動作します。  
  
|アクション|テスト ステップ|期待される結果を確認するには|  
|------------|----------------|--------------------------------|
