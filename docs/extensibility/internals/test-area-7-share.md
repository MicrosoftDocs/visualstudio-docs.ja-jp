# <a name="test-area-7-share"></a>テスト領域 7: 共有
このテスト領域では、[共有]コマンドを使用して、場所間でアイテムを**共有**できます。

 hhare 操作は、ソース管理ファイル階層内の複数の場所の間でファイルとフォルダ項目が重複している可能性があります。 複製はサーバー上で実際には発生しませんが、ユーザーは指定した 2 つ以上の場所に同じファイルを表示します。 共有アイテムに変更が加えられた場合は、その変更が他のすべての共有場所に反映されます。

 フォルダ内での共有は、ソース管理下に少なくとも 1 つのファイルがあるフォルダを選択すると機能します。 共有コマンドは、次の条件で無効になります。

- 選択したフォルダが空のフォルダの場合。

- 実際のフォルダーがある場合、ソース管理ファイルは含みなくなります。

- 仮想フォルダがある場合、ソース管理下のファイルが存在するかどうか。

- リモート サイト Web プロジェクトがある場合。

## <a name="command-menu-access"></a>コマンド メニューへのアクセス
 テスト[!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)]ケースでは、次の統合開発環境メニュー パスが使用されます。

 共有:**ファイル**->**ソース管理**->**共有**。

## <a name="expected-behavior"></a>想定されている動作

- 共有ファイルは共有の場所に表示されます。

- ソース管理のバージョン ストア履歴を表示すると、ファイルが共有されていることを示します。

- 共有ファイルを編集すると、ファイルの両方の場所が編集されます。

## <a name="test-cases"></a>テスト ケース
 次に、共有テスト領域の特定のテスト ケースを示します。

|プロジェクト間でファイルを共有する|1. 新しいプロジェクトを作成します。<br />2. ソース管理に追加します。<br />3. ソリューションを閉じます。<br />4. 2 つ目のプロジェクトを作成する (新しいソリューション)。<br />5. ソース管理にソリューションを追加します。<br />6. プロジェクトを選択します。<br />7. **[共有**] ダイアログ ボックス (**ファイル** -> **ソース管理** -> **共有**) を開きます。<br />8. 以前に追加したプロジェクトのファイルを開いているプロジェクトに共有します。<br />9. 確認メッセージが表示されたら **、チェックアウト**を受け入れます。|一般的な予期される動作。|
|ソース管理から現在読み込まれているプロジェクトにプロジェクトの一部ではないファイルを共有する|1. 新しいプロジェクトを作成します。<br />2. ソリューションをソース管理に追加します。<br />3. プロジェクトまたはソリューションの一部ではないソース管理にファイルを追加します。<br />4. プロジェクトを選択し、[**共有**] ダイアログ ボックス (**ファイル** -> **ソース管理** -> **共有**) を開きます。<br />5. [**共有**]ダイアログ ボックス内のファイルを選択し、現在のプロジェクトまたはソリューション内に存在しないファイルを共有します。<br />6. 確認メッセージが表示されたら **、チェックアウト**を受け入れます。|ソース管理ストアが Get を実行したので、ファイルはプロジェクトのローカルの場所に配置されます。|
|同じプロジェクト内のファイルを別のフォルダに共有する|1. **[ツール** -> オプション**ソース管理****]** -> で [**自動的にチェックアウト**] を選択します。<br />2. 新しいプロジェクトを作成し、ソース管理に追加します。<br />3. プロジェクトにフォルダを追加します。<br />4. フォルダにファイルを追加し、フォルダをチェックインします。<br />5. フォルダを選択します。<br />6. **[共有**] ダイアログ ボックス (**ファイル** -> **ソース管理** -> **共有**) を開きます。<br />7. 選択したフォルダにファイルを共有します。|一般的な予期される動作。<br /><br /> フォルダを共有に使用するには、フォルダ内のファイルをチェックインする必要があります。|
