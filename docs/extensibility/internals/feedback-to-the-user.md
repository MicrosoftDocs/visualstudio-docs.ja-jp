# <a name="feedback-to-the-user"></a>ユーザーへのフィードバック
[!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)]統合開発環境 (IDE) では、利用可能な機能に関する視覚的なフィードバックは、ユーザーの現在の選択とグローバル選択コンテキストに基づいています。 次の表は、さまざまな選択コンテキストで使用できる機能を示しています。

|選択コンテキスト|使用可能な機能|
|-----------------------|-----------------------------|
|IDE|グローバル|
|現在の製品セット|製品固有|
|アクティブ階層|階層タイプ固有|
|有効な階層品目|階層品目タイプ固有|
|アクティブドキュメント|ドキュメント タイプ固有|
|最上位のマルチドキュメント インターフェイス (MDI) ウィンドウ|ウィンドウタイプ固有|
|現在の選択コンテキスト|選択コンテキスト固有|

 ユーザーが必要とする機能だけを表示し、一貫した選択と環境コンテキストのフィードバックを継続的に提供する場合は、IDE の複雑さを軽減します。 IDE でウィンドウを開く場合は、次の規則が適用されます。

- ウィンドウの選択コンテキストが変更されると、選択フィードバックがウィンドウに明確に表示され、表示されている場合は **[ダイナミック ヘルプ**]ウィンドウが更新されて現在のコンテキストが反映されます。

- ウィンドウがグローバル選択コンテキストを変更すると、コンテキスト固有のすべてのメニュー、アクティブな階層ウィンドウ、およびアプリケーションタイトルバーが更新され、現在のコンテキストが反映されます。

- ウィンドウは **、[プロパティ]** ウィンドウで現在選択されているプロパティを表示し、必要に応じて [**プロパティ ページ]** ダイアログ ボックスを表示する必要があります。

- ウィンドウがプロパティを表示しないか、グローバル選択コンテキストを変更しない場合、選択フィードバックは、IDE でアクティブなウィンドウでなくなったウィンドウに残ってはなりません。

- ドキュメント固有のツール ウィンドウはすべて、アクティブなドキュメントを継続的に反映する必要があります。

- メニュー、ツールバー、およびアプリケーションタイトルバーは、最上位のマルチドキュメントインタフェース (MDI) クライアントウィンドウを反映する必要があります。

  たとえば、Visual Basic Web アプリケーション プロジェクト内の**Web フォーム**の HTML ビューを開き、`<td>`ユーザーがタグを選択すると、次のようにフィードバックが提供されます。

- 選択はアクティブウィンドウに表示され、**プロパティ**ウィンドウに反映されます。

- ドキュメント固有の**ツールボックス**が更新され、アクティブなドキュメントが反映されます。

- **[エディター** ] ツールバーと **[表]** メニューが表示され、タイトル バーが Web フォーム ウィンドウを反映するように更新されます。

- アクティブな階層ウィンドウ (通常は**ソリューション エクスプローラ**) とタイトル バーの更新により、現在のコンテキストとコンテキスト依存の**プロジェクト**メニュー コマンドがアクティブな Web アプリケーション プロジェクトに適用されるようになりました。
