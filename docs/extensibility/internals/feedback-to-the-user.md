# ユーザーへのフィードバック
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

[!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] の統合開発環境で \(IDE\) は使用可能な機能に関する視覚的フィードバックはユーザーの現在の選択項目およびグローバルのコンテキストに基づいています。  次の表は選択のコンテキストで使用できる機能を示します。  
  
|選択のコンテキスト|使用可能な機能|  
|---------------|-------------|  
|IDE|グローバル|  
|現在の積集合|製品固有|  
|アクティブな階層|階層の種類の指定|  
|アクティブな項目の階層|階層内の項目の種類の指定|  
|Active ドキュメント|ドキュメントの種類の指定|  
|最上位のマルチ ドキュメント インターフェイス\] \(MDI\) ウィンドウ|ウィンドウの種類の指定|  
|現在のコンテキスト|選択したコンテキストの指定|  
  
 領域のみ機能のユーザーのニーズ一貫した選択を提供し環境のコンテキスト フィードバックIDE の複雑さを軽減できます。  次の規則はウィンドウを IDE で開くたびに対象 :  
  
-   ウィンドウを選択のコンテキストを変更すると選択の情報はウィンドウで明瞭に表示されダイナミック ヘルプ ウィンドウが表示されると現在のコンテキストを反映するように更新されます。  
  
-   ウィンドウがグローバルな選択のコンテキストを変更した場合現在のコンテキストを反映するようにすべてのコンテキスト固有のメニューアクティブな階層\] ウィンドウおよびアプリケーションのタイトル バーが更新されます。  
  
-   ウィンドウが表示 ENT1ENT\[オプション\] ダイアログ ボックスの \[ENT0ENT\] ウィンドウでの現在の選択範囲に対して発生しプロパティ。  
  
-   プロパティ ウィンドウが発生せずグローバルな選択のコンテキストを変更しないと選択したウィンドウによりフィードバックはIDE のアクティブ ウィンドウである場合に維持する必要があります。  
  
-   すべてのドキュメント固有のツール ウィンドウはアクティブな文書を反映する必要があります。  
  
-   メニューツール バーアプリケーションのタイトル バーには最上位のマルチ ドキュメント インターフェイス \(MDI\) クライアント ウィンドウを反映する \(MDI\) 必要があります。  
  
 たとえばVisual Basic Web アプリケーション プロジェクトから Web フォームの HTML ビューを開きユーザーが `<td>` のタグを選択するとフィードバックは次の方法で提供されます :  
  
-   選択がアクティブ ウィンドウに表示されENT0ENT \[出力\] ウィンドウに反映されます。  
  
-   ドキュメント固有  **ツールボックス**  ではアクティブ ドキュメントを反映するように更新されます。  
  
-   \[入力\] ENT0ENT ツール バーと \[入力\] メニューの ENT1ENT はタイトル バーの更新 Web フォームのウィンドウを反映するように表示されます。  
  
-   通常  **ソリューション エクスプローラー**  であると現在のコンテキストを反映したタイトル バーの更新および状況依存の ENT2ENT \[入力\] メニューのコマンドはアクティブな Web アプリケーション プロジェクトに適用するとアクティブなウィンドウ階層の。
