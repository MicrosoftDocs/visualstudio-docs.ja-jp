# <a name="web-site-support-templates"></a>Web サイト サポートのテンプレート
[!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)]Web サイト プロジェクトおよび項目テンプレートは、再利用可能でカスタマイズ可能な Web サイト プロジェクトとアイテム スタブを提供します。 テンプレートの[!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)]詳細については、「[プロジェクト テンプレートと項目テンプレートの作成](../../ide/creating-project-and-item-templates.md)」を参照してください。

## <a name="project-template-folder"></a>プロジェクト テンプレート フォルダ
 Web プロジェクト テンプレートは通常 *、[Visual Studio インストール パス*] の [\Common7\IDE\ProjectTemplates\Web]\\にインストールされます。

## <a name="project-file"></a>プロジェクト ファイル
 [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)]統合開発環境 (IDE) では、テンプレートを正しいプロジェクトの種類にマップする方法として、プロジェクト ファイル拡張子が必要です。 Web プロジェクトにはプロジェクト ファイルがないため、ダミー プロジェクト ファイル拡張子 .webproj が登録され、テンプレートがプロジェクトの種類にマップされます。

 必要に応じて、テンプレートに基づく項目の [**新しい項目の追加**] ダイアログ ボックスで、Web プロジェクト システムが言語の既定値を設定できるように、言語名文字列をテンプレートに追加できます。 文字列はファイルの最初の行でなければなりません。 これは、IntelliSense エンジン登録の AddItemLanguageName に登録されている名前と、プロジェクト サブタイプ (VsTemplate) に登録されている名前の両方と一致する必要があります。 詳細については、「 [Web サイトのサポート属性](../../extensibility/internals/web-site-support-attributes.md)」を参照してください。

 文字列が存在しない場合、Web プロジェクト システムは、プロジェクト テンプレートによって Web プロジェクトに追加されたページの Language 属性およびファイル拡張子に基づいて、既定の言語を決定しようとします。

## <a name="project-templates"></a>プロジェクト テンプレート
 Web サイト プロジェクト テンプレートは、[**ファイル]** メニューの [新しい Web サイト] コマンドに応答して**新しい Web サイト**を作成するために使用されます。 現在、3 種類の Web サイト プロジェクトがサポートされています。

- 空の Web サイト プロジェクト

- Web サイト プロジェクト

- Web サービス プロジェクト

### <a name="empty-web-site-projects"></a>空の Web サイト プロジェクト
 これらのファイルは、[空の Web**サイト]** コマンドに応答して空の Web サイトを**File** > **作成**します。

- 空の Web.vs テンプレート

     新しい空の Web サイトの作成をガイドするテンプレート ファイル。

- ウェブプロジ

     このファイルは、プロジェクト テンプレート システムの成果物です。 これは、EmptyWeb.vstemplate ファイル内のプロジェクト ファイル参照を満たす。

### <a name="web-site-projects"></a>Web サイト プロジェクト
 これらのファイルは、[ASP.NET Web**サイト]** コマンドに応答して > **新しい Web サイト**を**作成**します。

- Default.aspx

     新しい Web サイトの既定のホーム ページ。 言語属性は分離コード言語を指定し、CodeFile 属性は、このページに関連付けられている分離コードを含む依存ファイルを指定します。

- Default.aspx。*拡張*

     既定のホーム ページの分離コードを含む依存ファイル。 分離コード言語によって、このファイルの*拡張子*が決まります。

- web.config

     ルート Web サイト構成ファイル。

- ウェブアプリケーション.vsテンプレート

     Web サイト ソリューションの内容を決定し、App_Data フォルダを強制的に作成するテンプレート ファイル。

- ウェブアプリケーション.ウェブプロジ

     このファイルは、プロジェクト テンプレート システムの成果物です。 これは、WebApplication.vstemplate ファイル内のプロジェクト ファイル参照を満たす。

### <a name="web-service-projects"></a>Web サービス プロジェクト
 これらのファイルは、[Web サービスの**ASP.NET]** コマンドに応答して > **新しい Web サイト**を**作成**します。

- サービス.asmx

     新しい Web サービスの HTML ページ。 言語属性は分離コード言語を指定し、CodeBehind 属性は、このサービスに関連付けられている分離コードを含む依存ファイルを指定します。

- サービス。 *拡張機能*

     サービス クラスを実装する依存ファイル。 分離コード言語によって、このファイルの*拡張子*が決まります。

- web.config

- ルート Web サイト構成ファイル。

- ウェブサービス.vsテンプレート

     Web サイト ソリューションの内容を決定し、App_DataフォルダとApp_Code フォルダを強制的に作成するテンプレート ファイル。 サービス。*拡張子*ファイルがApp_Codeフォルダにコピーされます。

- ウェブサービス.ウェブプロジ

     このファイルは、プロジェクト テンプレート システムの成果物です。 これは、WebService.vstemplate ファイル内のプロジェクト ファイル参照を満たす。

## <a name="project-item-template-folder"></a>プロジェクト項目テンプレート フォルダ
 Web プロジェクト項目テンプレートは、通常、 [*Visual Studio インストール パス*] にインストールされます\\。

## <a name="project-item-templates"></a>プロジェクト項目テンプレート
 Web サイト プロジェクト項目テンプレートは、[**既存項目の追加]** コマンドに応答して、新しい Web ページを Web サイトに追加するために使用されます。 現在、次の種類の Web ページがサポートされています。

- 新しいクラス

- 新しい HTML ページ

- 新しい Web フォーム

- 新しいマスター ページ

### <a name="new-class"></a>新しいクラス
 このテンプレートは、[新しいクラスの追加] コマンドに応答して空のクラスを定義する**新しいソース**ファイルを作成します。

- クラス。 *拡張機能*

     空のクラスを実装するソース ファイル。 分離コード言語によって、このファイルの*拡張子*が決まります。

- クラス.vsテンプレート

     ソース ファイルを作成し、その内容を決定するテンプレート ファイル。

### <a name="new-html-page"></a>新しい HTML ページ
 このテンプレートは、[新しい HTML ページの**追加]** コマンドに応答して新しい Web ページを作成します。

- HTML ページ.htm

     Web ページの開始コンテンツ。 この Web ページには、通常、関連付けられた分離コード依存ファイルはありません。 関連付けられた分離コード ファイルを含むスマート ページを作成するには、代わりに Web フォーム テンプレートを使用します。

- テンプレート

     Web ページを作成し、その内容を決定するテンプレート ファイル。

### <a name="new-webform"></a>新しい Web フォーム
 このテンプレートは、[新しい Web フォームの追加] コマンドに応答して **、新しいスマート Web**ページを作成します。

 依存する分離コード ソース ファイルを作成するには、[**コードを別のファイルに配置**する] チェック ボックスをオンにします。 それ以外の場合は、1 つの Web ページが作成され、\<そのブロックに空のスクリプト ブロックが存在し、% Page %> ディレクティブを持たない場合は、依存ファイルをフックします。

 選択したマスタ ページのコンテンツ ページを作成するには、[**マスタ ページの選択**] を選択します。

- ウェブフォーム.aspx

     Web ページの開始コンテンツ。 この Web ページには、関連付けられた分離コード依存ファイルがありません。

- WebForm_cb.aspx

     Web ページの開始コンテンツ。 この Web ページには、分離コード依存ファイルが関連付けられます。

- Codebehind。 *拡張機能*

     Web フォーム クラスを実装する依存ファイル。 分離コード言語によって、このファイルの*拡張子*が決まります。

- コンテンツページ.aspx

     コンテンツ ページとしての Web ページの開始コンテンツ。 この Web ページには、関連付けられた分離コード依存ファイルがありません。

- ContentPage_cb.aspx

     コンテンツ ページとしての Web ページの開始コンテンツ。 この Web ページには、分離コード依存ファイルが関連付けられます。

- ウェブフォーム.対テンプレート

     新しい Web ページとその依存ファイルの内容を決定するテンプレート ファイル(存在する場合)。

### <a name="new-master-page"></a>新しいマスタ ページ
 このテンプレートは、[新しいマスタ ページの追加] コマンドに応答して **、新しいマスター ページを**作成します。

 依存する分離コード ソース ファイルを作成するには、[**コードを別のファイルに配置**する] チェック ボックスをオンにします。 それ以外の場合は、1 つの Web ページが作成され\<、このページには、空のスクリプト ブロックが存在し、% Page %> ディレクティブを持たない場合は、依存ファイルをフックします。

- マスターページ.マスター

     マスター ページの開始コンテンツ。 このマスター ページには、関連付けられた分離コード依存ファイルはありません。

- MasterPage_cbマスター

     マスター ページの開始コンテンツ。 このマスター ページには、分離コード依存ファイルが関連付けられます。

- Codebehind。*拡張*

     マスター ページ クラスを実装する依存ファイル。 分離コード言語によって、このファイルの*拡張子*が決まります。

- マスターページ.vsテンプレート

     新しいマスター ページとその依存ファイルの内容を決定するテンプレート ファイル(存在する場合)。
