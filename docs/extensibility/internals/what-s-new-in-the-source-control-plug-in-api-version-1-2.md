# ソース管理プラグイン API バージョン 1.2 の新機能します。
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

ここではソース管理プラグイン API Version 1.2 の高度なソース・コード管理機能が強調表示されます。  次はバージョン 1.2 の新機能の一覧です :  
  
## 変更  
 [削除する ~ SAK ファイル](../../extensibility/internals/elimination-of-tilde-sak-files.md)  
 ソース管理プラグイン API 1.2 が一時ファイルがどのように相互作用するかを示します。  
  
 [複数プロジェクトの接続設定の適用](../../extensibility/internals/application-of-settings-across-multiple-project-connections.md)  
 複数のプロジェクトまたは複数のコネクション コンテキストにわたって同じソース管理操作を実行するためにソース管理プラグインをバッチ処理を使用する方法を示しています。  
  
 [ソリューションの親コンテナーのフォルダーを作成します。](../../extensibility/internals/creating-parent-container-folders-for-solutions.md)  
 ソリューション内のすべての Web プロジェクトで一つのルートのソース管理の対象がソース管理プラグイン API のバージョン 1.2 はユーザーがどのようにする方法について説明します。  
  
 [ソース管理ストアをローカルのプロジェクト フォルダーのオプションの比較](../../extensibility/internals/optional-comparison-of-local-project-folder-to-source-control-store.md)  
 ソース管理プラグイン API が 1.2 のファイルのバージョンを比較する方法を説明します。  
  
 [ソース コントロールからの情報の削除。Proj とします。Sln ファイル](../../extensibility/internals/removal-of-source-control-information-from-dot-proj-and-dot-sln-files.md)  
 ソース管理プラグイン API Version 1.2 のプロジェクトとソリューションに関する情報を保存する方法について説明します。
