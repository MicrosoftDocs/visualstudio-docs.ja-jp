# <a name="test-area-1-add-toopen-from-source-control"></a>テスト領域 1: ソース管理から追加/開く
このソース管理プラグインのテスト領域では、ソリューションまたはプロジェクトをソース管理下に配置し、ソース管理から取得します。

## <a name="command-menu-access"></a>コマンド メニューへのアクセス
 テスト[!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)]ケースでは、次の統合開発環境メニュー パスが使用されます。

- の[!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)]場合は、ソース管理から開く:**ファイル**、**開く**、**プロジェクト**/**ソリューション**、場所を[!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)]見てください。

- その他のソース管理プラグインの場合は、ソース管理から開きます:**ファイル**、**ソース管理**、**ソース管理から開く**。

- ソース管理への追加:**ファイル**、**ソース管理**、**ソース管理ファイルへのソリューションの追加**、**ソース管理**、**選択したプロジェクトのソース管理への追加**。

- ショートカット メニュー (プロジェクト/ソリューション)、**ソース管理へのソリューションの追加**。

- ソース管理から追加:**ファイル**、**ソース管理**、**ソース管理からのプロジェクトの追加**。

- の[!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)]場合は、ソース管理から追加する**ファイル**、**追加**、**既存のプロジェクト**からも使用できます。場所を[!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)]見てください。

  > [!NOTE]
  > このテストでは、ローカル ファイルまたはローカル IIS (Web サーバー) のパスを使用できます。

## <a name="expected-behavior"></a>想定されている動作

- サポートされている各プロジェクトの種類について、ユーザーはソース管理に "追加" および "開く" ソース管理を実行できる必要があります。

- プロジェクトをソース管理に追加すると、対応する\< *ProjectName*>.vspscc ファイル (プロジェクト ヒント ファイル) が作成されます。 除外ファイルの一覧と接続情報が含まれています。 このファイルにはプロジェクト固有の情報が含まれているため、このファイルは削除しないでください。

- ソリューションをソース管理に追加すると、対応する\< *SolutionName*>.vssscc (トリプル S) ファイルが作成されます。 テキスト ファイルには、プロジェクトヒント ファイルに類似した接続情報と除外ファイルリストが含まれています。 このファイルは一時的なファイルであり、ソース管理データベースにのみ存在します。

- ソリューションをソース管理から開くと、\<ソース管理データベースにのみ存在する*SolutionName*>.vsscc (ダブル S) ファイルが一時ファイルにローカルに作成されます。 このファイルには、ソリューション接続フォルダーからソリューション ファイルへのパスが含まれています。 このファイルは一時的なファイルであり、"ソース管理から開く" 操作が完了するとローカル コピーが削除されます。

- プロジェクトをソース管理に追加した後、そのプロジェクトに対して任意のソース管理アクション ([チェックアウト]、[取得]など) を実行できます。

## <a name="test-cases"></a>テスト ケース
 次に示す特定のテスト ケースは、[ソース管理から追加/ソース管理から開く] テスト領域です。

### <a name="case-1b-open-solution-from-source-control"></a>ケース 1b. ソース管理からソリューションを開く
 このテスト ケースでは、ソース管理からソリューションを開くことに重点を置いています。

|アクション|テストステップ|検証する期待される結果|
|------------|----------------|--------------------------------|
|ソース管理からクライアント プロジェクトを含むソリューションを開く|1. クライアント プロジェクトを作成します。<br />2. ソリューションをソース管理に追加します。<br />3. ソリューションを閉じます。<br />4. ソース管理から新しい場所にソリューションを開きます。|ソース管理からソリューション/プロジェクトを開いた。|
|ソース管理からローカルまたは IIS Web プロジェクトを含むソリューションを開く|1. ローカルまたは IIS Web プロジェクトを作成します。<br />2. ソリューションをソース管理に追加します。<br />3. ソリューションを閉じます。<br />4. ソース管理から新しい場所にソリューションを開きます。|ソース管理からソリューション/プロジェクトを開いた。|
|ソース管理からリモート サイト Web プロジェクトを含むソリューションを開く|1. リモート サイト Web プロジェクトを作成します。<br />2. ソリューションをソース管理に追加します。 このステップが成功した場合は、次のステップに進みます。<br />3. ソリューションを閉じます。<br />4. ソース管理から新しい場所にソリューションを開きます。|`Result from Step 2:`<br /><br /> リモート サイト Web がソース管理下にありません。<br /><br /> `Result from Step 4:`<br /><br /> ソース管理からソリューションを開いた。<br /><br /> リモート サイト プロジェクトが読み込まれますが、ソース管理下にありません。|

### <a name="case-1c-add-solution-from-source-control"></a>ケース 1c: ソース管理からソリューションを追加する

|アクション|テストステップ|検証する期待される結果|
|------------|----------------|--------------------------------|
|空のソリューションに追加 — 単一のプロジェクト ソリューション|1. 単一のプロジェクト ソリューションを作成します。<br />2. ソリューションをソース管理に追加します。<br />3. ソリューションを閉じます。<br />4. 2 つ目の空のソリューションを作成します。<br />5. ソース管理 (**ファイル**、**ソース管理**、 ソース管理からプロジェクトを追加 ) から、以前に管理されたソリューション**を追加**します。|追加したプロジェクトが**ソリューション エクスプローラ**に表示され、チェックインされます。|
