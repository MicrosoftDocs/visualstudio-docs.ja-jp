# [ページ ファイル] タブ ([プロセス プロパティ] ダイアログ ボックス)
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

\[ページ ファイル\] タブを使用すると、プロセスのページング ファイルを調べることができます。  [&#91;プロセス プロパティ&#93; ダイアログ ボックス](../debugger/process-properties-dialog-box.md)を表示するには、[プロセス ビュー](../debugger/processes-view.md) ウィンドウにフォーカスを移動します。  ツリーで任意のプロセス ノードを選択し、\[表示\] メニューの \[プロパティ\] をクリックします。  
  
 \[ページ ファイル\] タブには、次の項目があります。  
  
|エントリ|Description|  
|----------|-----------------|  
|**\[ページ ファイル サイズ\]**|このプロセスが使用している、ページング ファイル中のページの現在の数です。  ページング ファイルには、他のファイルに含まれるデータではなく、プロセスによって使用されるデータのページが格納されます。  ページング ファイルはすべてのプロセスによって使用されます。ページング ファイルの領域が不足すると、他のプロセスの実行中にエラーが発生する可能性があります。|  
|**\[ピーク ページ ファイルのサイズ\]**|このプロセスが使用した、ページング ファイル中のページの最大数です。|  
|**\[ページ違反の回数\]**|このプロセスで実行されているスレッドによるページ フォールトの回数です。  ページ フォールトは、スレッドがメイン メモリの作業セット内にない仮想メモリ ページを参照したときに発生します。  したがって、ページがスタンバイ リストに存在し、既にメイン メモリ内にある場合や、ページを共有している別のプロセスによって使用されている場合、ページはディスクから取得されません。|
