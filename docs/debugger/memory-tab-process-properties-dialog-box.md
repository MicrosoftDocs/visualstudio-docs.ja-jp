# <a name="memory-tab-process-properties-dialog-box"></a>[メモリ] タブ ([プロセス プロパティ] ダイアログ ボックス)
**[メモリ]** タブは、プロセスによってメモリがどのように使用されているかを表示するために使用します。 [[プロセス プロパティ] ダイアログ ボックス](../debugger/process-properties-dialog-box.md)を表示するには、[プロセス ビュー](../debugger/processes-view.md) ウィンドウにフォーカスを移動します。 ツリーで任意のプロセス ノードを選択し、 **[ビュー]** メニューから **[プロパティ]** を選択します。

 **[メモリ]** タブでは、次の設定を使用できます。

|入力|説明|
|-----------|-----------------|
|**仮想バイト**|プロセスで使用している仮想アドレス空間の現在のサイズ (バイト単位)。 仮想アドレス空間を使用していても、ディスク ページまたはメイン メモリ ページを使用しているとは限りません。 ただし、仮想領域には限界があり、使用しすぎると、ライブラリを読み込むプロセスの機能は制限されます。|
|**ピーク時の仮想空間サイズ**|プロセスが任意の時点までの間に一度に使用した仮想アドレス空間の最大バイト数。|
|**ワーキング セット**|プロセス内のスレッドが最近使用した一連のメモリ ページ。 コンピューターのメモリの空き容量がしきい値以上ある場合は、使用中でないページもプロセスのワーキング セットに残されます。 メモリの空き容量がしきい値を下回った場合は、ワーキング セットからページがトリミングされます。 必要な場合は、メイン メモリから出る前にワーキング セットに戻されます。|
|**ピーク時の作業セット**|このプロセスのワーキング セットの任意の時点での最大ページ数。|
|**ページ プール サイズ**|プロセスが割り当てたページ プールの現在の量。 ページ プールは、オペレーティング システム コンポーネントが指定されたタスクを実行する際に領域を取得するシステム メモリ領域です。 ページ プールのページは、システムによって一定期間アクセスされていないときにページング ファイルにページ アウトすることができます。|
|**非ページ プール サイズ**|プロセスによって割り当てられた非ページ プールの現在のバイト数。 非ページ プールは、オペレーティング システム コンポーネントが指定されたタスクを実行する際に領域を取得するシステム メモリ領域です。 非ページ プールのページをページング ファイルにページ アウトすることはできません。それらのページは、割り当てられている限りメイン メモリに残ります。|
|**プライベート バイト**|このプロセスが割り当てた、他のプロセスとは共有できない現在のバイト数。|
|**空き容量**|このプロセスの未使用の仮想アドレス空間の合計。|
|**予約容量**|このプロセスで今後使用するために予約されている仮想メモリの合計サイズ。|
|**空きイメージ容量**|このプロセス内のイメージによって使用されていない、または予約されていない仮想アドレス空間の量。|
|**予約イメージ容量**|このプロセス内で実行されるイメージによって予約されている、すべての仮想メモリの合計。|
