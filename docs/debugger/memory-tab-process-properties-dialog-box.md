# <a name="memory-tab-process-properties-dialog-box"></a>[メモリ] タブ ([プロセス プロパティ] ダイアログ ボックス)
使用して、**メモリ**プロセスによるメモリの使用方法を表示するタブです。 表示する、[プロセス プロパティ ダイアログ ボックス](../debugger/process-properties-dialog-box.md)にフォーカスを移動する、[プロセス ビュー](../debugger/processes-view.md)ウィンドウです。 ツリーで、プロセスの任意のノードを選択し、**プロパティ**から、**ビュー**メニュー。  
  
 次の設定が [利用可能な**メモリ**] タブ。  
  
|入力|説明|  
|-----------|-----------------|  
|**仮想バイト数**|現在のサイズ (バイト単位) の仮想アドレス領域のプロセスが使用されています。 仮想アドレス領域の使用を使用するディスクまたはメイン メモリ ページのいずれかを必ずしも意味しません。 ただし、仮想空間には限界が、ライブラリを読み込むプロセスの機能を使用する多すぎる制限可能性があります。|  
|**ピーク時の仮想バイト数**|仮想アドレス空間、プロセスのバイトの最大数は任意の時点で使用します。|  
|**ワーキング セット**|プロセス内のスレッドが最近使用したメモリ ページのセット。 コンピューターの空きメモリがしきい値を超えた場合は、ページは、使用されていない場合でも、プロセスのワーキング セットに残されます。 空きメモリ容量がしきい値を下回った場合は、ワーキング セットからページがトリミングされます。 場合必要であれば、されますワーキング セットに戻るメイン メモリから出る前にします。|  
|**ピーク ワーキング セット**|任意の時点では、このプロセスのワーキング セット内のページの最大数。|  
|**Pool paged Bytes**|現在のページ プール、処理の量が割り当てられます。 ページ プールは、指定されたタスクを実行するときに、オペレーティング システムのコンポーネントが領域を取得する、システム メモリ領域です。 ページ プールのページは、継続期間のシステムによってアクセスされない場合は、ページング ファイルにページ アウトできます。|  
|**非ページ プールのバイト数**|現在のプロセスによって割り当てられている非ページ プール内のバイト数。 非ページ プールは、取得されたオペレーティング システム コンポーネントによって指定されたタスクを実行するとき、システム メモリ領域です。 非ページ プールのページをページング ファイルにページ アウトできません。割り当てられている限り、メイン メモリに残ります。|  
|**Private Bytes**|現在のこのプロセスが割り当てたバイト数は、他のプロセスと共有できません。|  
|**空きバイト数**|このプロセスの合計使用されていない仮想アドレス空間です。|  
|**予約容量**|このプロセスで将来使用するために予約する仮想メモリの総量。|  
|**イメージの空きバイト数**|使用されていないか、このプロセス内のイメージで予約されている仮想アドレス空間の量。|  
|**予約済みのイメージのバイト数**|イメージによって予約されているすべての仮想メモリの合計は、このプロセス内で実行します。|
