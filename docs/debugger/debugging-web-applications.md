# <a name="debugging-web-applications"></a>Web アプリケーションのデバッグ

この記事では、さまざまな種類の web アプリケーションのデバッグに役立つリンクを示します。

- ASP.NET アプリケーションをデバッグする

  [!INCLUDE[vstecasp](../code-quality/includes/vstecasp_md.md)] アプリケーションのデバッグを有効にするために必要な手順については、「 [ASP.NET applications のデバッグ](how-to-enable-debugging-for-aspnet-applications.md)」を参照してください。

- ASP.NET でのクライアント側スクリプトのデバッグ

  必要な手順については、ブログ記事「 [Microsoft Edge での JavaScript のデバッグ](https://devblogs.microsoft.com/visualstudio/debug-javascript-in-microsoft-edge-from-visual-studio/)」および[Google Chrome のこの投稿](https://devblogs.microsoft.com/aspnet/client-side-debugging-of-asp-net-projects-in-google-chrome)を参照してください。

- JavaScript と TypeScript のデバッグ

  「 [JavaScript と TypeScript のデバッグ」を](../javascript/debug-nodejs.md)参照してください。

- スクリプト ドキュメントを表示する

  **ソリューション エクスプローラー**では、閲覧できるサーバー側およびクライアント側のスクリプト ドキュメントのリストを表示できます。 **ソリューション エクスプローラー**で、任意のスクリプト ドキュメントを開くことができます。 詳細については、「 [How to: View Script Documents](../debugger/how-to-view-script-documents.md)」を参照してください。

- AJAX スクリプト アプリケーションのデバッグ

  AJAX 対応の Web アプリケーションではスクリプト コードを頻繁に使用するため、デバッグは困難になる可能性があります。 AJAX のデバッグ手法の詳細については、「 [Ajax アプリケーションのデバッグとトレースの概要](https://msdn.microsoft.com/Library/92684ea0-7bb4-4a34-9203-3aa6394ce375)」を参照してください。
