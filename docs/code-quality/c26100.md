# <a name="c26100"></a>C26100
警告 C26100:競合状態。 変数 \<var > は lock \< lock > によって保護されている必要があります。

 コード内の @no__t 0 注釈は、共有変数を保護するために使用するロックを指定します。 警告 C26100 は、ガードコントラクトに違反したときに生成されます。

## <a name="example"></a>例
 次の例では、`_Guarded_by_` コントラクトの違反があるため、警告 C26100 が生成されます。

```
CRITICAL_SECTION gCS;

_Guarded_by_(gCS) int gData;

typedef struct _DATA {
   _Guarded_by_(cs) int data;
   CRITICAL_SECTION cs;
} DATA;

void Safe(DATA* p) {
   EnterCriticalSection(&p->cs);
   p->data = 1; // OK
   LeaveCriticalSection(&p->cs);
   EnterCriticalSection(&gCS);
   gData = 1; // OK
   LeaveCriticalSection(&gCS);
}

void Unsafe(DATA* p) {
   EnterCriticalSection(&p->cs);
   gData = 1; // Warning C26100 (wrong lock)
   LeaveCriticalSection(&p->cs);
}
```

 関数 `Unsafe` に正しくないロックが使用されているため、コントラクト違反が発生します。 この場合、`gCS` が適切なロックとして使用されます。

## <a name="example"></a>例
 共有変数は書き込みアクセスに対してのみ保護する必要がありますが、読み取りアクセスには保護されません。 その場合は、次の例に示すように、@no__t 0 の注釈を使用します。

```

CRITICAL_SECTION gCS;

_Guarded_by_(gCS) int gData;

typedef struct _DATA2 {
   _Write_guarded_by_(cs) int data;
   CRITICAL_SECTION cs;
} DATA2;

int Safe2(DATA2* p) {
   // OK: read does not have to be guarded
   int result = p->data;
   return result;
}

void Unsafe2(DATA2* p) {
   EnterCriticalSection(&gCS);
   // Warning C26100 (write has to be guarded by p->cs)
   p->data = 1;
   LeaveCriticalSection(&gCS);
}
```

 また、この例では、関数 `Unsafe2` の不適切なロックが使用されているため、警告が生成されます。
