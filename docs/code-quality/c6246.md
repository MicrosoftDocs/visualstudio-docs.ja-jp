# <a name="c6246"></a>C6246
警告 C6246:@No__t 0variable > のローカル宣言では、外側のスコープ内の同じ名前の宣言が非表示になります。 追加情報:@No__t 0location > の前の宣言を参照してください。

 この警告は、2つの宣言がローカルスコープで同じ名前を持っていることを示します。 外側のスコープの名前は、内部スコープの宣言によって非表示になります。 外側のスコープ宣言を使用する場合は、ローカル宣言が使用されます。

## <a name="example"></a>例
 この警告が発生するコード例を次に示します。

```
#include <stdlib.h>
#define UPPER_LIMIT 256
int DoSomething( );

int f( )
{
  int i = DoSomething( );
  if (i > UPPER_LIMIT)
  {
    int i;
    i = rand( );
  }
  return i;
}
```

 この警告を修正するには、次のコードに示すように、別の変数名を使用します。

```
#include <stdlib.h>
#define UPPER_LIMIT 256
int DoSomething( );

int f ( )
{
  int i = DoSomething( );
  if (i > UPPER_LIMIT)
  {
    int j = rand( );
    return j;
  }
  else
  {
    return i;
  }
}
```

 この警告は、スコープの重複を識別するだけです。
