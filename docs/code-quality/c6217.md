# <a name="c6217"></a>C6217

警告 C6217:意味の異なる整数型の間の暗黙的なキャストです: ' not ' で HRESULT をテストしています。 代わりに、 [SUCCEEDED](/windows/desktop/api/winerror/nf-winerror-succeeded)または[FAILED](/windows/desktop/api/winerror/nf-winerror-failed)マクロを使用することを検討してください。

この警告は、not (@no__t) 演算子を使用して @no__t 0 がテストされていることを示します。 @No__t-1 の成功 (`S_OK`) は、値0で示されます。 ただし、ブール型の場合、0 はエラーを表します。 Not 演算子 (`!`) を指定して `HRESULT` をテストすると、実行するコードブロックが間違ったコードパスの後に発生する可能性があります。 これにより、望ましくない結果が生じる可能性があります。

## <a name="example"></a>例

次のコードでは、not 演算子を使用して @no__t 0 値の成功または失敗を判断するため、この警告が生成されます。 この場合、`( !hr )` はエラーコードを実行するため、間違ったコードパスが実行されます。

```cpp
#include <windows.h>
#include <objbase.h>

void f( )
{
  HRESULT hr = CoInitialize(NULL);
  if (!hr)
  {
    // failure code ...
  }
  else
  {
    // success code ...
  }
}
```

この警告を修正するには、次のコードで `FAILED` マクロを使用してエラーを探します。

```cpp
#include <windows.h>
#include <objbase.h>

void f( )
{
  HRESULT hr = CoInitialize(NULL);
  if (FAILED(hr))
  {
    // failure code ...
  }
  else
  {
    // success code ...
  }
}
```

この警告では、`SCODE` 型は `HRESULT` と等価です。

ブール値としてテストされている場合の HRESULT (`S_OK`) の通常の成功値は `false` です。

@No__t-0 が成功したかどうかを確認するには、代わりに `SUCCEEDED` マクロを使用します。
