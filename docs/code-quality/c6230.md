# <a name="c6230"></a>C6230

警告 C6230: 意味の異なる整数型の間の暗黙的なキャストです: HRESULT をブール値のコンテキストで使用しています。

この警告は、`if` ステートメントなど、ブール型の結果が想定されるコンテキストで、ベア `HRESULT` が使用されていることを示します。 このテストでは、正しくない結果が生成される可能性があります。 たとえば、ブール値としてテストされている場合、`HRESULT` (`S_OK`) の通常の成功値は false になります。

## <a name="example"></a>例

この警告が発生するコード例を次に示します。

```cpp
#include <windows.h>

VOID f( )
{
  LPMALLOC pMalloc;
  HRESULT hr = CoGetMalloc(1, &pMalloc);

  if (hr)
  {

    // code ...
  }
  else
  {
    // code ...
  }
}
```

ほとんどの場合、`HRESULT` の値をテストするには、 [SUCCEEDED](/windows/desktop/api/winerror/nf-winerror-succeeded)または[FAILED](/windows/desktop/api/winerror/nf-winerror-failed)マクロを使用する必要があります。 この警告を解決するには、次のコードを使用します。

```cpp
#include <windows.h>

VOID f( )
{
  LPMALLOC pMalloc;
  HRESULT hr = CoGetMalloc(1, &pMalloc);

  if (SUCCEEDED(hr))
  {

    // code ...
  }
  else
  {
    // code ...
  }
}
```

この警告では、@no__t 0 の型は `HRESULT` として扱われます。

@No__t-0 および `free` (および関連する動的メモリ Api) の使用には、メモリリークと例外の原因として多くの落とし穴があります。 この種のリークや例外の問題を回避するには、 C++標準ライブラリに用意されているポインタークラスとコンテナークラスを使用します。 これには、 [shared_ptr](/cpp/standard-library/shared-ptr-class)、 [unique_ptr](/cpp/standard-library/unique-ptr-class)、 [vector](/cpp/standard-library/vector)が含まれます。 詳細については、「[スマートポインター](/cpp/cpp/smart-pointers-modern-cpp) 」と[ C++ 「標準ライブラリ](/cpp/standard-library/cpp-standard-library-reference)」を参照してください。
