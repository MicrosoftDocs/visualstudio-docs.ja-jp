# <a name="c6284"></a>C6284
警告 C6284: オブジェクトがパラメーター '% d ' として渡されました。 \<function > の呼び出しには文字列が必要です。

 この警告は、書式文字列によって文字列が指定されていることを示します。たとえば、`printf` または @no__t- C++ 2 の @no__t 0 の指定は、代わりにオブジェクトが渡されています。

 この欠陥により、正しくない出力やクラッシュが発生する可能性があります。

 このメッセージは、多くの場合、@no__t C++ -1、`CComBSTR`、`bstr_t` などの文字列型を実装するオブジェクトを、C @no__t の20-4 呼び出しに渡すことによって報告されます。 C++クラスの実装によっては、つまり、適切なキャスト演算子が定義されてC++いる場合、文字列オブジェクトは、C 文字列が必要になるたびに透過的に使用されることがよくあります。ただし、-2 スタイルの関数 @no__t のパラメーターは基本的に型指定されていないため、文字列への変換は行われません。

 オブジェクトによっては、適切な文字列型に @no__t 0 演算子を挿入することが適切な場合があります (たとえば、`char *` または `TCHAR``*`)。または、@no__t のインスタンスで文字列を返すメンバー関数 (たとえば、`c_str()`) を呼び出すこともできます。

## <a name="example"></a>例
 次のコードでは、`CComBSTR` が `sprintf` 関数に渡されるため、この警告が生成されます。

```
#include <atlbase.h>
#include <stdlib.h>

void f()
{
  char buff[50];
  CComBSTR bstrValue("Bye");

  sprintf(buff,"%ws",bstrValue);
}
```

 次のコードでは、静的キャストを使用してこの警告を修正しています。

```
#include <atlbase.h>
#include <stdlib.h>

void f()
{
  char buff[50];
  CComBSTR bstrValue("Bye");

  sprintf_s(buff,50,"%ws",static_cast<wchar_t *>(bstrValue));
}
```
