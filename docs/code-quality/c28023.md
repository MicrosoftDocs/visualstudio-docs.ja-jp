# <a name="c28023"></a>C28023
警告 C28023:割り当てまたは渡される関数には、次のクラスの少なくとも1つのクラスの @no__t 0 の注釈が含まれている必要があります: \< classlist >

 この警告は、通常、1つの関数クラスのみが使用されていて、適切な型のコールバックが宣言されていない場合に報告されます。

 この警告が発生するのは、代入の左側の関数 (または、関数呼び出しの場合は暗黙の代入) に注釈が付けられ、それが @no__t 0 の注釈を使用するドライバー固有の関数型であるか、またはこのようなを含む typedef であることを示しています。ゴム. 代入の右側にある関数に @no__t 0 の注釈がありません。 右側の関数は、左側の関数と同じ型であることを示す注釈が付けられている必要があります。 これは通常、\<ファンク name2 > の現在の最初の宣言の前に、@no__t-> 1> @no__t の宣言を追加することで最適です。
