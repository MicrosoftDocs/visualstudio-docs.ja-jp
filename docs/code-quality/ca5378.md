# <a name="ca5378-do-not-disable-servicepointmanagersecurityprotocols"></a>CA5378: ServicePointManagerSecurityProtocols を無効にしません

|||
|-|-|
|TypeName|% Setswitch|
|CheckId|CA5378|
|カテゴリ|Microsoft.Security|
|互換性に影響する変更点|なし|

## <a name="cause"></a>原因

メソッド<xref:System.AppContext.SetSwitch%2A?displayProperty=nameWithType>の呼び出しが`Switch.System.ServiceModel.DisableUsingServicePointManagerSecurityProtocols`に`true`設定されます。

## <a name="rule-description"></a>規則の説明

を`Switch.System.ServiceModel.DisableUsingServicePointManagerSecurityProtocols`に`true`設定すると、Windows Communication Framework の (WCF) Transport Layer Security (tls) 接続が tls 1.0 を使用するように制限されます。 このバージョンの TLS は非推奨とされます。 詳細については、 [.NET Framework を使用したトランスポート層セキュリティ (TLS) のベストプラクティス](/dotnet/framework/network-programming/tls#switchsystemservicemodeldisableusingservicepointmanagersecurityprotocols)に関する説明を参照してください。

## <a name="how-to-fix-violations"></a>違反の修正方法

- アプリケーションが .NET Framework v1.0 以降を対象としている場合は、 <xref:System.AppContext.SetSwitch%2A?displayProperty=nameWithType>メソッド呼び出しを削除するか、スイッチの値をに`false`設定できます。
- アプリケーションが .NET Framework v 4.6.2 以前のバージョンを対象としており、.NET Framework v1.0 以降で実行されている`false`場合は、スイッチの値をに設定します。
- それ以外の場合は、「 [Transport Layer Security (TLS) のベストプラクティス」で、](/dotnet/framework/network-programming/tls)軽減策については .NET Framework を参照してください。

## <a name="when-to-suppress-warnings"></a>警告を非表示にする場合

セキュリティで保護された TLS 構成を使用するためにアップグレードできないレガシサービスに接続する必要がある場合は、この警告を非表示にすることができます。
