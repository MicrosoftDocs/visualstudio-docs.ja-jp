# <a name="ca1507-use-nameof-in-place-of-string"></a>CA1507: 文字列の代わりに `nameof` を使用します。

|||
|-|-|
|TypeName|UseNameofInPlaceOfString|
|CheckId|CA1507|
|カテゴリ|Microsoft の保守容易性|
|互換性に影響する変更点|なし|

## <a name="cause"></a>原因

メソッドの引数として使用されるのは、含んでいるメソッドのパラメーターの名前またはそれを含んでいる型のプロパティの名前と一致する @no__t 0 リテラルまたは定数です。

## <a name="rule-description"></a>規則の説明

Rule CA1507 は、メソッドまたはコンストラクターの引数として `string` リテラルを使用するようにフラグを指定します。このメソッドまたはコンストラクターの場合、Visual Basic) 式は @no__t、保守容易[性を追加](/dotnet/csharp/language-reference/keywords/nameof)します。 この規則は、次のすべての条件が満たされた場合に発生します。

- 引数が @no__t 0 リテラルまたは定数です。

- @No__t-0 リテラルの定数値は、次のいずれかに一致します。

  - メソッド、ラムダ、またはメソッドまたはコンストラクターが呼び出されるローカル関数のパラメーターの名前。

  - メソッドまたはコンストラクターが呼び出されている型のプロパティの名前。

- 引数は、メソッドまたは呼び出されるコンストラクター (つまり、呼び出しサイトで変換が関係しない) の @no__t 横-0 型パラメーターに対応します。

ルール CA1507 を使用すると、将来、パラメーターの名前が変更される可能性がありますが、@no__t 0 リテラルの名前が誤って変更されない場合に、コードの保守性が向上します。 @No__t-0 を使用すると、リファクタリング操作を使用してパラメーターの名前を変更したときにシンボルの名前が変更されます。 また、パラメーターの名前のスペルミスは、コンパイラによってキャッチされます。

> [!NOTE]
> ルール CA1507 は、レガシ分析では使用できません。 これは、 [FxCop アナライザー](https://www.nuget.org/packages/Microsoft.CodeAnalysis.FxCopAnalyzers)バージョン2.6.0 で初めて導入されました。

## <a name="how-to-fix-violations"></a>違反の修正方法

違反を修正するには、@no__t 0 @no__t[リテラルを、](/dotnet/csharp/language-reference/keywords/nameof) Visual Basic) 式で置き換えます。 たとえば、次の2つのコードスニペットは、規則違反とその修正方法を示しています。

```csharp
public Book(string title)
{
    // Violates rule CA1507
    Title = title ?? throw new ArgumentNullException("title", "All books must have a title.");
}
```

```csharp
public Book(string title)
{
    // Resolves rule CA1507 violation
    Title = title ?? throw new ArgumentNullException(nameof(title), "All books must have a title.");
}
```

> [!TIP]
> このルールのコード修正プログラムは、Visual Studio で使用できます。 これを使用するには、カーソルを @no__t 0 のリテラルに置き、 **ctrl**+ キーを押し**ます。** (ピリオド) を押します。 表示されるオプションの一覧から**シンボル名を表すには、[名前を使用する**] を選択します。
>
