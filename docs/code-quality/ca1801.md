# <a name="ca1801-review-unused-parameters"></a>CA1801: 使用されていないパラメーターを再確認します

|||
|-|-|
|TypeName|ReviewUnusedParameters|
|CheckId|CA1801|
|カテゴリ|マイクロソフトの使用法|
|互換性に影響する変更点|非分割 - 変更に関係なく、メンバーがアセンブリの外部で表示されない場合。<br /><br /> 非分割 - メンバーの本体内でパラメーターを使用するように変更した場合。<br /><br /> [破断] - パラメータを削除し、アセンブリの外部に表示されている場合。|

## <a name="cause"></a>原因

メソッド シグネチャには、メソッド本体で使用されていないパラメーターが含まれています。

この規則では、次の種類のメソッドは検査されません。

- デリゲートによって参照されるメソッド。

- イベント ハンドラーとして使用されるメソッド。

- シリアル化コンストラクター ([ガイドラインを参照](/dotnet/standard/serialization/serialization-guidelines#supporting-runtime-serialization)してください)

- シリアル<xref:System.Runtime.Serialization.ISerializable.GetObjectData>化メソッド。

- 修飾子 ( `abstract` Visual`MustOverride` Basic で) を使用して宣言されたメソッド。

- 修飾子 ( `virtual` Visual`Overridable` Basic で) を使用して宣言されたメソッド。

- 修飾子 ( `override` Visual`Overrides` Basic で) を使用して宣言されたメソッド。

- (`Declare` Visual `extern` Basic のステートメント) 修飾子で宣言されたメソッド。

[FxCop アナライザ](install-fxcop-analyzers.md)ーを使用している場合、このルールは[、破棄](/dotnet/csharp/discards)シンボル`_`で名前が付けられたパラメータにフラグ`_1`を付けません。 `_2` これにより、デリゲートとして使用されるメソッド、特殊な属性を持つパラメーター、フレームワークによって実行時に暗黙的にアクセスされる値がコードで参照されていないパラメーターなど、シグネチャ要件に必要なパラメーターに対する警告ノイズが軽減されます。

## <a name="rule-description"></a>規則の説明

メソッド本体で使用されていない非仮想メソッドのパラメーターを確認して、メソッド本体へのアクセスの失敗の際に誤りがないことを確認します。 使用しないパラメータは、メンテナンスとパフォーマンスのコストを発生させます。

場合によっては、この規則違反がメソッドの実装バグを指している可能性があります。 たとえば、パラメーターはメソッド本体で使用されている必要があります。 下位互換性のためにパラメータが存在する必要がある場合は、この規則の警告を抑制します。

## <a name="how-to-fix-violations"></a>違反の修正方法

この規則違反を修正するには、未使用のパラメーター (互換性に影響する変更) を削除するか、メソッド本体でパラメーターを使用します (互換性のない変更)。

## <a name="when-to-suppress-warnings"></a>警告を抑制する場合

この規則からの警告を抑制しても安全です。

- 以前に出荷されたコードでは、修正が画期的な変更になります。

- のカスタム`this`拡張メソッドのパラメーターの場合<xref:Microsoft.VisualStudio.TestTools.UnitTesting.Assert?displayProperty=nameWithType>。 クラス内の<xref:Microsoft.VisualStudio.TestTools.UnitTesting.Assert>関数は静的であるため、メソッド本体の`this`パラメーターにアクセスする必要はありません。

## <a name="configurability"></a>構成可能の有無

このルールを[FxCop アナライザ](install-fxcop-analyzers.md)から実行している場合 (レガシ分析ではなく)、コードベースのどの部分でこのルールを実行するかは、そのアクセシビリティに基づいて設定できます。 たとえば、非パブリック API サーフェスに対してのみルールを実行するように指定するには、プロジェクトの .editorconfig ファイルに次のキーと値のペアを追加します。

```ini
dotnet_code_quality.ca1801.api_surface = private, internal
```

このオプションは、このルール、すべてのルール、またはこのカテゴリ (パフォーマンス) のすべてのルールに対して構成できます。 詳細については、 [FxCop アナライザーの設定](configure-fxcop-analyzers.md)を参照してください。
