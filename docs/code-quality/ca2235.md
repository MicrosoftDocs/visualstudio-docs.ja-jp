# <a name="ca2235-mark-all-non-serializable-fields"></a>CA2235: すべてのシリアル化不可能なフィールドを設定します

|||
|-|-|
|TypeName|MarkAllNonSerializableFields|
|CheckId|CA2235|
|カテゴリ|Microsoft. 使用方法|
|互換性に影響する変更点|なし|

## <a name="cause"></a>原因

シリアル化できない型のインスタンス フィールドが、シリアル化できる型で宣言されています。

## <a name="rule-description"></a>規則の説明

シリアル化可能な型は、<xref:System.SerializableAttribute?displayProperty=fullName> 属性でマークされた型です。 シリアル化できない型のインスタンスフィールドが型に含まれて*おり*、その型がシリアル化可能ではなく、@no__t 2 つのインターフェイスを実装していない場合、型がシリアル化されると <xref:System.Runtime.Serialization.SerializationException?displayProperty=fullName> 例外がスローされます。

> [!TIP]
> CA2235 は、独自のシリアル化ロジックを提供するため、<xref:System.Runtime.Serialization.ISerializable> を実装する型のインスタンスフィールドに対しては起動されません。

## <a name="how-to-fix-violations"></a>違反の修正方法

この規則違反を修正するには、シリアル化できないフィールドに <xref:System.NonSerializedAttribute?displayProperty=fullName> 属性を適用します。

## <a name="when-to-suppress-warnings"></a>警告を非表示にする場合

フィールドのインスタンスをシリアル化および逆シリアル化できるように @no__t 0 の型が宣言されている場合にのみ、この規則からの警告を抑制します。

## <a name="remarks"></a>Remarks

Rule CA2235 は、<xref:System.Runtime.Serialization.ISerializable> インターフェイスを実装する型を分析しません (<xref:System.SerializableAttribute> 属性でもマークされている場合を除く)。 これは、 [RULE CA2237](../code-quality/ca2237.md)は、@no__t 属性を使用して <xref:System.Runtime.Serialization.ISerializable> インターフェイスを実装するマークの種類を既に推奨しているためです。
