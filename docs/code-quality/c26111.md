# <a name="c26111"></a>C26111
警告 C26111:呼び出し元がロックを解放できませんでした \<lock > 関数を呼び出す前に \< func >。

 注釈 `_Requires_lock_not_held_` は、関数が呼び出されたときに、指定されたロックのロックカウントが0を超えることができないことを前提としています。 警告 C26111 は、関数が別の関数を呼び出す前にロックの解放に失敗したときに発行されます。

## <a name="example"></a>例
 次の例では、ロックされたセクション内の `DoNotLock` への呼び出しによって `_Requires_lock_not_held_` の前提条件に違反しているため、警告 C26111 が生成されます。

```

typedef struct _DATA
{
    CRITICAL_SECTION cs;
    int d;
} DATA;

_Requires_lock_not_held_(p->cs)

void DoNotLock(DATA* p)
{
    EnterCriticalSection(&p->cs);
    p->d = 0;
    LeaveCriticalSection(&p->cs);
}

void LockedFunction(DATA* p)
{
    EnterCriticalSection(&p->cs);
    DoNotLock(p); // Warning C26111
    LeaveCriticalSection(&p->cs);
}
```
