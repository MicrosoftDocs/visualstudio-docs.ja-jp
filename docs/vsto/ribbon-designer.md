# <a name="ribbon-designer"></a>リボン デザイナー
  リボン デザイナーは、ビジュアルなデザイン キャンバスです。 リボン デザイナーを使用すると、Microsoft Office アプリケーションのリボンにカスタム タブ、グループ、およびコントロールを追加します。

 [!INCLUDE[appliesto_ribbon](../vsto/includes/appliesto-ribbon-md.md)]

-   [リボン レイアウトをデザインします。](#DesigningRibbonLayout)

-   [イベントを処理し、コントロール プロパティの設定](#HandleEventsSetProperties)

-   [Backstage ビューにコントロールを追加します。](#CustomizingMicrosoftOfficeButton)

> [!NOTE]
>  リボン デザイナーでは行うことのできないタスクがあります。 これらのタスクとそれらを実現する方法の詳細については、次を参照してください。[リボンの概要](../vsto/ribbon-overview.md)します。

## <a name="add-a-ribbon-visual-designer-item-to-a-project"></a>リボン (ビジュアル デザイナー) 項目をプロジェクトに追加します。
 リボン デザイナーを使用するには、新しい追加**リボン (ビジュアル デザイナー)** 項目をプロジェクトにします。 詳細については、「[方法 :リボンのカスタマイズの概要](../vsto/how-to-get-started-customizing-the-ribbon.md)します。

 新しいを追加すると**リボン (ビジュアル デザイナー)** 項目は、Visual Studio に自動的に、次のファイル プロジェクトに追加します。

- リボン コード ファイル。 このファイルは、指定した名前、**リボン (ビジュアル デザイナー)** 内の項目、**新しい項目の追加** ダイアログ ボックス。 このファイルには、リボンのイベントを処理するコードを追加します。

- リボン デザイナー コード ファイル。 このファイルには、リボン デザイナーによって生成されるコードが格納されます。直接編集することはできません。

- リソース ファイル。 このファイルには、リボン上の各コントロールのプロパティ値が含まれています。

  既にある場合、**リボン (ビジュアル デザイナー)** 項目から別のプロジェクトを再利用できますが、現在のプロジェクトでを使用して、**既存項目の追加** ダイアログ ボックス。

##  <a name="DesigningRibbonLayout"></a> リボンをデザインします。
 リボン デザイナーを開くには、次の 3 つの方法があります。

- **ソリューション エクスプ ローラー**、リボン コード ファイルをダブルクリックします。

- **ソリューション エクスプ ローラー**リボン コード ファイルを右クリックし、クリックして**ビュー デザイナー**します。

- **ソリューション エクスプ ローラー**リボン コード ファイルを選択し、クリックして**デザイナー**上、**ビュー**メニュー。

  リボン デザイナーには、既定のタブとグループがあります。 既定のタブとグループは、リボン デザイナーから削除することができます。 右クリックし、既定のグループを削除する**Group1**、 をクリックし、**削除**します。 既定のタブを削除する、デザイン サーフェイスの空の領域を右クリックし、**リボン タブの削除**します。

  リボン デザイナーにカスタム タブ、グループ、およびコントロールを追加することもできます。 これらのコントロールを見つけることができます、**ツールボックス**の**Office リボン コントロール**グループ。 コントロールを追加する 3 つの方法、 **Office リボン コントロール**リボン デザイナーにグループ化します。

- リボン デザイナーの適切な領域にコントロールをドラッグします。

- コントロールをクリックし、リボン デザイナーの適切な領域をクリックします。

- デザイナーで、適切な領域を選択し、内のコントロールをダブルクリック、**ツールボックス**します。

### <a name="ribbon-design-workflow"></a>リボン デザインのワークフロー
 リボンのレイアウトをデザインするときの基本的な手順は次のとおりです。

1. [リボンにカスタム タブを追加](#AddTabToRibbon)します。

2. [タブにグループを追加](#AddGroupsToTab)します。

3. [コントロールをグループに追加](#AddControlsToGroups)します。

   コントロールのグループでのみ削除できます。直接タブまたはリボン コントロールをドラッグすることはできません。 グループをタブでのみ削除できます。リボンに直接グループをドラッグすることはできません。

   コントロールを正確な位置にドラッグして、配置します。 使用して、コントロールのプロパティを設定することができます、**プロパティ**ウィンドウ。

   リボンのタブ間でコントロールをドラッグできません。 使用する必要があります、コントロールを別のタブに移動する場合、**切り取り**コマンドを 1 つのタブからコントロールを削除し、別のタブ コントロールを貼り付けます。コントロールを切り取って貼り付けると、イベント ハンドラーは動作を停止します。 内のイベント ハンドラーを再接続することができます、**プロパティ**ウィンドウ。 詳細については、次を参照してください。[プロパティ ウィンドウ](../ide/reference/properties-window.md)します。

###  <a name="AddTabToRibbon"></a> カスタム タブをリボンに追加します。
 リボンにカスタム タブを追加する 3 つの方法はあります。

- タブを追加、**ツールボックス**します。

- リボン デザイナーを右クリックし、をクリックし、**リボン タブの追加**します。

- 開く、**タブ コレクション エディター**、 をクリックし、**追加**します。

   開くには、**タブ コレクション エディター**の**プロパティ**ウィンドウで、**タブ**プロパティ、省略記号ボタンをクリックして![ASP.NET モバイルデザイナー楕円](../sharepoint/media/mwellipsis.gif "ASP.NET モバイル デザイナー楕円")します。

  タブを追加すると、グループを追加してコントロールを含めることができます。

#### <a name="remove-custom-tabs-from-the-ribbon"></a>リボンからカスタム タブを削除します。
 リボンからカスタム タブを削除する 3 つの方法はあります。

-   デザイナーを右クリックし、**リボン タブの削除**します。

-   **コマンド**のウィンドウ、**プロパティ**ウィンドウで、をクリックして**リボン タブの削除**します。

-   開く、**タブ コレクション エディター**、タブを選択し、クリックして**削除**します。

#### <a name="change-the-position-of-a-tab-on-the-ribbon"></a>リボンのタブの位置を変更します。
 リボンにカスタム タブの順序を変更することができます。 リボン上の組み込みタブの前後にカスタム タブを配置することもできます。 詳細については、「[方法 :リボンのタブの位置を変更](../vsto/how-to-change-the-position-of-a-tab-on-the-ribbon.md)します。

#### <a name="customize-built-in-tabs-on-the-ribbon"></a>組み込みタブのリボンのカスタマイズします。
 組み込みタブは、Microsoft Office アプリケーションのリボンに登録されているタブです。 たとえば、**データ**タブは、Excel の組み込みタブ。

 組み込みタブには、グループやコントロールを追加できます。既定では、カスタム グループは組み込みタブの最後のグループとして表示されます。ただし、タブ上で組み込みグループの前または後ろにカスタム グループを配置することができます。

 組み込みグループを削除することはできません。

 組み込みタブをカスタマイズする方法の詳細については、次を参照してください。[方法。組み込みタブをカスタマイズする](../vsto/how-to-customize-a-built-in-tab.md)します。
