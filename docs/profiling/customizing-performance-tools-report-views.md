# プロファイリング ツールのレポート ビューのカスタマイズ
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

このセクションでは、[!INCLUDE[vsprvs](../code-quality/includes/vsprvs_md.md)] プロファイリング ツールで生成されるレポートのカスタマイズ方法について説明します。  
  
## 一般的なタスク  
  
|タスク|関連するコンテンツ|  
|---------|---------------|  
|**レポート ビューでの、列の追加、列の削除、および列の並べ替え:** テーブル ベースのビューに表示する列を指定できます。また、列の表示順序を指定することもできます。  さらに、レポートの表の行を、列の値で並べ替えることもできます。|-   [方法: レポート ビューの列をカスタマイズする](../profiling/how-to-customize-report-view-columns.md)|  
|**レポートからの小規模関数の削除:** 指定したしきい値を下回る関数をレポートから削除できます。|-   [方法: レポート ビューでノイズ除去を設定する](../profiling/how-to-configure-noise-reduction-in-report-views.md)|  
|**レポート ビューのデータのフィルター処理:** レポートに表示されるデータをプロファイリング実行の時間セグメントに限定できます。  概要ビューのタイムライン グラフで時間セグメントを指定するか、レポート ビュー フィルターで定義するクエリで時間セグメントを指定することができます。  また、コード ファイルで定義された関数だけを表示するように、レポートをフィルター処理することもできます。|-   [レポート ビューのフィルター処理](../profiling/filtering-report-views.md)<br />-   [方法: 概要ビューのタイムラインからレポート ビューをフィルター処理する](../Topic/How%20to:%20Filter%20Report%20Views%20from%20the%20Summary%20Timeline.md)<br />-   [方法: レポート ビューをフィルター処理してマイ コードのみを表示する](../Topic/How%20to:%20Filter%20Profiling%20Tools%20Report%20Views%20to%20Display%20Just%20My%20Code.md)<br />-   [プロファイリング ツールのレポート ビュー フィルター](../profiling/performance-report-view-filter.md)|
