# <a name="understand-performance-collection-methods"></a>パフォーマンス収集方法について

Visual Studio プロファイル ツールには、パフォーマンス データを収集するための 5 つの方法が用意されています。 この記事では、これらの方法について説明し、特定の方法がデータの収集方法としてどのようなシナリオに適しているのかを示します。

> [!NOTE]
> Windows 8 および Windows Server 2012 の強化されたセキュリティ機能によって、Visual Studio プロファイラーがこれらのプラットフォームでデータを収集する方法に大幅な変更が必要になりました。 UWP アプリにも新しい収集手法が必要です。 「[Windows 8 および Windows Server 2012 アプリケーションのパフォーマンス ツール](../profiling/performance-tools-on-windows-8-and-windows-server-2012-applications.md)」を参照してください。

|メソッド|説明|
|------------|-----------------|
|[サンプリング](#sampling)|アプリによって行われた作業に関する統計データを収集します。|
|[インストルメンテーション](#instrumentation)|各関数呼び出しのタイミングに関する詳細情報を取得します。|
|[コンカレンシー](#concurrency)|マルチスレッド アプリに関する詳細情報を収集します。|
|[.NET メモリ](#net-memory)|.NET メモリの割り当ておよびガベージ コレクションに関する詳細情報を収集します。|
|[階層の相互作用](#tier-interaction)|SQL Server データベースへの同期 ADO.NET 関数呼び出しに関する情報を収集します。<br /><br /> Visual Studio のどのエディションでも、階層の相互作用プロファイル データを収集できます。 ただし、そのデータは Visual Studio Enterprise でのみ表示できます。|

一部のプロファイル方法では、ソフトウェアおよびハードウェアのパフォーマンス カウンターなど、ここで示した以外の情報も収集できます。 詳細については、「[追加のパフォーマンス データの収集](../profiling/collecting-additional-performance-data.md)」を参照してください。

## <a name="sampling"></a>サンプリング

サンプリング プロファイル方式を使用すると、プロファイルの実行中にアプリで実行される作業に関する統計データを収集できます。 サンプリング方式は負荷が少ないため、アプリ メソッドの実行にほとんど影響しません。

サンプリングは、Visual Studio プロファイル ツールの既定の方式です。 これは次のタスクに役立ちます。

- アプリのパフォーマンスの初期探索
- マイクロプロセッサ (CPU) の使用に関連するパフォーマンス上の問題を調査する場合

サンプリング プロファイル方式では、コンピューター CPU を一定の間隔で中断し、関数の呼び出し履歴を収集します。 排他サンプル数は、実行中の関数に対してインクリメントされます。 包括数は、呼び出し履歴のすべての呼び出し元関数に対してインクリメントされます。 サンプリング レポートには、プロファイル対象のモジュール、関数、ソースコード行、および命令について、これらのカウントの合計が表示されます。

既定では、サンプリング間隔は CPU のサイクル数に設定されます。 ただし、この間隔の種類を別の CPU パフォーマンス カウンターに変更することや、カウンター イベントの回数を間隔として設定することもできます。 階層の相互作用プロファイル (TIP) データを収集することもできます。 このデータから、ADO.NET を介して SQL Server データベースに対して行われたクエリに関する情報がわかります。

## <a name="instrumentation"></a>インストルメンテーション

インストルメンテーション プロファイル方式では、プロファイル対象アプリにおける関数呼び出しのタイミングに関する詳細情報を収集します。 インストルメンテーション プロファイル方式は、次のようなタスクに適しています。

- ディスク I/O などの入出力のボトルネックを調査する場合
- 特定のモジュールまたは一連の関数について詳細に調査する場合

このインストルメンテーション方式では、コードをバイナリ ファイルに挿入します。 このコードを使用すると、インストルメント化されたファイル内のすべての関数と、それらの関数で行われる各関数呼び出しのタイミング情報をキャプチャできます。 また、この方式では、関数がファイルへの書き込みなどの操作をオペレーティング システムに要求した時間も記録されます。

インストルメンテーション レポートでは、関数またはソースコード行の実行にかかった時間の合計が、次の 4 つの値を使用して表されます。

- 包括経過 - 関数またはソースコード行の実行に費やされた合計時間。

- アプリケーション包括 - 関数またはソースコード行の実行に費やされた時間。 オペレーティング システムへの呼び出しは除外されます。

- 排他経過 - 関数またはソースコード行の実行に費やされた時間。 他の関数の呼び出しは除外されます。

- アプリケーション排他 - 関数またはソースコード行の実行に費やされた時間。 オペレーティング システムまたはその他の関数の呼び出しは除外されます。

インストルメンテーション方式を使用して、CPU およびソフトウェアの両方のパフォーマンス カウンターを収集することもできます。

## <a name="concurrency"></a>コンカレンシー

コンカレンシー プロファイルでは、マルチスレッド アプリに関する情報が収集されます。 リソース競合プロファイルでは、競合するスレッドによる共有リソースへのアクセスで待機が発生するたびに、詳細な呼び出し履歴情報が収集されます。 コンカレンシーの可視化を使用すると、マルチスレッド アプリと以下との対話に関するより一般的な情報も収集されます。

- それ自体。
- ハードウェア。
- オペレーティング システム。
- ホスト コンピューター上の他のプロセス。

リソース競合レポートには、競合の合計数が表示されます。 また、モジュール、関数、ソースコード行、および命令によってリソースが待機された合計時間も報告されます。 また、タイムライン グラフにも発生した競合が表示されます。

コンカレンシー ビジュアライザーには、以下を特定するために役立つグラフィカルな情報が表示されます。

- パフォーマンスのボトルネック。
- CPU 過小使用。
- スレッドの競合。
- スレッドの移行。
- 同期の遅延。
- 重複 I/O の領域。

  グラフィカルな出力は、呼び出し履歴とソース コードにできる限りリンクされます。 コンカレンシーの視覚化データは、コマンドライン アプリと Windows アプリについてのみ収集できます。

## <a name="net-memory"></a>.NET メモリ

.NET メモリ割り当てのプロファイル方式を使用すると、プロファイルされたアプリの .NET Framework オブジェクトの割り当てごとに CPU の割り込みが実行されます。 プロファイラーを使用してオブジェクトの有効期間データも収集する場合、.NET Framework のガベージ コレクションのたびに CPU の割り込みが実行されます。

プロファイラーは、割り当てで作成されたオブジェクト、またはガベージ コレクションで破棄されたオブジェクトの型、サイズ、および数に関する情報を収集します。

- メモリの割り当てイベントが発生すると、プロファイラーは関数の呼び出し履歴に関する追加情報を収集します。 プロファイラーによって、現在実行中の関数の排他割り当て数がインクリメントされます。 また、呼び出し履歴のすべての呼び出し元関数に対して包括数がインクリメントされます。 .NET レポートには、プロファイル対象の型、モジュール、関数、ソース コード行、および命令について、これらのカウントの合計が表示されます。

- ガベージ コレクションが発生すると、プロファイラーによって、破棄されたオブジェクトと各ガベージ コレクション世代のオブジェクトに関するデータが収集されます。 プロファイル実行の終了時には、明示的に破棄されなかったオブジェクトに関するデータがプロファイラーによって記録されます。 オブジェクトの有効期間レポートには、プロファイル実行中にメモリが割り当てられた各型の合計が表示されます。

.NET メモリ プロファイルは、サンプリング モードまたはインストルメンテーション モードで使用できます。 選択するモードによって、.NET メモリのプロファイルに固有のメモリ割り当てレポートおよびオブジェクトの有効期間レポートに影響が及ぶことはありません。

- .NET メモリ プロファイルをサンプリング モードで実行すると、プロファイラーでは間隔としてメモリ割り当てイベントが使用されます。 また、割り当てられたオブジェクトとバイトの合計数が包括的で排他的な値としてレポートに表示されます。

- .NET メモリのプロファイルをインストルメンテーション モードで実行すると、プロファイラーによって、タイミングに関する詳細情報と共に、包括的割り当て値および排他的割り当て値が収集されます。

## <a name="tier-interaction"></a>階層の相互作用

階層の相互作用のプロファイルでは、[!INCLUDE[vstecado](../data-tools/includes/vstecado_md.md)] ページまたはその他のアプリと [!INCLUDE[vstecasp](../code-quality/includes/vstecasp_md.md)] データベースとの間で行われた同期的な [!INCLUDE[ssNoVersion](../data-tools/includes/ssnoversion_md.md)] 呼び出しに関する情報が、プロファイル データ ファイルに追加されます。 データには、呼び出しの回数と時間、および最長時間と最短時間が含まれます。 階層の相互作用データは、サンプリング、インストルメンテーション、.NET メモリ、またはコンカレンシーの各方式で収集されたプロファイリング データに追加できます。
