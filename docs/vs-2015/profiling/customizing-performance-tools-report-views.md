# <a name="customizing-performance-tools-report-views"></a>パフォーマンス ツール のレポート ビューのカスタマイズ
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

このセクションでは、[!INCLUDE[vsprvs](../includes/vsprvs-md.md)] プロファイリング ツールで生成されたレポートのカスタマイズ方法について説明します。  
  
## <a name="common-tasks"></a>一般的なタスク  
  
|タスク|関連するコンテンツ|  
|----------|---------------------|  
|**レポート ビューで列を追加、削除、並べ替える:** テーブルベース ビューに表示する列を指定したり、列の表示順を指定したりできます。 レポート テーブルの行を列の値で並べ替えることもできます。|-   [方法: レポート ビューの列をカスタマイズする](../profiling/how-to-customize-report-view-columns.md)|  
|**レポートから小さい関数を除外する:** 指定したしきい値より小さい関数をレポートから除外できます。|-   [方法: レポート ビューでノイズ除去を設定する](../profiling/how-to-configure-noise-reduction-in-report-views.md)|  
|**レポート ビューの日付を絞り込む**: レポートに表示されるデータをプロファイル実行の時間枠に限定できます。 概要ビューのタイムライン グラフに時間枠を指定したり、レポート ビュー フィルターに定義するクエリに時間枠を指定したりできます。 レポートにフィルターを設定し、コード ファイルに定義されている関数のみを表示することもできます。|-   [レポート ビューのフィルター処理](../profiling/filtering-report-views.md)<br />-   [方法: 概要ビューのタイムラインからレポート ビューをフィルター処理する](../profiling/how-to-filter-report-views-from-the-summary-timeline.md)<br />-   [方法: プロファイリング ツールのレポート ビューをフィルター処理してマイ コードのみを表示する](../profiling/how-to-filter-profiling-tools-report-views-to-display-just-my-code.md)<br />-   [パフォーマンス レポート ビュー フィルター](../profiling/performance-report-view-filter.md)|
