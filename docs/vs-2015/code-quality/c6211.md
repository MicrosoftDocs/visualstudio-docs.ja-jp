# <a name="c6211"></a>C6211
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6211: メモリ リークを起こして\<ポインター > 例外が原因です。 メモリをクリーンアップするには、ローカルの catch ブロックを使用してください  
  
 この警告は、割り当てられたメモリを解放されていない例外がスローされたときを示します。 パスの最後のステートメントでは、例外をスローでした。 アナライザーは、`_Analysis_mode_(_Analysis_local_leak_checks_)` SAL 注釈が指定されている場合にのみこの条件をチェックします。 既定では、この注釈は Windows カーネル モード (ドライバー) コードに対して指定されます。 SAL 注釈の詳細については、次を参照してください。 [C と C++ コードの欠陥を削減する SAL 注釈を使って](../code-quality/using-sal-annotations-to-reduce-c-cpp-code-defects.md)します。  
  
## <a name="example"></a>例  
 次のコードはこの例外がスローされること、2 つ目の割り当て中になり、その結果、リーク、最初の割り当てのため、警告またはで表されるコードのどこかには例外をスローする可能性があります、"`code ...`"コメントとそれによって両方の割り当てのリークが発生します。  
  
```cpp  
// cl.exe /analyze /c /EHsc /nologo /W4   
#include <sal.h>  
  
_Analysis_mode_(_Analysis_local_leak_checks_)   
void f( )  
{  
    char *p1 = new char[10];  
    char *p2 = new char[10];  
  
    // code ...  
  
    delete[] p2;  
    delete[] p1;  
}  
  
```  
  
 同じ方法で割り当てし、この問題を解決するには、例外ハンドラーを追加します。  
  
```cpp  
// cl.exe /analyze /c /EHsc /nologo /W4  
#include <sal.h>  
#include <new>  
#include <iostream>  
using namespace std;  
  
_Analysis_mode_(_Analysis_local_leak_checks_)   
  
void f()  
{  
    char *p1 = nullptr;   
    char *p2 = nullptr;  
  
    try  
    {  
        p1 = new char[10];  
        p2 = new char[10];  
  
        // code ...  
  
        delete [] p2;  
        delete [] p1;  
    }  
    catch (const bad_alloc& ba)  
    {  
        cout << ba.what() << endl;  
        delete [] p2;  
        delete [] p1;  
    }  
    // code ...  
}  
  
```  
  
 この種のリークの可能性を完全を回避するには、C++ 標準テンプレート ライブラリ (STL) によって提供されるメカニズムを使用します。 以下の[shared_ptr](http://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860)、 [unique_ptr](http://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)、および[ベクター](http://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f)します。 詳細については、次を参照してください。[スマート ポインター](http://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435)と[C++ 標準ライブラリ](http://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee)します。  
  
```cpp  
// cl.exe /analyze /c /EHsc /nologo /W4  
#include <sal.h>  
#include <vector>  
#include <memory>  
  
using namespace std;  
  
_Analysis_mode_(_Analysis_local_leak_checks_)   
  
void f( )  
{  
    // use 10-element vectors in place of char[10]  
    vector<char> v1;  
    vector<char> v2;  
  
    for (int i=0; i<10; ++i) {  
        v1.push_back('a');  
        v2.push_back('b');  
    }  
    // code ...  
  
    // use unique_ptr if you still want char[10]  
    unique_ptr<char[]> a1(new char[10]);  
    unique_ptr<char[]> a2(new char[10]);  
  
    // code ...  
  
    // No need for delete; vector and unique_ptr   
    // clean up when out of scope.  
}  
  
```
