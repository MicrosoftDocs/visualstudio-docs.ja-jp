# <a name="how-to-create-a-work-item-for-a-managed-code-defect"></a>方法: マネージド コードの障害に対する作業項目を作成する
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

作業項目トラッキング機能を使用して、[!INCLUDE[vsPreLong](../includes/vsprelong-md.md)] 内から作業項目をログに記録できます。 この機能を使用するには、プロジェクトを [!INCLUDE[esprfound](../includes/esprfound-md.md)] のチームプロジェクトに含める必要があります。

### <a name="to-create-a-work-item-for-managed-code-defect"></a>マネージコードの欠陥の作業項目を作成するには

1. **[コード分析]** ウィンドウで、警告を選択します。

2. **[アクション]** を選択し、 **[作業項目の作成]** を選択して、作成する作業項目の種類を選択します。

     新しい作業項目が作成され、欠陥情報を指定します。

### <a name="to-create-a-work-item-for-multiple-managed-code-defects"></a>複数のマネージコード障害に対する作業項目を作成するには

1. **エラー一覧**で、複数の警告 を選択し、警告を右クリックします。

2. **[作業項目の作成]** をポイントし、作成する作業項目の種類をクリックします。

     バグ情報を指定するために、選択したすべての警告に対して1つの作業項目が作成されます。
