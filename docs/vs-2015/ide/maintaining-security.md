# <a name="maintaining-security"></a>セキュリティの維持
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

セキュリティの価値は継続的な警戒にある、とよく言われます。 アプリケーションのデザイン時や開発時にいかに万全のセキュリティ対策を行っても、配置後にセキュリティの欠陥が明らかになる可能性も想定する必要があります。 アプリケーションの監査とイベント ログの分析から、それまでわからなかった欠陥が見つかることもあります。

 さらに、自分が作成したアプリケーションのセキュリティを維持するだけでなく、アプリケーションが実行されるプラットフォームやアプリケーションが依存している他の製品についても、セキュリティの脅威や欠陥に関する最新情報に目を配る必要があります。

 [セキュリティ、プライバシー、アカウント](http://go.microsoft.com/fwlink/?LinkId=72881)ウイルス、パスワード、保護者による制限、ファイアウォール、ドライブの暗号化に関する情報など、セキュリティ、プライバシー、およびユーザーアカウントに関するヘルプを表示します。

 [Microsoft セキュリティ情報検索](/security-updates/)このページでは、以前にリリースされた情報を簡単に見つけることができます。 IT 専門家を対象に、セキュリティ更新プログラムに関する詳しい情報を提供しています。

 [Windows Server 2003 セキュリティガイド](/previous-versions/tn-archive/cc163140 (v = technet. 10) は、IT プロフェッショナルが Windows Server 2003 環境を安全に運用するために必要なガイダンスを提供し、適切な場所から適切なコンテンツに適切にアクセスできるようにします。

 [企業セキュリティのベストプラクティス](/previous-versions/tn-archive/cc750076%28v%3dtechnet.10%29)エンタープライズネットワークにおけるセキュリティのさまざまな側面に焦点を当てるホワイトペーパーのコレクションです。

 [Microsoft Baseline Security Analyzer](/windows/security/threat-protection/mbsa-removal-and-guidance)Microsoft Baseline Security Analyzer (MBSA) は、個々のホームユーザー、企業ユーザー、または管理者が、1つまたは複数の Windows ベースのコンピューターをスキャンして、一般的なセキュリティ構成の誤りを検出できるようにするツールです。
