## <a name="animation-fundamentals"></a>アニメーションの基礎

### <a name="animation-best-practices-in-visual-studio"></a>Visual Studio でのアニメーションのベストプラクティス
 これらの規則に従って、Visual Studio の IDE 全体で一貫したわかりやすいアニメーションスタイルを確保します。

- **選択的です。** 特定の目的を果たすアニメーションを制限します。

- 移行が迅速かつ自然になるようにするに**は、タイミングと速度が重要**です。

  - アニメーションの遷移を1秒 (500 ミリ秒) 以内に完了します。

  - 頻繁に発生するアニメーションは、ユーザーのワークフローを中断しないようにする必要があります。

  - アニメーションはあまり速くも目障りも理解しにくいのではありませんが、移行が完了するまでに1つの待ちきれずを作成することはできません。

  - 重要度を強調するには、変数のタイミングを使用します。 たとえば、クラスダイアグラムの項目のシーケンスを移動するときに、項目間の遷移の速度を下げ、重要な項目に焦点を当てることができます。

- ある状態から別の状態への**段階的な非直線的なイージングを使用**して、穏やかで自然な動きを示す

- 可能な場合は、マウスの下に対話型の要素を示すために、 **ホバー時に微妙なアニメーションを使用** します。

- 機能のアニメーションに大きく依存している場合は、[**ツール > オプション**] ダイアログボックスでオプションとしてローカルに**オフにする手段を提供**します (すべての機能について)。

- 一度**に1つのアニメーションのみを実行**し、情報を1つだけ伝達する必要があります。

- **はらみが重要です。** ほとんどの場合、アニメーションでは目的を果たすためにユーザーに注意する必要がありません。 タイミング、シーケンス処理、および動作の微妙な変更により、知覚に大きな影響が及ぶ可能性があります。また、有効なアニメーションと効果のないアニメーションの違いを考えてください。

- アニメーションを使用して何らかの注意を引く場合は、ユーザーの思考を **中断する価値がある**ことを確認してください。

- アニメーションを使用して**進行状況または状態を表示する場合**:

  - 基になるプロセスが進められていないときに、進行状況の移動を表示しないようにします。

  - 不確定のプロセスを確定したプロセスと区別します。

  - アニメーションの完了と失敗の状態が識別可能であることを確認します。

  - 状態を表示する効果アニメーションの使用を最小限に抑え、実際の使用に関する追加情報を提供することによって実際の値を持つことを確認します。 一時的な状態の変更や緊急時の例が含まれます。

#### <a name="do-not"></a>次の行為は禁止とします。

- 小さな移動 (小さなフットプリントでの移動) を使用して、フェードを優先し、オブジェクトの移動を変更します。

- 画面上の大きな領域に対して実行されるアニメーションを使用します。 サイズに関係なく、このスタイルのアニメーションはユーザーにとって煩雑です。

- ユーザーが現在フォーカスしているオブジェクトや対話するオブジェクトに関係のないアニメーションを使用します。

- ユーザーの操作を必要とするアニメーションを使用して状態をリセットします。たとえば、点滅する通知を停止するために、点滅している通知にユーザーが応答するように強制します。 どのような方法でも、それらを破棄するだけで十分です。

  これらのベストプラクティスのアプリケーションの詳細については、「 [アニメーションパターン](../../extensibility/ux-guidelines/animations-for-visual-studio.md#BKMK_AnimationPatterns)」を参照してください。

### <a name="animation-metrics"></a>アニメーションのメトリック

- システムは、10ミリ秒未満のユーザージェスチャに視覚的に反応する必要があります。

- アニメーション化された遷移は、完了するまでに500ミリ秒以上かかることはありません。

- より長い時間を必要とする遷移を補う方法の1つは、2つの部分に分けることです。たとえば、アニメーションの最初の部分は空のコンテンツコンテナー (最大500ミリ秒) であり、その後、コンテンツがコンテナーにフェードします (最大500ミリ秒)。

- 計算できる読み込み時間については、決定の進行状況インジケーター (パーセント完了の進行状況インジケーター) をお勧めします。

- 計算できない読み込み時間については、カーソルや埋め込みスピンアニメーション (読み込みまたは動作インジケーター) などのビジーインジケーターが適しています。

### <a name="animation-as-communicator"></a>Communicator としてのアニメーション
 Visual Studio の UI では、アニメーションは通信ツールとしてのみ機能します。  UI の構造上の変更など、さまざまな情報を伝達するために使用されます。たとえば、メニューを開いたり閉じたりする場合などです。 アニメーションは、インストールの進行状況の視覚化など、複雑なシステムの時間に依存する動作を視覚化するのに役立ちます。また、アラートや通知に注目するために使用されます。

 通常、UI アニメーションは、視覚化、強調表示、シミュレーション、応答時間/進行状況を示す4つの方法で機能します。

#### <a name="visualize"></a>視覚化する
 アニメーションは、オブジェクトの3次元の性質を強調し、ユーザーが空間構造を簡単に視覚化できるようにすることができます。 これを実現するために、アニメーションでは、オブジェクトを完全な円で回転させたり、ゆっくりと変化させたり、オブジェクトのサイズを少し増やして、ロールオーバーやフォーカスを強調することが必要になる場合があります。

 3次元オブジェクトはユーザーコントロールと共に移動できますが、デザイナーは、オブジェクトの最適な理解を提供する動きを最適にアニメーション化する方法を、プログラムまたは手動で決定する必要があります。 このプログラミングされたアニメーションは、ユーザーがオブジェクトの上にカーソルを置くことによってアクティブ化できます。一方、ユーザー制御の移動では、ユーザーがオブジェクトの操作方法を理解する必要があります。 一度に1つの軸または向きに移動するように制限します。スケール、回転、変換のいずれかを実行しますが、同時に複数の操作を行うことはできません。

 [視覚化] カテゴリには、データ、リレーションシップ、状態、構造、シーケンス、および時間の側面が含まれます。

##### <a name="data"></a>Data
 複雑な情報と変数の情報を示します。

- グラフやグラフなどの情報の視覚エフェクトの移動

- シーケンス、ガイドツアー、ページングのステップ実行

- 特定の情報の詳細、ポイント、および強調表示

- オーバーレイの詳細と、フォーカスされた要素の上に追加する情報

- ある構造体または組織の表現から別の構造体へのモーフィング

- 時間の経過と共に変化を表す時間スライダー、ジョグ/シャトル車輪、およびトランスポートコントロール (再生、停止、および一時停止) を使用します。

##### <a name="relationships"></a>リレーションシップ

- 階層と親子関係または兄弟リレーションシップを表示する

- 1つの要素が別の要素を生成する

- 1つの要素が別の要素に最小化される

- 1つの要素が別の要素にテザリングさ

##### <a name="state"></a>State

- コンテンツの更新。

- ユーザーのフォーカスと選択

- 進捗状況

- エラー

##### <a name="structure"></a>構造体

- 1つのノードでの構造のピボット

- 再回転

- 最小化と最大化、または展開と折りたたみ

##### <a name="sequence"></a>シーケンス

- スライドショーシーケンス

- 反転 (画像を)

##### <a name="time"></a>Time

- 時間の経過に伴う変更、時間の短縮、スクリーンキャストの表示

- ごみ箱へ移動、元に戻す、やり直し

- 履歴状態の復元

#### <a name="attract-attention"></a>注目を集める
 ユーザーの注意を複数の要素に対して、または更新された情報をユーザーに通知することが目的である場合は、アニメーションが適切な場合があります。 たとえば、アプリケーションのスタートページでは、ページが読み込まれた後にスライドするはじめにボタンが使用される場合があります。

 ルールとして、画面上の最後に移動する要素は、ユーザーの注意を引きます。  一連のアニメーション要素では、ユーザーの注意は最後に移動するオブジェクトに従います。

##### <a name="alert"></a>アラート:

- ユーザーにアラートを通知し、注意を喚起し、進行状況を表示する

- 何かが正しく行われているか、間違っているか、進行状況または進行状況の変化を示していることを示します。

- タスクの実行中にユーザーにメッセージを表示する (オンラインで詳細情報を検索したり、現在のタスクについて学習したりするなど)

##### <a name="notifications"></a>通知

- エラー状態についてユーザーに通知する

- ユーザーが他のユーザーに参加するかどうかを確認する

- ダウンロードが完了したときなど、プロセスが完了または変更されたことをユーザーにゆっくりと伝えます。

#### <a name="simulate"></a>シミュレート
 このカテゴリは、physicality と次元について説明します。

- オブジェクトの取得元または移動先を示す

- 展開と折りたたみ、または開閉を行う

- パン、スクロール、およびページめくり

- スタックと z オーダー

- カルーセルとアコーディオン

- UI の反転と回転

#### <a name="response-and-progress-indicators"></a>応答インジケーターと進行状況インジケーター
 進行状況インジケーターには、いくつかの注目すべき利点があります。

- 確定および不確定の進行状況インジケーターはどちらも、システムがクラッシュしておらず、問題を解決していることをユーザーに知らせるためます。

- 確定インジケーターは、操作の進行状況をユーザーにわかりやすく示すと共に、完了に近づいたことを示します。

## <a name="animation-patterns"></a><a name="BKMK_AnimationPatterns"></a> アニメーションパターン

### <a name="overview"></a>概要
 Visual Studio でのアニメーションは、特定の機能を提供することを目的としており、ユーザーの生産性を低下させるものではありません。 一般的なアニメーションの特性は次のとおりです。

- 小規模で控えめ

- 自然で現実的

- 微妙なものと、

- 高速で効率的

- 上ではなく、緩やか

#### <a name="appear-and-disappear"></a>表示と非表示
 このパターンでは、要素は、遷移アニメーションを使用せずに、可視から非ビューに切り替えます。

##### <a name="correct-usage"></a>正しい使用方法
 ユーザーが気にしたり妨げたりすることがないように、すぐに表示または非表示にする必要がある新しい UI 要素。 また、低速の移動アニメーションはパフォーマンスドラッグとして認識される場合があります。これは、表示と非表示のスタイルでは発生しません。

##### <a name="incorrect-usage"></a>不適切な使用法
 UI が突然表示され、ユーザーは何が起こったのかわかりません。また、アニメーションを追加すると、コンテキストを理解するのに役立ちます。

#### <a name="fade-in-and-fade-out"></a>フェードインとフェードアウト

 ![Visual Studio でフェード&#45;out アニメーションをフェード&#45;&#47;](../../extensibility/ux-guidelines/media/1202-c-fadeinfadeout.png "1202-c_FadeInFadeOut")

##### <a name="correct-usage"></a>正しい使用方法
 これは、最も一般的に推奨される UI アニメーションです。 フローを中断せずに関心を追加する微妙な効果です。 場合によっては、アニメーションがあることをユーザーが認識していないこともあります。これは、スムーズなフローの UI システムを認識するだけです。

- イージングスタイル: 正弦 InOut
