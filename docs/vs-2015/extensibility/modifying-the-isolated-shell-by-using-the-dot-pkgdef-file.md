# <a name="modifying-the-isolated-shell-by-using-the-pkgdef-file"></a>.Pkgdef ファイルを使用した分離シェルの変更
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Pkgdef ファイルは、分離シェルアプリケーションをカスタマイズするために使用できる設定をサポートしています。 アプリケーションをコンピューターにインストールし、アプリケーションを起動するときに Visual Studio シェルによって参照されるときに作成される値を指定します。 設定は、該当するレジストリキーに基づいてファイルに整理されます。  

> [!WARNING]
> VSPackage の source.extension.vsixmanifest ファイルで宣言されていない pkgdef ファイルは、Visual Studio の起動時にはスキャンされないことに注意してください。  

 Pkgdef ファイルには、キー (またはサブキー) で識別されるセクションが含まれます。 `[$RootKey$]` `[$RootKey$\` *subkey* `]` ここで $RootKey $ はアプリケーションのルートキーです。  

 各セクションには、次の形式の名前と値のペアが含まれています: `"` *ValueName* `"=` *値*。  

 値は、引用符で囲まれた文字列か、または dword として表される32ビット整数です。 値には次の制約があります。  

- すべての dword 値は、たとえば、16進形式 `dword:00000001` です。  

   ブール値の場合、1は true を表し、0は false を表します。  

- すべての GUID 文字列は、などのレジストリ形式 `"{00000000-0000-0000-0000-000000000000}"` です。  

- ローカライズ可能なすべてのリソース識別子の形式は、"@*resourceid*" または "#*resourceID*" です。ここで、 *resourceID* はアプリケーション UI パッケージのリソース識別子 (たとえば、) `"@102"` です。 アプリケーション UI パッケージは、AppLocalizationPackage 設定で参照されるパッケージです。  

  たとえば、次のように入力します。  

```  
"HideSolutionConcept"=dword:00000001  
"DefaultDebugEngine"="{00000000-0000-0000-0000-000000000000}"  
```  

 Pkgdef ファイルにコメントを追加できます。 1行のコメントには、最初の2文字として2つのスラッシュがあります。  

 置換文字列の一覧については、「」を参照してください [。Pkgdef と。Pkgundef ファイル](../extensibility/substitution-strings-used-in-dot-pkgdef-and-dot-pkgundef-files.md)。  

 以下のセクションでは、分離モードでの Visual Studio シェルの動作に影響する特定のレジストリ値について説明します。 このファイルには、アプリケーションの追加のレジストリ値を定義することもできます。  

> [!NOTE]
> Pkgdef ファイルに設定が指定されていない場合、対応するエントリはレジストリに作成されません。

## <a name="binding-path-settings"></a>バインドパスの設定  
 [$RootKey $ \ bindingpaths] キーには、 \\ {00000000-0000-0000-0000-000000000000} シェルがアセンブリをチェックするディレクトリの一覧が含まれています。 これらのディレクトリは、シェルがアプリケーションのプライベートアセンブリをプローブするディレクトリの一覧に追加されます。  

 既定では、関連付けパスのエントリは、pkgdef ファイルに追加されません。 ただし、Visual Studio インストールディレクトリの次のサブディレクトリは、レジストリのアプリケーションバインドパスの一覧に自動的に追加されます。  

- Common7\IDE\  

- Common7\IDE/ \\ privateassemblies  

- Common7\IDE \\ \ publicassemblies  

  バインドパスにディレクトリを追加するには、"*directoryName*" = "" という形式のエントリを追加します。ここで、 *directoryName* は絶対パスです。 たとえば、次のように入力します。  

```  
[$RootKey$\BindingPaths\{00000000-0000-0000-0000-000000000000}]  
"$RootFolder$\directory1"=""  
"%CommonProgramFiles%\directory2"=""  
```

## <a name="package-menu-item-settings"></a>パッケージのメニュー項目の設定  
 [$RootKey $ \ メニュー] レジストリキーは、アプリケーションの UI リソースファイルを定義します。  

 メニュー項目の値の形式は "{*vsuipackageguid*}" = ", *resourceId*, *versionNumber*" です。ここで、 *vsuipackageguid* はアプリケーション ui パッケージの GUID、 *resourceid* は ui 要素を含む CTMENU リソースのリソース識別子、 *versionNumber* は、ctmenu リソースの仮想バージョン番号です。 詳細については、「 [相互運用機能アセンブリコマンドハンドラーの登録](../extensibility/internals/registering-interop-assembly-command-handlers.md)」を参照してください。  

 既定では、アプリケーション UI パッケージの pkgdef ファイルにメニュー項目エントリが作成されます。  

 メニュー項目を提供し、アプリケーションの一部として配布される各パッケージについて、パッケージのメニュー項目エントリを追加します。
