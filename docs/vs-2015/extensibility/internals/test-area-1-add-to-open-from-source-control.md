# <a name="test-area-1-add-toopen-from-source-control"></a>テスト領域 1: ソース管理への追加とオープン
[!INCLUDE[vs2017banner](../../includes/vs2017banner.md)]

このソース管理プラグインのテスト領域では、ソース管理下にソリューションまたはプロジェクトを配置し、ソース管理からそのプロジェクトを取得します。  
  
## <a name="command-menu-access"></a>コマンドメニューへのアクセス  
 [!INCLUDE[vsprvs](../../includes/vsprvs-md.md)]テストケースでは、次の統合開発環境メニューパスが使用されます。  
  
- の場合 [!INCLUDE[vsvss](../../includes/vsvss-md.md)] 、ソース管理から開く: **File**、 **open**、 **Project** / **Solution**、場所を検索 [!INCLUDE[vsvss](../../includes/vsvss-md.md)] します。  
  
- その他のソース管理プラグインの場合は、ソース管理から開く、[ **ファイル**]、[ **ソース管理**]、[ **ソース管理から開く**]。  
  
- ソース管理に追加: **ファイル**、 **ソース管理**、 **ソース管理ファイルへのソリューションの追加**、 **ソース管理**、 **選択したプロジェクトをソース管理に追加**します。  
  
- ショートカットメニュー (プロジェクト/ソリューション) で、 **ソース管理にソリューションを追加**します。  
  
- ソース管理から追加する: **ファイル**、 **ソース管理**、 **ソース管理からプロジェクトを追加**します。  
  
- で [!INCLUDE[vsvss](../../includes/vsvss-md.md)] は、ソース管理からの追加は、[ **ファイル**]、[ **追加**]、[ **既存のプロジェクト**] からも利用できます。場所を検索 [!INCLUDE[vsvss](../../includes/vsvss-md.md)] します。  
  
    > [!NOTE]
    > このテストでは、ローカルファイルまたはローカル IIS (web サーバー) のパスを使用できます。  
  
## <a name="expected-behavior"></a>想定されている動作  
  
- サポートされているプロジェクトの種類ごとに、ユーザーがソース管理を "追加" および "開く" ことができる必要があります。  
  
- プロジェクトがソース管理に追加されると、対応する \<*ProjectName*> vspscc ファイル (プロジェクトヒントファイル) が作成されます。 除外ファイルの一覧と接続情報が含まれています。 このファイルには、プロジェクト固有の情報が含まれているため、削除しないでください。  
  
- ソリューションがソース管理に追加されると、対応する \<*SolutionName*> vssscc (トリプル S) ファイルが作成されます。 このテキストファイルには、プロジェクトヒントファイルと同様に、接続情報と除外ファイルの一覧が含まれています。 このファイルは一時的なものであり、ソース管理データベースにのみ存在します。  
  
- ソース管理からソリューションを開くと、 \<*SolutionName*> ソース管理データベースにのみ存在する vsscc (Double S) ファイルが、一時ファイルにローカルに作成されます。 このファイルには、ソリューション接続フォルダーからソリューションファイルへのパスが含まれています。 このファイルは一時的なもので、"ソース管理から開く" 操作が完了すると、ローカルコピーが削除されます。  
  
- ソース管理にプロジェクトを追加すると、そのプロジェクトに対してソース管理アクションを実行できます (チェックアウト、取得など)。  
  
## <a name="test-cases"></a>テスト ケース  
 [ソース管理から開く] テスト領域の特定のテストケースを次に示します。  
  
### <a name="case-1a-add-solution-to-source-control"></a>ケース 1a: ソース管理へのソリューションの追加  
 このテストケースでは、ソース管理にソリューションを追加することに重点を置いています。  
  
|操作|テストステップ|確認が必要な結果|  
|------------|----------------|--------------------------------|  
|クライアントプロジェクトを含むソリューションをソース管理に追加する|1. クライアントプロジェクトを作成します。<br />2. ソリューションをソース管理に追加します ([**ファイル**]、[ **ソース管理**]、[ **ソリューションをソース管理に追加**])。|ソリューション/プロジェクトがソース管理に追加されました。|  
|ファイルシステムまたはローカル IIS Web プロジェクトを含むソリューションをソース管理に追加する|1. ファイルシステムまたはローカル IIS Web プロジェクトを作成します ([参照] ボタンを使用してプロジェクトの場所を指定します。パスによって、作成される Web プロジェクトの種類が決まります)。<br />2. ソリューションをソース管理に追加します ([**ファイル**]、[ **ソース管理**]、[ **ソリューションをソース管理に追加**])。|ソリューション/プロジェクトがソース管理に追加されました。|  
|リモートサイト Web プロジェクトを含むソリューションをソース管理に追加する|1. リモートサイトの Web プロジェクトを作成します。<br />2. ソリューションをソース管理に追加します ([**ファイル**]、[ **ソース管理**]、[ **ソリューションをソース管理に追加**])。<br />3. [FrontPage のアクセス警告] ダイアログボックスで [ **OK]** をクリックします。|ソリューションがソース管理に追加されました。<br /><br /> リモートサイトプロジェクトがソース管理下にありません。 (リモートサイトプロジェクトは、独自の IIS サーバーから制御する必要があります)。|  
|**選択したプロジェクトをソース管理に追加**して、ソース管理に単一のプロジェクトソリューションを追加します。|1. プロジェクトソリューションを1つ作成します。<br />2. 選択としてソース管理にソリューションのみを追加します (**ファイル**、 **ソース管理**、 **選択したプロジェクトをソース管理に追加**します)。 この手順が成功した場合は、次の手順に進みます。<br />3. 選択項目としてプロジェクトをソース管理に追加します (**ファイル**、 **ソース管理**、 **選択したプロジェクトをソース管理に追加**します)。<br />4. [ **はい** ] をクリックして、同じ場所にプロジェクトを追加します。<br />5. [**編集**] ダイアログ**ボックスで [チェックアウト] をクリック**します。|`Result from Step 2:`<br /><br /> プロジェクトとプロジェクト内のすべてのファイルにはチェックアウトされたソース管理インジケーターがあり、ツールヒントに "ソース管理下にありません" と表示されます。<br /><br /> `Result from Step 5:`<br /><br /> プロジェクトファイルとソリューションファイルは、ソース管理内の同じフォルダーにあります。|  
|ソース管理へのソリューションの追加をキャンセルする|1. プロジェクトソリューションを1つ作成します。<br />2. プロジェクトとソリューションをソース管理に追加しようとしています。 この手順が成功した場合は、次の手順に進みます。<br />3. ソース管理システムに移行した後、キャンセルします。|`Result from Step 2:`<br /><br /> [プロジェクトの場所の設定] の [ソース管理] ダイアログボックスは、1回だけ表示されます。<br /><br /> `Result from Step 3:`<br /><br /> プロジェクトの追加が取り消されました。プロジェクト/ソリューションがソース管理下にありません。また、[ソース管理に追加] メニューは引き続き利用できます。|  
  
### <a name="case-1b-open-solution-from-source-control"></a>ケース1b。 ソース管理からソリューションを開く  
 このテストケースでは、ソース管理からソリューションを開くことに重点を置いています。  
  
|操作|テストステップ|確認が必要な結果|  
|------------|----------------|--------------------------------|  
|ソース管理からクライアントプロジェクトを含むソリューションを開く|1. クライアントプロジェクトを作成します。<br />2. ソース管理にソリューションを追加します。<br />3. ソリューションを閉じます。<br />4. ソース管理から新しい場所にソリューションを開きます。|ソリューション/プロジェクトがソース管理から開かれました。|  
|ソース管理からローカルまたは IIS Web プロジェクトを含むソリューションを開く|1. ローカルまたは IIS Web プロジェクトを作成します。<br />2. ソース管理にソリューションを追加します。<br />3. ソリューションを閉じます。<br />4. ソース管理から新しい場所にソリューションを開きます。|ソリューション/プロジェクトがソース管理から開かれました。|  
|ソース管理からリモートサイトの Web プロジェクトを含むソリューションを開く|1. リモートサイトの Web プロジェクトを作成します。<br />2. ソース管理にソリューションを追加します。 この手順が成功した場合は、次の手順に進みます。<br />3. ソリューションを閉じます。<br />4. ソース管理から新しい場所にソリューションを開きます。|`Result from Step 2:`<br /><br /> リモートサイト Web がソース管理下にありません。<br /><br /> `Result from Step 4:`<br /><br /> ソリューションがソース管理から開かれました。<br /><br /> リモートサイトプロジェクトは読み込まれていますが、ソース管理下にありません。|  
  
### <a name="case-1c-add-solution-from-source-control"></a>ケース 1c: ソース管理からのソリューションの追加  
 このテストケースは、ソース管理からソリューションを追加することに重点を置いています。  
  
|操作|テストステップ|確認が必要な結果|  
|------------|----------------|--------------------------------|  
|空のソリューションに追加—単一のプロジェクトソリューション|1. プロジェクトソリューションを1つ作成します。<br />2. ソース管理にソリューションを追加します。<br />3. ソリューションを閉じます。<br />4. 2 番目の空のソリューションを作成します。<br />5. 以前に制御したソリューションをソース管理 (**ファイル**、 **ソース管理**、 **ソース管理からのプロジェクトの追加**) から追加します。|追加したプロジェクトが **ソリューションエクスプローラー** に表示され、チェックインされます。|  
|単一のプロジェクトを使用したソリューションへの追加-単一プロジェクト|1. 1 つのプロジェクトでソリューションを作成します。<br />2. ソース管理にソリューションを追加します。<br />3. ソリューションを閉じます。<br />4. 2 番目の空のソリューションを作成します。<br />5. 以前に制御したソリューションをソース管理 (**ファイル**、 **ソース管理**、 **ソース管理からのプロジェクトの追加**) から追加します。|追加したプロジェクトが **ソリューションエクスプローラー** に表示され、チェックインされます。|  
|ソリューションへの追加—選択によってソース管理に追加されたソリューション|1. プロジェクトを含むソリューションを作成します。<br />2. 選択項目としてソース管理にソリューションのみを追加します。 この手順が成功した場合は、次の手順に進みます。<br />3. ソリューションを閉じます。<br />4. 新しいソリューションを作成します。<br />5. 以前に制御したソリューションをソース管理 (**ファイル**、 **ソース管理**、 **ソース管理からのプロジェクトの追加**) から追加します。|`Result from Step 2:`<br /><br /> プロジェクトがソース管理下にありません。<br /><br /> `Result from Step 5:`<br /><br /> 最初のソリューションにソリューション項目がある場合は、ソース管理から追加することはできないため、表示されません。<br /><br /> 最初のソリューションのプロジェクトは使用不可として表示されます。|
