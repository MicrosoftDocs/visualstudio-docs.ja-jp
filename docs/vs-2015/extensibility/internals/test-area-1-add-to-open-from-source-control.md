# <a name="test-area-1-add-toopen-from-source-control"></a>テスト領域 1: ソース管理から開くとを追加します。
[!INCLUDE[vs2017banner](../../includes/vs2017banner.md)]

このトピックの最新バージョンをご覧[テスト領域の 1: ソース管理から開くへの追加](https://docs.microsoft.com/visualstudio/extensibility/internals/test-area-1-add-to-open-from-source-control)します。  
  
このソース管理プラグインは、領域ではソース管理下のプロジェクトまたはソリューションを配置して、ソース管理から取得をテストします。  
  
## <a name="command-menu-access"></a>コマンド メニューへのアクセス  
 次[!INCLUDE[vsprvs](../../includes/vsprvs-md.md)]統合開発環境のメニューのパスは、テスト_ケースで使用されます。  
  
-   [!INCLUDE[vsvss](../../includes/vsvss-md.md)]、ソース管理から開く:**ファイル**、**開く**、**プロジェクト**/**ソリューション**記述を探して、で[!INCLUDE[vsvss](../../includes/vsvss-md.md)]場所。  
  
-   その他のソース管理プラグインをソース管理から開く:**ファイル**、**ソース管理**、**ソース管理から開く**します。  
  
-   ソース管理に追加:**ファイル**、**ソース管理**、**ソリューションをソース管理ファイルに追加**、**ソース管理**、**追加選択したプロジェクトをソース管理に**します。  
  
-   ショートカット メニュー (プロジェクト/ソリューション)**ソリューションをソース管理に追加**します。  
  
-   ソース管理から追加:**ファイル**、**ソース管理**、**ソース管理からプロジェクトの追加**します。  
  
-   [!INCLUDE[vsvss](../../includes/vsvss-md.md)]、追加のソースからコントロールが表示されます**ファイル**、**追加**、**既存のプロジェクト**記述を探してで、[!INCLUDE[vsvss](../../includes/vsvss-md.md)]場所。  
  
    > [!NOTE]
    >  ローカル ファイルまたはローカルの IIS (web サーバー) のパスは、このテストで使用できます。  
  
## <a name="expected-behavior"></a>想定される動作  
  
-   ユーザーがサポートされているプロジェクトの種類ごとには、「追加」し、ソース管理から「開く」する必要がありますできます。  
  
-   プロジェクトが、対応するソース管理に追加されたとき\< *ProjectName*> .vspscc ファイル (プロジェクト ヒント ファイル) が作成されます。 除外ファイルの一覧と接続情報が含まれています。 プロジェクトに固有の情報が含まれているために、このファイルは削除しないでください。  
  
-   ソース管理に対応するソリューションを追加するときに\< *SolutionName*> .vssscc (トリプル S) ファイルが作成されます。 テキスト ファイルには、接続情報と除外ファイル リストをプロジェクトのヒント ファイルに似ていますが含まれています。 このファイルは一時的なものと、ソース管理データベースにのみ存在します。  
  
-   ソース管理からソリューションを開いたときに、 \< *SolutionName*> 一時ファイルにのみ、ソース管理データベースに存在する .vsscc (二重 S) ファイルがローカルに作成されます。 このファイルには、ソリューションの接続フォルダーからソリューション ファイルへのパスが含まれています。 このファイルは一時的なものと、「ソース管理からの開いている」の操作が完了したら、ローカルのコピーは削除されます。  
  
-   プロジェクトがソース管理に追加されると、(チェック アウト、Get など) に任意のソース コントロールの操作を実行できます。  
  
## <a name="test-cases"></a>テスト ケース  
 次に、追加の特定のテスト_ケースとテストの領域をソース管理から開く。

|ファイル システムまたはローカルの IIS Web プロジェクトをソース管理を含むソリューションを追加します。|1.ファイル システムまたはローカルの IIS Web プロジェクトの作成 (プロジェクトの場所を指す参照ボタンを使用して Web プロジェクトの種類が作成されたパスを決定します)。<br />2.ソリューションをソース管理に追加 (**ファイル**、**ソース管理**、**ソリューションをソース管理に追加**)。|ソリューション/プロジェクトは、ソース管理に追加されました。|  
|ソース管理にリモート サイトの Web プロジェクトを含むソリューションを追加します。|1.リモート サイトの Web プロジェクトを作成します。<br />2.ソリューションをソース管理に追加 (**ファイル**、**ソース管理**、**ソリューションをソース管理に追加**)。<br />3.クリックして**OK** FrontPage のアクセスの警告 ダイアログ ボックス。|ソリューションは、ソース管理に追加されました。<br /><br /> リモート サイトのプロジェクトはソース管理されません。 (リモート サイトのプロジェクトは独自の IIS サーバーから制御する必要があります)。|  
|1 つのプロジェクト ソリューションをソース コントロールを使用して追加**選択されたプロジェクトをソース管理に追加**します。|1.1 つのプロジェクトのソリューションを作成します。<br />2.選択内容のソース管理にのみのソリューションを追加 (**ファイル**、**ソース管理**、 **選択されたプロジェクトをソース管理に追加**)。 このステップが成功した場合は、次の手順に進みます。<br />3.選択範囲とソース管理にプロジェクトの追加 (**ファイル**、**ソース管理**、 **選択されたプロジェクトをソース管理に追加**)。<br />4.クリックして**はい**同じ場所にプロジェクトを追加します。<br />5.クリックして**チェック アウト**で**編集用にチェック アウト** ダイアログ ボックス。|`Result from Step 2:`<br /><br /> プロジェクトおよびプロジェクト内のすべてのファイルをチェック ソース コントロールのインジケーターを持ち、"ソースの管理"ではなく、ツールヒントに表示されます。<br /><br /> `Result from Step 5:`<br /><br /> プロジェクトとソリューション ファイルは、ソース管理内の同一フォルダーには。|  
|キャンセルのソース管理にソリューションを追加します。|1.1 つのプロジェクトのソリューションを作成します。<br />2.ソース管理にプロジェクトとソリューションを追加しようとしてください。 このステップが成功した場合は、次の手順に進みます。<br />3.ソース管理システムでは後をキャンセルします。|`Result from Step 2:`<br /><br /> プロジェクトの場所のソース コントロールを設定 ダイアログ ボックスは、1 回だけが表示されます。<br /><br /> `Result from Step 3:`<br /><br /> プロジェクト追加が取り消されましたプロジェクト/ソリューション、ソース管理下にない場合、まだ使用できるソース コントロールのメニューにすべてを追加します。|  
  
### <a name="case-1b-open-solution-from-source-control"></a>ケース 1 b。 ソース管理からソリューションを開く  
 このテスト_ケースは、ソース管理からソリューションを開くときに焦点を当てます。  
  
|アクション|テスト ステップ|予想される結果を確認します|  
|------------|----------------|--------------------------------|  
|ソース管理からクライアント プロジェクトを含むソリューションを開く|1.クライアント プロジェクトを作成します。<br />2.ソース管理にソリューションを追加します。<br />3.ソリューションを閉じます。<br />4.新しい場所に、ソース管理からソリューションを開きます。|ソリューション/プロジェクトをソース管理から開きます。|  
|ローカルまたはソース管理からの IIS Web プロジェクトを含むソリューションを開く|1.ローカルまたは IIS Web プロジェクトを作成します。<br />2.ソース管理にソリューションを追加します。<br />3.ソリューションを閉じます。<br />4.新しい場所に、ソース管理からソリューションを開きます。|ソリューション/プロジェクトをソース管理から開きます。|  
|ソース管理からのリモート サイトの Web プロジェクトを含むソリューションを開く|1.リモート サイトの Web プロジェクトを作成します。<br />2.ソース管理にソリューションを追加します。 このステップが成功した場合は、次の手順に進みます。<br />3.ソリューションを閉じます。<br />4.新しい場所に、ソース管理からソリューションを開きます。|`Result from Step 2:`<br /><br /> リモート サイトの Web ソース管理下ではありません。<br /><br /> `Result from Step 4:`<br /><br /> ソース管理から開くソリューションです。<br /><br /> リモート サイトのプロジェクトが読み込まれるが、ソース管理下ではありません。|
