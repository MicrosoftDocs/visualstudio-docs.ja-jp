# <a name="overview-of-the-domain-specific-language-tools-user-interface"></a>ドメイン固有言語ツールのユーザー インターフェイスの概要
ドメイン固有言語ツール (DSL ツール) ソリューションを初めて開く[!INCLUDE[vsprvs](../code-quality/includes/vsprvs_md.md)]、ユーザー インターフェイスには、次の図はようになります。  
  
 ![dsl デザイナー](../modeling/media/dsl_designer.png "dsl_designer")  
  
 次の表では、UI の部分を使用する方法について説明します。  
  
|**要素**|**定義**|  
|-----------------|--------------------|  
|図|図では、ドメイン モデルを表示します。<br /><br /> 図は、2 つの側面があります。 一方の側では、モデル内の要素の型を定義します。 もう一方の側では、画面上のモデルがどのように表示されるかを定義します。|  
|ツールボックス|ドメイン クラスを追加して、ダイアグラムの種類の図形をツールボックスからツールをドラッグします。 リレーションシップ、コネクタ、および図形マップを追加するには、ツールをクリックし、ダイアグラムで、ソース ノードとをターゲット ノードをクリックします。|  
|DSL エクスプローラー|**DSL エクスプ ローラー** DSL 定義が、アクティブなウィンドウが表示されます。 DSL がツリーとして表示されます。 DSL エクスプ ローラーでは、モデルのダイアグラムで表示されていない機能を編集できます。 たとえば、ツールボックス項目を追加し、検証プロセスを使用してスイッチ、 **DSL エクスプ ローラー**します。|  
|DSL 詳細 ウィンドウ|**DSL 詳細**ウィンドウは、要素の表示方法と、要素をコピーおよび削除する方法を制御するためのモデルの要素のドメインのプロパティを示しています。<br /><br /> -既定で、 **DSL 詳細**ウィンドウが横に表示されます、**エラー一覧**と**出力**windows です。|  
  
## <a name="the-domain-model-diagram"></a>ドメイン モデルの図  
 ドメイン モデルの図は、2 つの部分に分かれています。 ダイアグラムの一方の側は、モデル要素および関係を示します。 もう一方の側は、モデルが表示される方法を示しています。 要素とモデル図のプロパティを表示するための図形が含まれています。 次の図は、図の要素を示しています。  
  
 ![スイムレーンを含む dsl デザイナー](../modeling/media/dsl_desinger.png "dsl_desinger")  
  
 次の表では、いくつかのドメイン モデル図の要素について説明します。  
  
|**用語**|**定義**|  
|--------------|--------------------|  
|ドメイン クラス|ドメイン クラスは、モデル内の要素の型です。<br /><br /> ドメイン クラスは、1 つ以上のリレーションシップのターゲットである場合も、図では、複数回にわたって表示できます。<br /><br /> ドメイン クラスを追加するから、ドメイン クラスのツールをドラッグ、**ツールボックス**に、**クラスとリレーションシップ**ダイアグラムの側面です。|  
|ドメインの関係|ドメイン間の関係は、モデル内の要素間のリンクの種類です。<br /><br /> *埋め込みリレーションシップ*実線として表示され、対象となる要素が所有しているか、ソース要素に含まれることを示します。 モデル内のすべての要素は、モデル ツリーを形成するようにの&1; つの埋め込みリレーションシップのターゲットを配置する必要があります。 A*参照リレーションシップ*破線として表示され、モデル要素間の一般的なリンクを示します。 任意の要素には、参照リンクのいくつかが存在します。<br /><br /> ツールをクリックしてリレーションシップを作成、**ツールボックス**をソース ドメイン クラスをクリックし、ターゲット クラスをクリックします。|  
|シェイプとコネクタ|コネクタでは、行を指定する DSL 図の関係の表示に使用することができる図形は、DSL の図にモデル要素を表示する方法を指定します。<br /><br /> 図形またはコネクタを作成するツールをドラッグして、**図要素**ダイアグラムの側面です。|  
|シェイプ マップ|シェイプ マップは、行が表示されるドメイン クラスに図形をリンク ドメイン モデルの図に、表示されるドメイン リレーションシップにコネクタとして表示されます。|
