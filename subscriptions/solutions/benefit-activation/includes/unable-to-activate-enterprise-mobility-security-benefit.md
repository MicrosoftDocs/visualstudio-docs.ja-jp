## <a name="im-unable-to-activate-my-enterprise-mobility--security-ems-benefit"></a>Enterprise Mobility + Security (EMS) 特典をアクティブにすることができません。

2019 年 9 月より、コンプライアンスと情報保護のための Enterprise Mobility + Security が Microsoft 365 E5 開発者サブスクリプションに含まれます。 これは Office 365 E3 開発者サブスクリプションと以前利用できた別個の Enterprise Mobility + Security 特典に代わるものです。 

## <a name="activate-microsoft-365-e5-developer-subscription"></a>Microsoft 365 E5 開発者サブスクリプションをアクティブにする  

1. [サブスクリプション ポータル](https://my.visualstudio.com/benefits)にサインインします。 
1. [ツール] カテゴリ内で Microsoft 365 特典タイルを見つけ、[アクティブにする] をクリックします。 
1. 画面の指示に従い、Microsoft 355 E5 サブスクリプションを設定します。 
1. [Add new users]\(新しいユーザーを追加する\) ページに、ライセンスが割り当てられるすべてのユーザーを追加するために使用できるグリッドが表示されます。 さらに多くのユーザーを追加する必要がある場合は、[+ Add another user]\(+ 別のユーザーを追加する\) をクリックします。 
1. ユーザーを追加した後、チームの作成やアプリケーションのインストールなどの操作を行うことができます。 ここで、これらの操作のいずれかまたはすべてを行うことができます。または、いつでも管理センターに戻って設定することができます。 

## <a name="migrate-from-office-365-e3-to-microsoft-365-e5"></a>Office 365 E3 から Microsoft 365 E5 に移行する 

Visual Studio のサブスクライバーは、Office 365 E3 開発者サンドボックス サブスクリプションから Microsoft 365 E5 に移行できるようになりました。 新しい Microsoft 365 E5 開発者サブスクリプションに移行する場合、今すぐ現在のサブスクリプションを削除し、置換できます。 

新しい Microsoft 365 E5 開発者サブスクリプションを入手するには、ダッシュボードに移動し、次を行います。 
1. サブスクリプション タイルで、[サブスクリプションの削除] をクリックします。 
1. ダイアログ ボックスにドメインの名前を入力し、サブスクリプションを削除することを確認します。 
1. 新しい Microsoft 365 E5 サブスクリプションのセットアップ プロセスが自動的に開始されます。 

Office 365 E3 サブスクリプションを削除し、代わりの Microsoft 365 E5 サブスクリプションを作成したら、Office 365 E3 テナントが 60 日間利用できるのでデータを移行できます。
